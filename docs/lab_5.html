5. 企业破产预警与财务风险识别¶

一、实验简介¶

本实验围绕 企业破产预警（Corporate Bankruptcy Prediction） 场景展开，使用 Logistic 回归、支持向量机（SVM）、随机森林（Random Forest）和 XGBoost 四种模型，基于企业财务报表指标识别企业是否进入财务困境或破产状态，并从解释性与预测性两个维度比较模型表现。

为什么企业破产预警如此重要？

  1. 信贷风险管理：银行在发放企业贷款前需要评估企业的违约概率，破产预警模型是核心工具。
  2. 投资决策：投资者需要识别潜在的"地雷股"，避免投资于即将破产的企业。
  3. 监管合规：金融监管机构要求对系统性风险进行持续监测，企业破产预警是其中的关键环节。

为什么同时使用四种模型？

模型 解释性 预测性 适用场景
Logistic 回归 ⭐⭐⭐⭐⭐ ⭐⭐ 监管报告、系数解释
SVM ⭐⭐ ⭐⭐⭐ 高维小样本场景
Random Forest ⭐⭐⭐ ⭐⭐⭐⭐ 稳健预测 + 特征重要性
XGBoost ⭐⭐⭐ ⭐⭐⭐⭐⭐ 追求最优预测性能

本实验使用 UCI Polish Companies Bankruptcy Data 数据集，通过 scikit-learn 的 OpenML 接口获取。

该数据集关于波兰企业破产预测，破产企业分析期为 2000–2012 年，存续企业评估期为 2007–2013 年。

数据包含 64 个财务比率特征，涵盖多个预测年份子集。

二、实验目标¶

  • 理解 Logistic 回归如何通过 对数几率（Log-Odds） 将线性模型用于分类，并通过系数符号与大小解释变量影响方向。
  • 掌握 SVM 的 最大间隔（Maximum Margin） 思想，以及核函数（Kernel）如何处理非线性问题。
  • 理解随机森林的 Bagging + 特征随机采样 机制与 XGBoost 的 梯度提升残差拟合 策略。
  • 学会处理 严重类别不平衡（~5% 破产 vs ~95% 存续） 和 大量缺失值 的实际数据问题。
  • 从解释性与预测性两个维度系统比较四种模型，输出企业破产预警报告。

三、实验要求¶

  • 使用 OpenML 获取 Polish Companies Bankruptcy 数据集。
  • 完成数据预处理：缺失值填充、异常值处理、类别不平衡应对。
  • 分别构建 Logistic 回归、SVM、Random Forest 和 XGBoost 模型。
  • 对四个模型进行系统性评估与对比（重点关注 AUC-ROC 和 Recall）。
  • 提取关键财务变量，生成财务指标解释图和破产预警报告。

四、思考提示¶

  1. 在极端不平衡的数据（破产企业仅占 ~5%）中，Accuracy 还有参考价值吗？
  2. Logistic 回归的系数为正/为负分别意味着什么？如何据此给出财务建议？
  3. SVM 使用 RBF 核时，为什么必须先对特征进行标准化？
  4. 随机森林和 XGBoost 在处理缺失值方面有什么不同？
  5. 如果你是银行的风控经理，你会选择哪个模型部署？为什么？

2.4 数据预处理¶

对于缺失值，我们采用 中位数填充（Median Imputation） 策略。中位数比均值更鲁棒，不容易受极端值的影响——而财务数据中经常出现极端值（如即将破产的企业可能有极端的负债率）。

同时，我们裁剪极端异常值，避免其对 Logistic 回归和 SVM 产生过大影响。

3. 核心概念¶

3.1 Logistic 回归（Logit 模型）¶

Logistic 回归是金融领域最经典的信用风险模型，其核心思想是：

$$P(\text{破产} = 1 \mid \mathbf{x}) = \frac{1}{1 + e^{-(\beta_0 + \beta_1 x_1 + \cdots + \beta_p x_p)}}$$

关键优势：系数可直接解释。

  • $\beta_i > 0$：该指标越大，破产概率越高（如负债率）。
  • $\beta_i < 0$：该指标越大，破产概率越低（如利润率）。
  • $e^{\beta_i}$ 是优势比（Odds Ratio），表示该指标每增加一个单位，破产的"赔率"变化多少倍。

这种可解释性使 Logistic 回归至今仍是巴塞尔协议框架下的标准模型。

3.2 支持向量机（SVM）¶

SVM 的核心思想是寻找一个最大间隔超平面，使得两类样本之间的"距离"最大。

  • 线性 SVM：当数据线性可分时直接找超平面。
  • 核 SVM（Kernel SVM）：使用核函数（如 RBF）将数据映射到高维空间，在高维空间中找线性分离面。

注意：SVM 对特征尺度非常敏感，因此必须先进行标准化（StandardScaler）。

3.3 随机森林与 XGBoost¶

（与实验 1 类似，此处简要回顾）

  • 随机森林：通过 Bagging 策略并行训练多棵决策树，降低方差。对缺失值和异常值鲁棒。
  • XGBoost：通过梯度提升串行训练，每棵新树拟合前一轮的残差。支持正则化，内置缺失值处理。

3.4 类别不平衡问题¶

本数据集中破产企业仅占约 5%，这是典型的 类别不平衡（Class Imbalance） 问题。如果不加处理，模型会倾向于将所有样本预测为"存续"，获得 ~95% 的准确率但完全无法识别破产企业。

应对策略：

  • class_weight="balanced"：自动根据类别频率调整损失函数中的权重，少数类获得更高权重。
  • scale_pos_weight（XGBoost 专用）：直接设置正类权重比例。
  • 评估指标选择：放弃 Accuracy，重点关注 AUC-ROC、Recall 和 F1。

4. 构建 Logistic 回归模型¶

4.1 训练 Logistic 回归¶

Logistic 回归要求特征尺度一致，因此我们使用 Pipeline 将 StandardScaler 和 LogisticRegression 组合在一起。

In [11]:
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.linear_model.LogisticRegression.html
logit_clf = make_pipeline(
    StandardScaler(),
    LogisticRegression(
        max_iter=1000,
        class_weight="balanced",  # 处理类别不平衡
        C=1.0,                    # 正则化强度（越小正则化越强）
        solver="lbfgs",
        random_state=42,
    ),
)
logit_clf.fit(X_train, y_train)

y_pred_logit = logit_clf.predict(X_test)
y_prob_logit = logit_clf.predict_proba(X_test)[:, 1]

print("=== Logistic 回归模型评估 ===")
print(classification_report(y_test, y_pred_logit, target_names=["存续", "破产"]))
print(f"AUC-ROC: {roc_auc_score(y_test, y_prob_logit):.4f}")

5. 构建 SVM 模型¶

5.1 训练 SVM¶

SVM 对特征尺度敏感，同样需要先标准化。我们使用 RBF 核函数，并启用 probability=True 以获取概率估计（用于计算 AUC）。

注意：SVM 在大样本数据上训练较慢（时间复杂度约 $O(n^2)$ 到 $O(n^3)$），因此对于本数据集（~10000 样本），训练可能需要几分钟。

In [13]:
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.svm.SVC.html
svm_clf = make_pipeline(
    StandardScaler(),
    SVC(
        kernel="rbf",
        C=1.0,
        gamma="scale",
        class_weight="balanced",
        probability=True,   # 启用概率估计（用于 AUC 计算）
        random_state=42,
    ),
)
svm_clf.fit(X_train, y_train)

y_pred_svm = svm_clf.predict(X_test)
y_prob_svm = svm_clf.predict_proba(X_test)[:, 1]

print("=== SVM 模型评估 ===")
print(classification_report(y_test, y_pred_svm, target_names=["存续", "破产"]))
print(f"AUC-ROC: {roc_auc_score(y_test, y_prob_svm):.4f}")

10.3 预警阈值分析¶

不同的预警阈值会影响模型的 Precision 和 Recall 表现。银行需要根据业务需求选择合适的阈值。

thresholds = np.arange(0.1, 0.9, 0.05)

for t in thresholds:

业务决策：如果银行对破产风险零容忍（宁可多查也不能漏掉），应降低阈值以提高 Recall；如果银行资源有限、不希望误判太多存续企业，应提高阈值以提高 Precision。

11. 实验结论¶

11.1 四种模型的比较总结¶

  • Logistic 回归：解释性最强，系数直接对应财务指标的影响方向与大小，是监管合规的首选。但在处理非线性关系和特征交互时表现有限。
  • SVM：在高维空间中寻找最优分离面，理论优美，但在大样本数据上训练较慢，且模型本身缺乏解释性。
  • 随机森林：性能稳健，对异常值和缺失值鲁棒，特征重要性可靠。是"可解释性与预测性"之间的良好折中。
  • XGBoost：通常预测性能最优，支持正则化和缺失值处理，是工业级风控系统的首选。

11.2 关键财务指标的启示¶

通过三种模型的交叉验证，最重要的财务指标通常包括：

  • 盈利能力指标（如净利润/总资产）：破产企业普遍盈利能力差。
  • 杠杆率指标（如总负债/总资产）：破产企业负债水平过高。
  • 流动性指标（如营运资金/总资产）：破产企业短期偿债能力不足。

11.3 解释性 vs 预测性的取舍¶

在实际业务中，不存在"一刀切"的最优模型。常见的做法是：

  • 对外报告使用 Logistic 回归（满足监管要求的可解释性）。
  • 内部决策使用 XGBoost（追求最优预测性能）。
  • 两者结合：用 XGBoost 做预测，用 Logistic 回归系数和树模型特征重要性做解释。
