# 4.5 股票收益预测与量化投资案例

股票收益预测是金融智能分析中最具挑战性的任务之一。价格序列同时受到宏观经济、行业景气、公司基本面、投资者情绪、资金流动、市场微观结构和突发事件影响。即使模型在历史数据中表现良好，也可能因为市场状态变化而迅速失效。因此，本章不把股票预测理解为简单的曲线拟合问题，而把它理解为一个必须严格控制前视偏差、必须采用样本外验证、必须结合交易成本和风险控制的量化研究流程。

本章围绕股票或指数的未来收益率方向预测展开。数据来源为 Yahoo Finance 公开行情接口，示例资产为标普 500 指数 `^GSPC`。模型部分使用 Random Forest、XGBoost 和 LSTM 三类方法。特征部分以技术指标为主，包括移动平均、指数移动平均、MACD、RSI、布林带、波动率、成交量和滞后收益率。验证部分采用滚动窗口，也称 Walk-Forward Validation，用过去数据训练模型，用未来窗口进行样本外预测。策略部分把方向预测转化为简单持有或空仓信号，并用累积收益、年度准确率和风险指标进行分析。

本章的目标不是证明某个模型一定可以稳定战胜市场，而是建立一个规范的量化建模范式。学生需要理解，金融时间序列建模中最常见的错误不是模型不够复杂，而是目标定义错误、样本划分错误、数据泄漏、回测假设过于乐观，以及忽视交易成本和模型衰减。

## 4.5.1 技术指标提取与因子构建

### 4.5.1.1 为什么预测收益率方向而不是直接拟合收盘价

许多初学者会用前若干日收盘价预测下一日收盘价，并且因为价格序列具有强自相关而得到看似很小的误差。这类模型通常学到的是“明天价格接近今天价格”。如果今天收盘价为 5000 点，明天收盘价大概率仍在 5000 点附近，模型只要输出一个接近当前价格的数值，就会得到较高的拟合精度。但是，这种精度对交易决策帮助有限，因为交易更关心的是未来收益率是否为正、收益幅度能否覆盖成本、风险暴露是否可控。

收益率方向预测把问题转化为二分类问题。若未来一期收益率为正，则标签为 1，表示上涨；若未来一期收益率小于或等于 0，则标签为 0，表示下跌或持平。这个目标直接对应交易动作。预测为上涨时可以持有或增配，预测为下跌时可以空仓、减仓或做空。虽然方向预测比价格拟合更难，但它更接近量化投资中的决策目标。

设 `P_t` 为第 t 日收盘价。对数收益率定义为

$$r_t=\ln\left(\frac{P_t}{P_{t-1}}\right)$$

未来一日方向标签定义为

$$y_t=\begin{cases}1,& r_{t+1}>0\\0,& r_{t+1}\le 0\end{cases}$$

对数收益率具有可加性。例如从 t 到 t+2 的对数收益率等于两个单日对数收益率之和。这一性质使它在统计建模和回测中比简单收益率更方便。但在向投资者报告收益时，通常还需要把对数收益率转换为普通收益率。

### 4.5.1.4 收益率与预测标签

```python
df = raw[["Open", "High", "Low", "Close", "Volume"]].copy()

df["log_return"] = np.log(df["Close"] / df["Close"].shift(1))

df["target"] = (df["log_return"].shift(-1) > 0).astype(int)

print(f"日收益率统计：")
display(df["log_return"].describe())
print(f"\n目标变量分布（0=下跌/持平, 1=上涨）：")
print(df["target"].value_counts())
print(f"上涨比例：{df['target'].mean():.4f}")
```

`df = raw[[...]].copy()` 保留建模需要的 OHLCV 字段。OHLCV 分别指开盘价、最高价、最低价、收盘价和成交量。`np.log(df["Close"] / df["Close"].shift(1))` 计算当日对数收益率，其中 `shift(1)` 表示上一交易日收盘价。第一行收益率会因为没有上一日价格而变成缺失值。

`df["log_return"].shift(-1)` 把下一交易日收益率移动到当前行，因此当前行特征对应未来一日方向标签。若下一日对数收益率大于 0，则目标变量为 1；否则为 0。这里的 `shift(-1)` 只能用于生成标签，不能用于生成特征。若把未来收益率或未来价格用于特征，就会产生前视偏差。

实验结果中，日收益率均值通常为一个很小的正数，标准差约为 1% 左右。上涨标签比例可能略高于 0.5。方向预测的基准不是机械的 50%，而应包括“永远预测上涨”的多数类基准。因此，若上涨比例为 54%，模型方向准确率达到 52% 反而可能低于多数类基准。模型评估必须同时观察 Precision、Recall、AUC 和策略回测。

### 4.5.1.5 技术指标作为量化因子

技术指标把历史价格和成交量转化为趋势、动量、波动和交易活跃度特征。它们不是神秘信号，而是历史市场信息的函数。技术指标能够被机器学习模型使用，是因为市场可能存在短期动量、均值回复、波动聚集和成交量确认等统计现象。但这些现象并不稳定，且很容易受到交易成本和市场拥挤影响。

简单移动平均线定义为

$$SMA_t^{(n)}=\frac{1}{n}\sum_{i=0}^{n-1}P_{t-i}$$

它用最近 n 日价格均值表示趋势水平。价格高于短期或长期均线时，通常表示当前价格强于过去平均水平。

```python
for window in [5, 10, 20, 60]:
    df[f"SMA_{window}"] = df["Close"].rolling(window).mean()

for window in [5, 10, 20, 60]:
    df[f"Close_SMA{window}_ratio"] = df["Close"] / df[f"SMA_{window}"]
```

第一段循环计算 5 日、10 日、20 日和 60 日简单移动平均。`rolling(window).mean()` 只使用当前日及之前 window 个观测值，因此不会使用未来数据。第二段循环计算收盘价相对于均线的位置。若 `Close_SMA20_ratio` 大于 1，表示收盘价高于 20 日均线；若小于 1，表示收盘价低于 20 日均线。使用比值而不是直接使用均线，可以减弱价格量纲变化的影响。

指数移动平均线给予近期价格更高权重。递推形式为

$$EMA_t=\alpha P_t+(1-\alpha)EMA_{t-1}$$

其中，`α=2/(span+1)`。短期 EMA 对价格变化更敏感，长期 EMA 更平滑。

```python
for span in [12, 26]:
    df[f"EMA_{span}"] = df["Close"].ewm(span=span, adjust=False).mean()
```

`ewm(span=span, adjust=False).mean()` 计算指数加权均值。`span=12` 对应较快的均线，`span=26` 对应较慢的均线。EMA 常用于构建 MACD。

MACD 由快线、信号线和柱状图构成。快线是短期 EMA 与长期 EMA 的差。信号线是快线的 9 日 EMA。柱状图是快线与信号线的差。

```python
df["MACD"] = df["EMA_12"] - df["EMA_26"]
df["MACD_signal"] = df["MACD"].ewm(span=9, adjust=False).mean()
df["MACD_hist"] = df["MACD"] - df["MACD_signal"]
```

`MACD` 为正，说明短期趋势强于长期趋势。`MACD_signal` 对 MACD 做平滑。`MACD_hist` 可以理解为趋势动量的加速或减速。MACD 适合刻画趋势变化，但在震荡市场中容易产生频繁噪声信号。

RSI 衡量一段时间内上涨幅度与下跌幅度的相对强弱。常见周期为 14 日。若 RSI 高于 70，技术分析中通常称为超买；若 RSI 低于 30，通常称为超卖。超买不等于一定下跌，超卖也不等于一定上涨，它们只是描述当前价格变化在历史窗口中的相对位置。

```python
def compute_rsi(series: pd.Series, period: int = 14) -> pd.Series:
    """计算 RSI 指标。"""
    delta = series.diff()
    gain = delta.clip(lower=0)
    loss = (-delta).clip(lower=0)
    avg_gain = gain.rolling(period).mean()
    avg_loss = loss.rolling(period).mean()
    rs = avg_gain / avg_loss
    return 100 - (100 / (1 + rs))


df["RSI_14"] = compute_rsi(df["Close"], 14)
```

`series.diff()` 计算价格变动。`delta.clip(lower=0)` 保留上涨部分，把下跌部分置为 0。`(-delta).clip(lower=0)` 保留下跌幅度的正数形式。`rolling(period).mean()` 分别计算平均上涨幅度和平均下跌幅度。`rs` 是相对强度。最后一行按 RSI 公式返回 0 到 100 之间的指标。若平均下跌幅度为 0，RSI 可能出现无穷或缺失，后续清洗会处理这些值。

布林带由中轨、上轨和下轨构成。中轨通常为 20 日均线，上轨和下轨分别为中轨加减 2 倍标准差。

```python
bb_window = 20
df["BB_mid"] = df["Close"].rolling(bb_window).mean()
bb_std = df["Close"].rolling(bb_window).std()
df["BB_upper"] = df["BB_mid"] + 2 * bb_std
df["BB_lower"] = df["BB_mid"] - 2 * bb_std

df["BB_pctB"] = (df["Close"] - df["BB_lower"]) / (df["BB_upper"] - df["BB_lower"])
```

`BB_mid` 是布林带中轨。`bb_std` 是 20 日价格标准差。`BB_upper` 和 `BB_lower` 形成价格波动通道。`BB_pctB` 表示价格在上下轨之间的相对位置。若 `BB_pctB` 接近 1，说明价格接近上轨；若接近 0，说明价格接近下轨；若超过 1 或低于 0，说明价格突破布林带。

波动率是风险的核心度量。金融收益率常出现波动聚集，即高波动之后仍可能高波动，低波动之后仍可能低波动。

```python
for window in [5, 10, 20]:
    df[f"volatility_{window}"] = df["log_return"].rolling(window).std()
```

这段代码计算 5 日、10 日和 20 日历史波动率。`rolling(window).std()` 使用历史窗口内的收益率标准差。高波动阶段方向预测通常更困难，也更需要风险控制。

成交量反映市场参与活跃程度。价格上涨若伴随成交量放大，可能表示趋势受到更多资金确认。价格上涨但成交量萎缩，则可能表示趋势动能不足。成交量本身也存在强烈的制度和市场状态差异，因此通常需要标准化。

```python
df["volume_change"] = df["Volume"].pct_change()

df["volume_SMA20_ratio"] = df["Volume"] / df["Volume"].rolling(20).mean()
```

`pct_change()` 计算成交量相对上一日的变化率。`volume_SMA20_ratio` 表示当日成交量相对于 20 日平均成交量的倍数。若该比值大于 1，表示成交量高于近期均量。

滞后收益率特征用于刻画短期动量或短期反转。如果过去几日上涨后未来仍容易上涨，则存在动量信号。如果过去几日上涨后未来更容易下跌，则存在均值回复信号。

```python
for lag in [1, 2, 3, 5, 10]:
    df[f"return_lag_{lag}"] = df["log_return"].shift(lag)
```

`shift(lag)` 把过去 lag 日收益率移动到当前行。`return_lag_1` 是上一交易日收益率，`return_lag_10` 是十个交易日前的收益率。这里使用正向 lag，因此不会使用未来信息。

### 4.5.1.6 最终特征集与数据清洗

```python
feature_cols = [
    "Close_SMA5_ratio", "Close_SMA10_ratio", "Close_SMA20_ratio", "Close_SMA60_ratio",
    "MACD", "MACD_signal", "MACD_hist",
    "RSI_14",
    "BB_pctB",
    "volatility_5", "volatility_10", "volatility_20",
    "volume_change", "volume_SMA20_ratio",
    "return_lag_1", "return_lag_2", "return_lag_3", "return_lag_5", "return_lag_10",
]

df_model = df[feature_cols + ["target"]].copy()

df_model.replace([np.inf, -np.inf], np.nan, inplace=True)
df_model.dropna(inplace=True)

print(f"最终建模数据集维度：{df_model.shape}")
print(f"时间区间：{df_model.index.min().date()} — {df_model.index.max().date()}")
print(f"特征数量：{len(feature_cols)}")
print(f"\n特征列表：")
for i, col in enumerate(feature_cols, 1):
    print(f"  {i:2d}. {col}")
```

`feature_cols` 明确列出所有输入特征。本章共使用 19 个特征，覆盖趋势、动量、波动、成交量和滞后收益率。`df_model` 只保留特征和目标变量，避免原始价格列直接进入模型。直接使用价格水平可能让模型学习到长期趋势或量纲变化，而不是可迁移的预测关系。

`replace([np.inf, -np.inf], np.nan)` 将无穷值替换为缺失值。无穷值可能来自除零，例如布林带上下轨相同或 RSI 平均下跌幅度为 0。`dropna(inplace=True)` 删除由于滚动窗口、滞后项和无穷替换产生的缺失行。实验结果中，最终建模数据通常约为 4218 行和 20 列，其中 19 列为特征，1 列为目标变量。

`corr()` 计算特征之间的 Pearson 相关系数。`np.triu` 生成上三角遮罩，因为相关性矩阵是对称的，只显示一半即可。`cmap="RdBu_r"` 使用红蓝配色，`center=0` 让正相关和负相关围绕 0 对称显示。

```python
high_corr_pairs = []
for i in range(len(corr_matrix)):
    for j in range(i + 1, len(corr_matrix)):
        if abs(corr_matrix.iloc[i, j]) > 0.8:
            high_corr_pairs.append(
                (corr_matrix.index[i], corr_matrix.columns[j],
                 round(corr_matrix.iloc[i, j], 3))
            )

if high_corr_pairs:
    print("相关系数绝对值 > 0.8 的特征对：")
    for f1, f2, c in high_corr_pairs:

else:

```

这段代码逐对检查特征相关性。若相关系数绝对值超过 0.8，就记录为高度相关特征对。实验结果中，短期和中期均线比率、MACD 与信号线、不同窗口波动率之间通常存在较高相关性。树模型对多重共线性不如线性模型敏感，因此不一定需要删除这些特征。若使用 Logistic 回归、线性因子模型或需要解释系数大小的模型，就应考虑降维、正则化或删除冗余特征。

技术指标特征的局限需要明确。第一，它们几乎全部由历史价格和成交量变换而来，信息源有限。第二，技术指标参数具有任意性，例如 14 日 RSI 或 20 日布林带并非理论上唯一正确。第三，若大量尝试指标组合并只报告最优结果，会产生数据挖掘偏差。第四，技术因子一旦被大量投资者使用，可能因为拥挤交易而衰减。

## 4.5.2 预测模型设计与策略回测

### 4.5.2.1 时间序列为什么不能随机划分

普通分类任务经常使用随机训练测试划分。但金融时间序列具有时间顺序，随机划分会让训练集中包含测试日期之后的数据。即使特征本身没有直接使用未来值，模型也会从未来市场状态中学习参数，从而高估真实样本外能力。

合理的方式是 Walk-Forward Validation。每一轮只使用预测窗口之前的数据训练模型，再对未来一段时间进行预测。预测窗口结束后，把新数据纳入训练集，继续向前滚动。这一过程模拟真实交易中不断更新模型的方式。

设训练截止日为 `T_i`，预测窗口为 `(T_i, T_{i+1}]`。模型训练只允许使用 `t <= T_i` 的样本，预测只发生在 `T_i < t <= T_{i+1}`。若标准化器、特征选择器、模型参数或阈值调优使用了预测窗口数据，也会形成信息泄漏。

### 4.5.2.2 滚动窗口参数

```python
TRAIN_START = "2010-01-01"
PREDICT_START = "2018-01-01"
STEP_MONTHS = 6
LOOKBACK = 20

print(f"训练数据起始：{TRAIN_START}")
print(f"样本外预测起始：{PREDICT_START}")
print(f"预测窗口步长：{STEP_MONTHS} 个月")
print(f"LSTM 回看天数：{LOOKBACK}")
```

`TRAIN_START` 表示最早使用的训练数据日期。`PREDICT_START` 表示从 2018 年开始进行样本外预测。这样做使模型先拥有 2010 至 2017 年的历史数据，再从 2018 年开始接受样本外检验。`STEP_MONTHS=6` 表示每个预测窗口约为半年。`LOOKBACK=20` 表示 LSTM 每个样本使用过去 20 个交易日的特征序列。

在真实研究中，窗口长度是一项重要设定。训练历史越长，样本越多，但包含的旧市场状态可能与当前市场不一致。训练历史越短，模型更贴近当前市场，但样本量不足、参数估计更不稳定。本章使用扩展窗口，即随着时间推进训练集逐步扩大。

```python
predict_start_ts = pd.Timestamp(PREDICT_START)
data_end_ts = df_model.index.max()

fold_dates = []
current = predict_start_ts
while current < data_end_ts:
    next_point = current + pd.DateOffset(months=STEP_MONTHS)
    if next_point > data_end_ts:
        next_point = data_end_ts
    fold_dates.append((current, next_point))
    current = next_point

print(f"共 {len(fold_dates)} 个预测窗口：")
for i, (start, end) in enumerate(fold_dates):
    print(f"  窗口 {i + 1}: {start.date()} — {end.date()}")
```

`pd.Timestamp` 把字符串日期转换为 pandas 时间戳。`data_end_ts` 是建模数据最后日期。`fold_dates` 存储所有预测窗口。循环中，`next_point` 是当前预测窗口结束日期。若它超过数据最后日期，则使用数据最后日期作为终点。每次循环后，`current` 移动到下一窗口起点。实验中通常得到 14 个半年窗口，覆盖 2018 年初至 2024 年底。

### 4.5.2.3 Random Forest、XGBoost 与 LSTM 的选择理由

Random Forest 是多个决策树的集成模型。每棵树在不同样本和不同特征子集上训练，最后通过投票或平均得到预测结果。它的优点是稳健、训练速度快、对异常值和非线性关系有较好适应性，并能输出特征重要性。它的局限是无法天然建模时间顺序，必须依赖人工构造滞后特征和技术指标。

XGBoost 是梯度提升树模型。它通过逐步拟合前一轮模型的残差来提高预测能力，并使用正则化、学习率、子采样和树深度控制过拟合。XGBoost 在结构化表格数据上表现强，因此适合作为量化因子建模的强基线。它的局限与其他树模型类似，不能直接理解序列结构，且超参数选择会影响结果。

LSTM 是循环神经网络的一种。它通过输入门、遗忘门和输出门控制信息在时间上的保留与更新，理论上能够捕捉较长时间依赖。金融时间序列中，LSTM 的输入通常是三维张量，形状为 `(样本数, 时间步长, 特征数)`。它的局限是训练慢、容易过拟合、对超参数敏感，并不保证在日频金融噪声数据上优于树模型。

本章同时使用这三类模型，是为了展示从稳健基线到强表格模型再到序列模型的比较思路。模型复杂度不是目标，样本外经济效果才是目标。

### 4.5.2.4 LSTM 序列数据构造

```python
def create_lstm_sequences(
    X: np.ndarray, y: np.ndarray, lookback: int
) -> tuple[np.ndarray, np.ndarray]:
    """
    将二维特征矩阵转为 LSTM 所需的三维序列。
    
    参数:
        X: (n_samples, n_features)
        y: (n_samples,)
        lookback: 回看天数
    
    返回:
        X_seq: (n_samples - lookback, lookback, n_features)
        y_seq: (n_samples - lookback,)
    """
    X_seq, y_seq = [], []
    for i in range(lookback, len(X)):
        X_seq.append(X[i - lookback : i])
        y_seq.append(y[i])
    return np.array(X_seq), np.array(y_seq)
```

`X` 是二维特征矩阵。每一行对应一个交易日，每一列对应一个特征。`y` 是方向标签。`lookback` 是回看天数。函数内部创建两个列表。循环从 `lookback` 开始，因为前 lookback 行之前没有足够的历史序列。`X[i - lookback : i]` 取第 i 行之前的 lookback 天特征，形成一个序列样本。`y[i]` 是该序列对应的标签。函数最后把列表转换为 numpy 数组。

需要注意，当前写法使用第 i 行之前的 lookback 天特征预测第 i 行标签。由于标签本身已经表示下一日收益率方向，因此序列对应的预测时点要与前文标签定义保持一致。只要特征不使用未来数据，这种构造是合法的。

### 4.5.2.5 LSTM 模型结构

```python
class LSTMClassifier(nn.Module):
    """两层 LSTM + 全连接层，输出二分类概率。"""

    def __init__(self, n_features: int, hidden_size: int = 64, num_layers: int = 2,
                 dropout: float = 0.3):
        super().__init__()
        # Ref: https://pytorch.org/docs/stable/generated/torch.nn.LSTM.html
        self.lstm = nn.LSTM(
            input_size=n_features,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout,
        )
        self.fc = nn.Linear(hidden_size, 1)
```

`LSTMClassifier` 继承自 `nn.Module`。`n_features` 是每个时间步的特征数量，本章为 19。`hidden_size=64` 表示 LSTM 隐状态维度为 64。`num_layers=2` 表示堆叠两层 LSTM。`dropout=0.3` 在多层 LSTM 的层间使用随机失活，以减轻过拟合。`batch_first=True` 表示输入张量形状为 `(batch, seq_len, features)`。`self.fc` 是全连接层，把最后时间步的隐藏状态映射为一个二分类 logit。

```python
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        lstm_out, _ = self.lstm(x)
        last_hidden = lstm_out[:, -1, :]
        logit = self.fc(last_hidden)
        return logit.squeeze(-1)
```

`forward` 定义模型前向传播。`self.lstm(x)` 返回所有时间步的输出和最终隐藏状态。这里使用 `lstm_out[:, -1, :]` 取最后一个时间步输出，因为它汇总了过去 lookback 天的信息。`self.fc(last_hidden)` 输出形状为 `(batch, 1)` 的 logit。`squeeze(-1)` 删除最后一维，得到 `(batch,)`。后续使用 `BCEWithLogitsLoss`，因此这里不提前使用 sigmoid。

### 4.5.2.6 LSTM 训练函数

```python
def train_lstm(
    X_train_seq: np.ndarray,
    y_train_seq: np.ndarray,
    n_features: int,
    epochs: int = 30,
    batch_size: int = 64,
    patience: int = 5,
    lr: float = 1e-3,
) -> LSTMClassifier:
    """训练 LSTM 模型，支持 early stopping。"""
    val_size = max(1, int(len(X_train_seq) * 0.1))
    X_val_t = torch.tensor(X_train_seq[-val_size:], dtype=torch.float32).to(device)
    y_val_t = torch.tensor(y_train_seq[-val_size:], dtype=torch.float32).to(device)
    X_tr_t = torch.tensor(X_train_seq[:-val_size], dtype=torch.float32).to(device)
    y_tr_t = torch.tensor(y_train_seq[:-val_size], dtype=torch.float32).to(device)
```

函数接收训练序列、训练标签和特征数量。`epochs` 是最大训练轮数。`batch_size` 是每批样本数。`patience` 是早停等待轮数。`lr` 是学习率。`val_size` 取最后 10% 训练序列作为验证集。时间序列中验证集应位于训练集末端，而不是随机抽样，否则仍可能破坏时间结构。`torch.tensor(...).to(device)` 把 numpy 数组转为 PyTorch 张量，并移动到 GPU 或 CPU。

```python
    train_loader = DataLoader(
        TensorDataset(X_tr_t, y_tr_t), batch_size=batch_size, shuffle=True,
    )

    model = LSTMClassifier(n_features).to(device)
    # Ref: https://pytorch.org/docs/stable/generated/torch.nn.BCEWithLogitsLoss.html
    criterion = nn.BCEWithLogitsLoss()
    optimizer = torch.optim.Adam(model.parameters(), lr=lr)
```

`TensorDataset` 把训练特征和标签绑定成数据集。`DataLoader` 按批次迭代训练样本。这里设置 `shuffle=True`，表示在训练集内部打乱批次顺序。由于 LSTM 的序列已经在每个样本内部保留时间顺序，打乱样本顺序通常不会造成前视偏差。`model` 是 LSTM 分类器。`BCEWithLogitsLoss` 适用于二分类 logit 输出，它内部会结合 sigmoid 和二元交叉熵。`Adam` 是常用自适应梯度优化器。

```python
    best_val_loss = float("inf")
    best_state = None
    wait = 0

    for epoch in range(epochs):
        model.train()
        for X_batch, y_batch in train_loader:
            optimizer.zero_grad()
            logits = model(X_batch)
            loss = criterion(logits, y_batch)
            loss.backward()
            optimizer.step()
```

`best_val_loss` 记录当前最佳验证损失。`best_state` 保存最佳模型权重。`wait` 记录连续多少轮没有改善。每个 epoch 中，`model.train()` 启用训练模式。`optimizer.zero_grad()` 清空上一批次梯度。`logits = model(X_batch)` 得到预测 logit。`loss = criterion(logits, y_batch)` 计算损失。`loss.backward()` 反向传播。`optimizer.step()` 更新参数。

```python
        model.eval()
        with torch.no_grad():
            val_logits = model(X_val_t)
            val_loss = criterion(val_logits, y_val_t).item()

        if val_loss < best_val_loss:
            best_val_loss = val_loss
            best_state = model.state_dict().copy()
            wait = 0
        else:
            wait += 1
            if wait >= patience:
                break

    if best_state is not None:
        model.load_state_dict(best_state)
    model.eval()
    return model
```

验证阶段使用 `model.eval()`，关闭 dropout 等训练行为。`torch.no_grad()` 避免记录梯度，节省内存和计算。若验证损失改善，则保存权重并重置等待次数。若连续 `patience` 轮没有改善，则提前停止训练。最后恢复最佳权重，并返回评估模式下的模型。早停有助于降低 LSTM 在小样本金融数据上的过拟合。

```python
def predict_lstm(model: LSTMClassifier, X_seq: np.ndarray) -> np.ndarray:
    """用训练好的 LSTM 输出预测概率。"""
    model.eval()
    X_t = torch.tensor(X_seq, dtype=torch.float32).to(device)
    with torch.no_grad():
        logits = model(X_t)
        probs = torch.sigmoid(logits).cpu().numpy()
    return probs
```

`predict_lstm` 把测试序列转换为张量，输入模型得到 logit，再用 `torch.sigmoid` 转换为上涨概率。`.cpu().numpy()` 把结果移回 CPU 并转换为 numpy 数组。输出概率可以用于 AUC、阈值调整和交易信号生成。

```python
    test_dates = test_data.index[-n_lstm:]

    all_results["date"].extend(test_dates.tolist())
    all_results["y_true"].extend(y_test[-n_lstm:].tolist())
    all_results["rf_prob"].extend(rf_prob[-n_lstm:].tolist())
    all_results["xgb_prob"].extend(xgb_prob[-n_lstm:].tolist())
    all_results["lstm_prob"].extend(lstm_prob.tolist())

print(f"\n滚动预测完成，共获得 {len(all_results['date'])} 条样本外预测。")
```

由于 LSTM 因回看窗口损失前 `LOOKBACK` 条测试样本，三类模型需要对齐到同一组日期。`n_lstm` 是 LSTM 有效预测数量。`test_data.index[-n_lstm:]` 取测试窗口最后 n_lstm 个日期。Random Forest 和 XGBoost 原本对整个测试窗口都有预测，这里也取最后 n_lstm 条，以保证三模型在完全相同日期上比较。实验结果中，滚动预测通常得到约 1484 条样本外预测。

方向准确率定义为

$$Accuracy=\frac{TP+TN}{TP+TN+FP+FN}$$

Precision 定义为

$$Precision=\frac{TP}{TP+FP}$$

Recall 定义为

$$Recall=\frac{TP}{TP+FN}$$

F1 定义为

$$F1=\frac{2\times Precision\times Recall}{Precision+Recall}$$

AUC 衡量模型概率排序能力。若 AUC 等于 0.5，说明排序能力接近随机。若 AUC 大于 0.5，说明模型倾向于给真实上涨日更高概率。金融场景中，AUC 略高于 0.5 也可能有价值，但必须经过成本和风险检验。

### 4.5.2.11 策略回测的基本形式

预测模型只有转化为交易规则，才能讨论经济价值。本章使用一个简化策略。若模型预测上涨，则持有指数；若预测下跌或持平，则空仓。策略收益为

$$R_t^{strategy}=signal_t\times r_{t+1}$$

其中，`signal_t` 为 1 表示持有，为 0 表示空仓。累积对数收益为

$$CR_T=\sum_{t=1}^{T}R_t^{strategy}$$

```python
results_df["actual_return"] = df.loc[results_df.index, "log_return"].shift(-1)
```

这只是教学回测。它没有考虑手续费、滑点、买卖价差、冲击成本、融资融券约束、税费、资金占用收益、涨跌停限制、不可交易日期和成交量约束。若把它用于真实交易评估，结果会过于乐观。一个方向准确率略高于 50% 的日频策略，在加入交易成本后可能不盈利。

### 4.5.2.13 回测指标扩展

为了更接近量化投资实践，可以在累积收益之外补充年化收益、年化波动率、夏普比率、最大回撤和交易次数。夏普比率的简化形式为

$$Sharpe=\frac{E[R_t]}{Std(R_t)}\sqrt{252}$$

最大回撤衡量策略从历史高点到之后低点的最大损失幅度。

```python
def evaluate_strategy_returns(return_series: pd.Series) -> dict:
    """计算简化策略评价指标。"""
    clean_ret = return_series.dropna()
    if clean_ret.empty:
        return {
            "累计对数收益": np.nan,
            "年化收益": np.nan,
            "年化波动": np.nan,
            "夏普比率": np.nan,
            "最大回撤": np.nan,
        }

    cum_log_return = clean_ret.sum()
    ann_return = clean_ret.mean() * 252
    ann_vol = clean_ret.std() * np.sqrt(252)
    sharpe = ann_return / ann_vol if ann_vol > 0 else np.nan
    wealth = np.exp(clean_ret.cumsum())
    running_max = wealth.cummax()
    drawdown = wealth / running_max - 1
    max_drawdown = drawdown.min()

    return {
        "累计对数收益": round(cum_log_return, 4),
        "年化收益": round(ann_return, 4),
        "年化波动": round(ann_vol, 4),
        "夏普比率": round(sharpe, 4) if pd.notna(sharpe) else np.nan,
        "最大回撤": round(max_drawdown, 4),
    }
```

`clean_ret` 删除缺失收益。`cum_log_return` 是累计对数收益。`ann_return` 用日均收益乘以 252 近似年化收益。`ann_vol` 用日收益标准差乘以根号 252 近似年化波动。`wealth = np.exp(clean_ret.cumsum())` 把累计对数收益转换为财富指数。`running_max` 是历史最高财富。`drawdown` 是当前财富相对历史最高点的跌幅。`max_drawdown` 是最小回撤值。

```python
strategy_rows = []

    "策略": "买入并持有",
    **evaluate_strategy_returns(results_df["actual_return"]),
```

## 4.5.3 Agentic AI 驱动量化策略优化

### 4.5.3.1 Agent 在量化研究中的角色

量化策略优化不能让智能体自由决定买卖。金融市场具有高噪声、高风险和强监管特征。Agent 更适合承担研究编排、诊断、报告生成、参数检查和风控提醒等任务，而不是直接替代投资经理或交易系统。

在本章场景中，Agent 可以读取模型评估表、年度准确率、回测收益和特征重要性，完成以下工作。第一，检查模型是否存在明显过拟合或概率排序失败。第二，比较方向准确率、AUC、收益、回撤和持仓比例。第三，基于预先定义的规则给出候选模型和阈值建议。第四，生成审计日志，记录使用的数据区间、模型版本、阈值、风险限制和人工复核要求。

本节构建一个规则型量化策略 Agent。它不调用外部大语言模型，因此结果可复现、可审计。若未来接入 LLM，应只让 LLM 生成解释和研究备忘录，不应让 LLM 绕过回测和风控规则直接发出交易指令。

### 4.5.3.2 概率阈值与交易频率

0.5 阈值适合普通二分类，但交易需要覆盖成本。若模型预测上涨概率为 0.501，虽然高于 0.5，但优势极弱，可能不足以支付交易成本。可以引入更高阈值，例如只有概率大于 0.53 或 0.55 时才持有。阈值提高后，交易次数通常减少，信号更保守，但也可能错过上涨日。

```python
def backtest_probability_threshold(
    results: pd.DataFrame,
    model_key: str,
    threshold: float,
) -> dict:
    """在给定概率阈值下回测持有或空仓策略。"""
    prob_col = f"{model_key}_prob"
    signal = (results[prob_col] >= threshold).astype(int)
    strategy_return = results["actual_return"] * signal
    metrics = evaluate_strategy_returns(strategy_return)
    metrics["模型"] = model_key
    metrics["阈值"] = round(threshold, 4)
    metrics["持仓比例"] = round(signal.mean(), 4)
    metrics["交易次数近似"] = int(signal.diff().abs().fillna(0).sum())
    return metrics
```

`prob_col` 找到模型概率列。`signal` 是阈值交易信号。概率大于等于阈值时持有，否则空仓。`strategy_return` 是策略收益。`evaluate_strategy_returns` 计算收益和风险指标。`持仓比例` 衡量策略暴露程度。`signal.diff().abs().sum()` 近似统计持仓状态变化次数，可以作为交易频率代理。

```python
threshold_grid = np.arange(0.45, 0.61, 0.01)
threshold_rows = []

for key in model_names.keys():
    for threshold in threshold_grid:
        threshold_rows.append(
            backtest_probability_threshold(results_df, key, float(threshold))
        )

threshold_eval_df = pd.DataFrame(threshold_rows)
display(
    threshold_eval_df.sort_values(
        ["夏普比率", "累计对数收益"], ascending=False
    ).head(10)
)
```

`threshold_grid` 从 0.45 到 0.60 搜索阈值。低于 0.5 的阈值表示更积极持有，高于 0.5 的阈值表示更保守持有。循环对每个模型和每个阈值回测。结果按夏普比率和累计收益排序。

必须强调，不能直接把样本外测试集上最优阈值用于宣称真实可交易收益。阈值搜索本身也是调参。如果在样本外测试集上挑选最佳阈值，就会产生二次过拟合。严格做法应使用训练集内部验证窗口选择阈值，再在独立测试窗口评估。

### 4.5.3.3 策略诊断规则

```python
def diagnose_model_quality(
    eval_df: pd.DataFrame,
    strategy_eval_df: pd.DataFrame,
    auc_floor: float = 0.50,
    max_drawdown_floor: float = -0.30,
) -> pd.DataFrame:
    """根据分类指标和回测指标生成模型诊断结论。"""
    rows = []
    for _, row in eval_df.iterrows():
        model_name = row["模型"]
        strategy_row = strategy_eval_df[strategy_eval_df["策略"] == f"{model_name} 策略"]
        if strategy_row.empty:
            continue
        strategy_row = strategy_row.iloc[0]

        warnings_list = []
        if row["AUC-ROC"] < auc_floor:
            warnings_list.append("AUC 未超过随机基准")
        if row["方向准确率"] < 0.5:
            warnings_list.append("方向准确率低于 50%")
        if strategy_row["最大回撤"] < max_drawdown_floor:
            warnings_list.append("最大回撤过大")
        if strategy_row["交易日持仓比例"] > 0.9:
            warnings_list.append("策略接近长期满仓，择时贡献有限")
        if strategy_row["交易日持仓比例"] < 0.1:
            warnings_list.append("持仓比例过低，可能错过主要行情")

        if not warnings_list:
            conclusion = "可作为候选模型，但仍需成本检验和独立样本验证"
        else:
            conclusion = "；".join(warnings_list)

        rows.append({
            "模型": model_name,
            "方向准确率": row["方向准确率"],
            "AUC-ROC": row["AUC-ROC"],
            "累计对数收益": strategy_row["累计对数收益"],
            "夏普比率": strategy_row["夏普比率"],
            "最大回撤": strategy_row["最大回撤"],
            "诊断结论": conclusion,
        })
    return pd.DataFrame(rows)


diagnosis_df = diagnose_model_quality(eval_df, strategy_eval_df)
display(diagnosis_df)
```

`diagnose_model_quality` 把分类指标和回测指标合并成诊断表。`auc_floor=0.50` 要求 AUC 至少超过随机基准。`max_drawdown_floor=-0.30` 表示最大回撤若低于 -30% 则风险过大。函数逐个模型读取分类评估和策略评估。若 AUC、准确率、回撤或持仓比例触发规则，就生成风险提示。没有触发风险提示的模型也不能直接上线，只能作为候选模型。

这种规则型诊断的优点是透明、稳定、可审计。局限是规则阈值本身需要业务经验和历史验证。不同机构的风险偏好不同，阈值也应不同。

### 4.5.3.4 规则型量化策略 Agent

```python
class QuantStrategyAgent:
    """基于模型评估与回测结果的规则型量化策略 Agent。"""

    def __init__(
        self,
        eval_table: pd.DataFrame,
        strategy_table: pd.DataFrame,
        threshold_table: pd.DataFrame,
        yearly_table: pd.DataFrame,
        policy_version: str = "quant_strategy_policy_v1",
    ) -> None:
        self.eval_table = eval_table.copy()
        self.strategy_table = strategy_table.copy()
        self.threshold_table = threshold_table.copy()
        self.yearly_table = yearly_table.copy()
        self.policy_version = policy_version
```

`QuantStrategyAgent` 接收四类输入。`eval_table` 是模型分类指标。`strategy_table` 是回测指标。`threshold_table` 是不同概率阈值下的回测结果。`yearly_table` 是年度准确率。`policy_version` 记录策略规则版本。复制输入表可以避免 Agent 内部操作意外修改原始结果。

```python
    def select_candidate_thresholds(self, top_n: int = 5) -> pd.DataFrame:
        candidates = self.threshold_table.copy()
        candidates = candidates[
            (candidates["持仓比例"] >= 0.2) &
            (candidates["持仓比例"] <= 0.8) &
            (candidates["最大回撤"] >= -0.35)
        ]
        if candidates.empty:
            return pd.DataFrame()
        return (
            candidates.sort_values(["夏普比率", "累计对数收益"], ascending=False)
            .head(top_n)
            .reset_index(drop=True)
        )
```

`select_candidate_thresholds` 根据持仓比例和最大回撤筛选候选阈值。持仓比例太高说明策略接近买入并持有。持仓比例太低说明交易机会太少。最大回撤限制用于避免收益较高但风险不可接受的阈值。排序时优先看夏普比率，再看累计对数收益。

```python
    def check_yearly_stability(self, min_yearly_acc: float = 0.45) -> pd.DataFrame:
        rows = []
        for model_name in self.eval_table["模型"]:
            if model_name not in self.yearly_table.columns:
                continue
            acc_series = self.yearly_table[model_name]
            rows.append({
                "模型": model_name,
                "最低年度准确率": round(acc_series.min(), 4),
                "年度准确率标准差": round(acc_series.std(), 4),
                "是否触发稳定性警告": bool(acc_series.min() < min_yearly_acc),
            })
        return pd.DataFrame(rows)
```

`check_yearly_stability` 检查模型跨年度表现。若最低年度准确率低于 0.45，则触发稳定性警告。`年度准确率标准差` 表示模型表现波动程度。稳定性并不保证盈利，但不稳定通常意味着模型对市场状态敏感。

```python
    def build_report(self) -> tuple[pd.DataFrame, pd.DataFrame]:
        threshold_candidates = self.select_candidate_thresholds(top_n=5)
        stability_report = self.check_yearly_stability()

        if threshold_candidates.empty:
            decision = "没有满足基本风险约束的阈值候选，建议停止上线并重新研究特征。"
            selected_model = "无"
            selected_threshold = np.nan
        else:
            best = threshold_candidates.iloc[0]
            selected_model = best["模型"]
            selected_threshold = best["阈值"]
            decision = (
                f"候选模型为 {selected_model}，候选阈值为 {selected_threshold}。"
                "该结论只用于研究排序，必须在独立验证集和含成本回测中复核。"
            )

        audit_log = pd.DataFrame([{
            "策略版本": self.policy_version,
            "候选模型": selected_model,
            "候选阈值": selected_threshold,
            "决策说明": decision,
            "是否允许自动交易": False,
        }])

        return threshold_candidates, stability_report, audit_log
```

`build_report` 生成候选阈值、稳定性报告和审计日志。若没有阈值满足基本风险约束，则建议停止上线。若有候选阈值，则选择排序第一项作为研究候选。`是否允许自动交易` 固定为 False，表示本章 Agent 只做研究辅助，不直接执行交易。

### 4.5.3.5 为什么不用更复杂的 Agent 直接优化策略

从技术上看，可以让 LLM 或自动化 Agent 搜索特征、模型、阈值和交易规则。但在量化投资中，这种做法容易导致过拟合。搜索空间越大，越容易找到历史上表现优秀但未来无效的策略。若 Agent 不受约束地反复试验参数，并根据同一测试集选择最优结果，本质上就是数据挖掘偏差。

本章选择规则型 Agent，而不是让外部智能体直接生成交易策略，原因有三点。第一，本章重点是规范研究流程，而不是展示不可审计的自动调参。第二，规则型 Agent 的输入、输出和阈值都可检查，适合金融模型治理。第三，策略上线需要人工审批、风险预算、交易成本验证和合规审查，不能由语言模型直接决定。

未来可以把 Agent 扩展为研究助手。它可以自动生成回测报告，检查是否存在前视偏差，提醒缺失交易成本，比较不同市场状态下的表现，生成特征稳定性图表，记录实验配置，并把候选策略提交给人工投研人员复核。

## 本章模型选择的综合讨论

预测价格水平通常比预测收益率方向更容易得到漂亮图形，但更容易产生误导。价格序列强自相关导致滞后复制模型也能取得很低误差。收益率方向预测更难，但更贴近交易决策。本章选择方向预测，是为了让模型评估具有经济含义。

技术指标适合作为入门因子，因为它们容易从 OHLCV 数据构造，能够覆盖趋势、动量、波动和成交量信息。它们的局限是信息源单一、参数任意、容易拥挤，并且在有效市场中预测能力通常较弱。若要提升策略质量，可以引入基本面因子、宏观因子、行业轮动因子、期权隐含波动率、新闻情绪和资金流数据。

Random Forest 是稳健基线。它适合快速验证特征是否具有非线性预测能力。XGBoost 是结构化数据中的强模型，适合在表格型因子上进行预测。LSTM 理论上更适合序列，但在日频低信噪比市场中不一定优于树模型。模型复杂度不是优势本身，只有稳定样本外收益和可控风险才是优势。

Walk-Forward Validation 是时间序列建模的基本要求。任何随机划分、全样本标准化、使用未来收益构造特征、用测试集调阈值和只报告最佳年份表现的做法，都会高估策略能力。真实量化研究应保留最终独立测试集，并尽量使用从未参与特征选择和参数选择的数据进行最终验证。

简化回测只能用于教学。真实策略必须加入交易成本、滑点、成交约束、仓位限制、停牌或熔断规则、风险预算和容量估计。方向准确率 53% 不一定盈利，因为错误交易的平均损失可能大于正确交易的平均收益，也可能因为频繁换手被成本吞噬。

Agentic AI 在量化投资中的价值是流程编排和治理增强，而不是魔法式预测。一个可信的 Agent 应该帮助研究者发现错误、记录实验、生成报告和执行风险检查。它不应绕过样本外验证，也不应把历史最优参数包装成确定盈利策略。

## 拓展方向

1. 将预测目标从未来 1 日方向扩展到未来 5 日或 20 日方向，比较噪声水平、样本数量和交易频率变化。
2. 使用固定长度滚动窗口替代扩展窗口，例如每轮只使用最近 2 年数据，检验模型是否更适应近期市场。
3. 引入交易成本和滑点，比较不同阈值下策略是否仍有正收益。
4. 用 Logistic 回归作为线性基线，并比较正则化系数、AUC 和可解释性。
5. 用 SHAP 分析 XGBoost 的特征贡献，观察特征作用是否随市场状态变化。
6. 记录每个滚动窗口的特征重要性，评估技术因子的稳定性。
7. 将模型输出从二分类方向扩展为三分类，即上涨、震荡和下跌，从而减少低置信度交易。
8. 使用模型集成方法融合 Random Forest、XGBoost 和 LSTM，例如概率平均、加权平均、堆叠模型和投票模型。
9. 接入新闻情绪、宏观变量、利率、波动率指数和行业因子，构建多源信息预测模型。
10. 将策略从单资产择时扩展为多资产横截面选股，预测股票相对收益而非单指数方向。
11. 在 Agent 中加入实验追踪工具，自动记录数据版本、特征版本、模型参数、回测结果和审批状态。
12. 设计严格的样本外检验流程，把研究集、验证集和最终测试集完全分离，避免因多次试验导致的数据挖掘偏差。
