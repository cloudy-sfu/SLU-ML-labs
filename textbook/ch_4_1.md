# 4.1 信用评分与消费贷风控案例

信用评分是金融风控中最基础也最重要的建模任务之一。银行、消费金融公司、信用卡中心和互联网信贷平台，都需要在放款之前评估申请人的违约风险，并在放款之后持续监控客户风险变化。一个合格的信用评分模型，不只是给出一个分类结果，还要支持审批策略、额度策略、定价策略、人工复核和贷后预警。

本章围绕三个公开数据案例展开。German Credit 数据用于说明个人信用评分的基本建模流程，重点展示目标变量定义、类别编码、决策树、随机森林和 XGBoost 的建模思想。Default of Credit Card Clients 数据用于说明信用卡客户违约预测，重点展示标准化、核函数和 SVM 的影响。Lending Club 消费贷数据用于构建更接近真实业务的消费贷违约预警流程，重点展示时间切分、训练集内预处理、LightGBM、MLP、阈值选择、PSI 稳定性分析和客户风险分级。

信用评分模型通常解决的是二分类问题。给定客户特征向量 x，模型估计客户未来违约概率 P(y=1|x)。其中，1 表示违约、坏账、核销或 charge-off，0 表示正常还款。这个定义看似简单，但它决定了后续 Precision、Recall、F1、AUC 和 KS 的业务解释。如果正类含义定义错误，模型评估结论会发生根本性偏差。

信用风险建模应遵循三个基本原则。第一，标签必须来自明确的业务结局，不能把尚未成熟的贷款状态直接当作好坏标签。第二，训练集、验证集和测试集之间不能发生数据泄漏，所有填充、编码、标准化和阈值选择都应只使用允许使用的历史信息。第三，模型选择不能只看单次 AUC，还要同时考虑可解释性、稳定性、计算成本、监管要求和业务落地方式。

## 4.1.1 数据集准备与清洗

数据准备阶段要回答三个问题。第一，样本是否能够代表要预测的业务场景。第二，目标变量是否准确表达风险事件。第三，数据切分和预处理是否符合真实预测时间顺序。信用评分中的很多建模错误并不来自算法本身，而来自标签定义、样本选择和预处理流程。

### 4.1.1.2 German Credit 数据的加载和目标变量修正

German Credit 数据集包含 1000 条贷款申请样本。每条记录描述申请人的支票账户状态、贷款期限、贷款金额、储蓄状态、就业年限、年龄、住房状况和历史信用情况等信息。原始目标变量为 good 或 bad。风控建模更关心坏客户识别，因此本章将 bad 设置为正类 1，将 good 设置为负类 0。

```python
from sklearn.datasets import fetch_openml
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import LabelEncoder

# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.datasets.fetch_openml.html
credit_data = fetch_openml(data_id=31, as_frame=True, parser="auto")
df_german = credit_data.frame.copy()
target_col = credit_data.target_names[0] if credit_data.target_names else "class"

print(f"数据集维度：{df_german.shape}")
print(f"目标变量：{target_col}")
print(df_german[target_col].value_counts())
```

`fetch_openml` 从 OpenML 获取数据。`data_id=31` 指向 German Credit 数据。`as_frame=True` 表示返回 pandas DataFrame，便于后续清洗。`parser="auto"` 让 scikit-learn 自动选择解析器。`df_german.copy()` 避免后续处理修改原始返回对象。`target_col` 用于兼容不同版本接口，如果 OpenML 返回了目标列名称，就使用返回值，否则默认目标列为 `class`。最后三行用于确认数据维度、目标列名称和类别分布。

```python
X_german = df_german.drop(columns=[target_col]).copy()
y_german_raw = df_german[target_col].astype(str)

label_map = {"good": 0, "bad": 1}
y_german = y_german_raw.map(label_map).astype(int)

print("目标变量业务映射：good=0，bad=1")
print(y_german.value_counts().rename(index={0: "good", 1: "bad"}))
```

这里没有直接使用 `LabelEncoder` 编码目标变量，而是显式定义 `label_map`。这样做可以保证正类一定是坏信用客户。`astype(str)` 防止原始数据为 category 类型时映射异常。`map` 将文本标签转换为数值标签。`astype(int)` 确保后续模型接收整数型目标变量。若把 good 误设为正类，Precision 和 Recall 将变成对好客户识别能力的描述，不能再解释为坏客户识别能力。

```python
X_german_encoded = X_german.copy()
label_encoders_german = {}

for col in X_german_encoded.select_dtypes(include=["category", "object"]).columns:
    le = LabelEncoder()
    X_german_encoded[col] = le.fit_transform(X_german_encoded[col].astype(str))
    label_encoders_german[col] = le

print(f"编码后特征矩阵维度：{X_german_encoded.shape}")
display(X_german_encoded.head())
```

`X_german.copy()` 避免修改原始特征矩阵。`label_encoders_german` 保存每个分类变量对应的编码器，便于将来追溯整数编码含义。循环只处理 category 和 object 类型字段。`fit_transform` 同时学习类别到整数的映射并完成转换。这里使用标签编码，是因为后续树模型可以根据阈值分裂数值变量。学生需要注意，标签编码会给无序类别引入人为顺序。对于 Logistic 回归、SVM 和神经网络，One-Hot 编码通常更合理；对于 LightGBM，也可以使用原生类别特征处理。


`train_test_split` 将样本分成训练集和测试集。`test_size=0.2` 表示 20% 样本用于测试。`random_state=42` 保证重复运行时划分一致。`stratify=y_german` 使训练集和测试集保持相近违约比例。`Xg_train_raw` 和 `Xg_test_raw` 保留未标签编码的原始字段，后面用于 Logistic 回归的 One-Hot 编码基线。

### 4.1.1.3 信用卡违约数据的加载和清洗

信用卡违约数据包含客户额度、人口统计信息、近六个月还款状态、账单金额和还款金额。目标变量为 `default payment next month`，其中 1 表示下个月违约，0 表示未违约。本节默认抽样 1000 条样本，便于在课堂环境中快速比较 SVM 核函数和标准化效果。完整研究可以使用全部 30000 条样本。

```python
DEFAULT_CREDIT_COLUMNS = [
    "LIMIT_BAL", "SEX", "EDUCATION", "MARRIAGE", "AGE",
    "PAY_0", "PAY_2", "PAY_3", "PAY_4", "PAY_5", "PAY_6",
    "BILL_AMT1", "BILL_AMT2", "BILL_AMT3", "BILL_AMT4", "BILL_AMT5", "BILL_AMT6",
    "PAY_AMT1", "PAY_AMT2", "PAY_AMT3", "PAY_AMT4", "PAY_AMT5", "PAY_AMT6",
]

DEFAULT_SAMPLE_SIZE = 1000
```

`DEFAULT_CREDIT_COLUMNS` 保存信用卡数据的标准字段名。OpenML 版本有时会把字段命名成 x1、x2 等通用名称，因此加载后需要恢复业务字段名。`DEFAULT_SAMPLE_SIZE` 控制抽样规模。若设为 `None`，函数将返回完整数据。

```python
def load_credit_card_default_data(
    sample_size: int | None = DEFAULT_SAMPLE_SIZE,
    random_state: int = 42,
) -> tuple[pd.DataFrame, pd.Series]:
    """从 OpenML 读取信用卡违约预测数据集。"""
    # Ref: https://scikit-learn.org/stable/modules/generated/sklearn.datasets.fetch_openml.html
    dataset = fetch_openml(data_id=42477, as_frame=True, parser="auto")
    X = dataset.data.copy()
    y = pd.Series(dataset.target)

    X.columns = [str(c).strip() for c in X.columns]
    if len(X.columns) == 23 and all(re.fullmatch(r"x\d+", str(col).lower()) for col in X.columns):
        X.columns = DEFAULT_CREDIT_COLUMNS

    y = pd.to_numeric(pd.Series(y), errors="coerce").astype(int)
    y.name = "default_payment_next_month"

    if sample_size is not None:
        if sample_size <= 0:
            raise ValueError("sample_size 必须为正整数或 None")
        if sample_size < len(X):
            X, _, y, _ = train_test_split(
                X, y,
                train_size=sample_size,
                stratify=y,
                random_state=random_state,
            )
            X = X.reset_index(drop=True)
            y = y.reset_index(drop=True)

    return X, y
```

函数参数 `sample_size` 控制样本量，`random_state` 控制抽样复现性。`fetch_openml(data_id=42477)` 是数据读取路径。`X.columns` 去掉列名两侧空格，避免列名匹配失败。`re.fullmatch` 判断字段是否为 x1、x2 这类匿名格式，如果是，就替换为标准字段名。`pd.to_numeric` 将目标变量转为数值，`errors="coerce"` 会把不能解析的值变成缺失值。抽样阶段使用分层抽样，目的是保持违约比例稳定。

```python
X_card, y_card = load_credit_card_default_data()
print("特征矩阵维度：", X_card.shape)
print("目标变量名称：", y_card.name)
print(y_card.value_counts().rename(index={0: "未违约", 1: "违约"}))
```

这段代码执行数据加载，并检查样本规模和类别分布。信用卡违约数据通常存在明显类别不平衡。如果只看准确率，模型可能把大多数客户都预测为未违约，仍然获得看似不错的结果。

```python
X_card = X_card.copy()

if "EDUCATION" in X_card.columns:
    X_card["EDUCATION"] = X_card["EDUCATION"].replace({0: 4, 5: 4, 6: 4})

if "MARRIAGE" in X_card.columns:
    X_card["MARRIAGE"] = X_card["MARRIAGE"].replace({0: 3})

for col in X_card.columns:
    X_card[col] = pd.to_numeric(X_card[col], errors="coerce")

valid_mask = X_card.notna().all(axis=1) & y_card.notna()
X_card = X_card.loc[valid_mask].reset_index(drop=True)
y_card = y_card.loc[valid_mask].reset_index(drop=True)
```

`EDUCATION` 中的 0、5、6 不属于主要教育类别，通常合并为 4，即其他类别。`MARRIAGE` 中的 0 合并为 3，即其他类别。循环把所有字段转成数值。`valid_mask` 只保留特征和目标变量均不缺失的样本。教学案例中直接删除缺失样本比较清晰；生产场景应记录缺失原因，并比较删除、均值填充、中位数填充、缺失指示变量和模型填充等方案。

```python
Xc_train, Xc_test, yc_train, yc_test = train_test_split(
    X_card,
    y_card,
    test_size=0.2,
    random_state=42,
    stratify=y_card,

print("训练集大小：", Xc_train.shape)
print("测试集大小：", Xc_test.shape)
print(f"训练集违约率：{yc_train.mean():.4f}")
print(f"测试集违约率：{yc_test.mean():.4f}")
```

信用卡案例采用随机分层切分，主要用于比较 SVM 核函数和标准化效果。这个切分适合教学演示。若模型用于真实信用卡客户风险预测，更建议加入时间切分，因为客户行为会随经济周期、营销策略、额度政策和还款提醒策略变化。

### 4.1.1.4 Lending Club 消费贷款数据的加载和目标变量定义

Lending Club 数据更接近真实消费贷业务。原始数据包含贷款金额、期限、利率、每月还款额、信用等级、收入、贷款用途、债务收入比、循环信用余额、账户数量和公开不良记录等字段。目标变量来自 `loan_status`。本章只保留 `Fully Paid` 和 `Charged Off` 两类成熟状态，把 `Charged Off` 设为违约正类。

```python

test_lending_raw_df = df_lending_model_raw.iloc[val_end:].copy()
```

这段代码先按发放月份排序，再划分训练集、验证集和测试集。训练集用于拟合模型和预处理规则。验证集用于早停、调参和阈值探索。测试集只用于最终评估。真实消费贷系统永远是用过去数据训练模型，再预测未来客户。若采用随机切分，训练集中可能混入未来时期的数据分布，从而使测试结果偏乐观。若数据缺少 `issue_d`，代码只能退化为按当前行顺序切分，此时评估结论应被视为教学演示，而不是严格的上线回测。

```python
def parse_percent_series(s: pd.Series) -> pd.Series:
    """把百分数字符串转换为浮点数。"""
    return pd.to_numeric(
        s.astype(str).str.replace("%", "", regex=False).str.strip(),
        errors="coerce",
    )


def parse_term_series(s: pd.Series) -> pd.Series:
    """把 36 months 或 60 months 转换为月份数。"""
    return pd.to_numeric(s.astype(str).str.extract(r"(\d+)")[0], errors="coerce")


def parse_emp_length_series(s: pd.Series) -> pd.Series:
    """把工作年限字段转换为数值，< 1 year 记为 0，10+ years 记为 10。"""
    text = s.astype(str).str.strip().str.lower()
    result = pd.Series(np.nan, index=s.index, dtype="float")
    result[text.eq("< 1 year")] = 0.0
    result[text.str.contains("10\+", na=False)] = 10.0
    extracted = pd.to_numeric(text.str.extract(r"(\d+)")[0], errors="coerce")
    result = result.fillna(extracted)
    result[text.isin(["nan", "none", "n/a", ""])] = np.nan
    return result


def clean_lending_raw_features(data: pd.DataFrame) -> pd.DataFrame:
    """执行不依赖样本分布的字段清洗。"""
    cleaned = data[FEATURES].copy()

    if "term" in cleaned.columns:
        cleaned["term"] = parse_term_series(cleaned["term"])

    if "emp_length" in cleaned.columns:
        cleaned["emp_length"] = parse_emp_length_series(cleaned["emp_length"])

    for col in ["int_rate", "revol_util"]:
        if col in cleaned.columns:
            cleaned[col] = parse_percent_series(cleaned[col])

    for col in cleaned.columns:
        if cleaned[col].dtype == "object" or str(cleaned[col].dtype) == "category":
            continue
        cleaned[col] = pd.to_numeric(cleaned[col], errors="coerce")

    return cleaned
```

`parse_percent_series` 防止利率和循环信用利用率被误当作字符串类别。`parse_term_series` 把贷款期限统一为月份数。`parse_emp_length_series` 修正工作年限字段中的特殊值，`< 1 year` 表示不足一年，记为 0，`10+ years` 表示十年及以上，记为 10。`clean_lending_raw_features` 只执行不依赖样本分布的字段清洗，因此不会把验证集或测试集统计信息泄漏到训练阶段。

```python
def fit_lending_preprocessor(train_data: pd.DataFrame) -> dict:
    """只在训练集上学习类别映射和数值填充值。"""
    cleaned_train = clean_lending_raw_features(train_data)
    categorical_columns = cleaned_train.select_dtypes(include=["object", "category"]).columns.tolist()
    numeric_columns = [c for c in FEATURES if c not in categorical_columns]

    medians = cleaned_train[numeric_columns].median()
    category_maps = {}
    for col in categorical_columns:
        values = cleaned_train[col].where(cleaned_train[col].notna(), "__missing__").astype(str)
        categories = sorted(values.unique().tolist())
        category_maps[col] = {cat: idx for idx, cat in enumerate(categories)}

    return {
        "numeric_columns": numeric_columns,
        "categorical_columns": categorical_columns,
        "medians": medians,
        "category_maps": category_maps,
    }


def transform_lending_features(data: pd.DataFrame, preprocessor: dict) -> pd.DataFrame:
    """使用训练集学到的规则转换任意样本。"""
    cleaned = clean_lending_raw_features(data)
    transformed = pd.DataFrame(index=cleaned.index)

    for col in preprocessor["numeric_columns"]:
        transformed[col] = pd.to_numeric(cleaned[col], errors="coerce")
        transformed[col] = transformed[col].fillna(preprocessor["medians"][col])

    for col in preprocessor["categorical_columns"]:
        values = cleaned[col].where(cleaned[col].notna(), "__missing__").astype(str)
        mapping = preprocessor["category_maps"][col]
        transformed[col] = values.map(mapping).fillna(-1).astype(int)

    return transformed[FEATURES]


lending_preprocessor = fit_lending_preprocessor(train_lending_raw_df)
label_encoders_lending = lending_preprocessor["category_maps"]
categorical_features_lending = lending_preprocessor["categorical_columns"]

print(f"验证集大小：{Xl_val.shape[0]}，违约率：{yl_val.mean():.4f}")
print(f"测试集大小：{Xl_test.shape[0]}，违约率：{yl_test.mean():.4f}")

    print(f"验证集时间范围：{val_lending_raw_df['issue_d'].min()} ~ {val_lending_raw_df['issue_d'].max()}")
```

`fit_lending_preprocessor` 只读取训练集。数值变量的中位数只来自训练期，分类变量的类别映射也只来自训练期。`transform_lending_features` 把验证集和测试集中的未知类别映射为 -1，而不是重新拟合编码器。这样做符合真实预测环境，因为未来样本可能出现训练期未见过的类别。最终得到的 `Xl_train`、`Xl_val` 和 `Xl_test` 具有相同列顺序，可以直接进入后续模型。

时间切分后的训练集、验证集和测试集违约率可能并不完全相同。这不是错误，而是风控建模中常见的数据漂移信号。宏观经济、获客渠道、信用政策、利率环境和催收策略都会改变客户风险结构，因此模型上线后必须持续监控。

## 4.1.2 特征工程与模型训练

特征工程和模型训练阶段要回答两个问题。第一，哪些变量能够表达还款能力、还款意愿、债务压力和信用历史。第二，为什么选择某种模型，而不是另一种模型。

传统信用评分常使用 Logistic 回归评分卡，因为它可解释、稳定、便于转化为分数，也便于满足监管沟通需求。树模型和梯度提升模型通常在结构化表格数据上具有更强非线性表达能力。SVM 能展示最大间隔和核函数思想。MLP 能展示神经网络训练过程。实际业务中，模型选择不应只看单次 AUC，而要同时考虑解释性、稳定性、部署成本、监管要求和维护成本。

### 4.1.2.1 Logistic 回归评分卡基线

Logistic 回归是信用评分卡的经典模型。它把线性函数映射为 0 到 1 之间的违约概率。

$$P(y=1 \mid x)=\frac{1}{1+e^{-(\beta_0+\beta_1x_1+\cdots+\beta_px_p)}}$$

如果某个系数 `β_i` 大于 0，在其他变量不变时，该变量增加会提高违约对数几率。如果 `β_i` 小于 0，该变量增加会降低违约对数几率。Logistic 回归的优势在于解释清晰、稳定性较好、部署简单。它的局限在于对非线性关系和特征交互的表达能力有限，因此常作为强可解释基线，而不是所有场景中的最高性能模型。

```python
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import OneHotEncoder, StandardScaler

numeric_cols_german = Xg_train_raw.select_dtypes(include=[np.number]).columns.tolist()
categorical_cols_german = [c for c in Xg_train_raw.columns if c not in numeric_cols_german]

# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.compose.ColumnTransformer.html
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.linear_model.LogisticRegression.html
german_logit_preprocess = ColumnTransformer(
    transformers=[
        ("num", Pipeline([
            ("imputer", SimpleImputer(strategy="median")),
            ("scaler", StandardScaler()),
        ]), numeric_cols_german),
        ("cat", Pipeline([
            ("imputer", SimpleImputer(strategy="most_frequent")),
            ("onehot", OneHotEncoder(handle_unknown="ignore")),
        ]), categorical_cols_german),
    ]
)

german_logit_clf = Pipeline([
    ("preprocess", german_logit_preprocess),
    ("model", LogisticRegression(
        max_iter=1000,
        class_weight="balanced",
        C=1.0,
        solver="lbfgs",
        random_state=42,
    )),
])
```

`ColumnTransformer` 允许数值变量和类别变量使用不同预处理流程。数值变量先用中位数填充，再标准化。类别变量先用众数填充，再使用 One-Hot 编码。`OneHotEncoder(handle_unknown="ignore")` 能处理测试集中出现的新类别，避免预测时报错。`class_weight="balanced"` 根据类别频率调整损失权重，使坏客户样本在训练时获得更高关注。`C=1.0` 是正则化强度的倒数，C 越小，正则化越强。

```python
german_logit_clf.fit(Xg_train_raw, yg_train)

yg_pred_logit = german_logit_clf.predict(Xg_test_raw)
yg_prob_logit = german_logit_clf.predict_proba(Xg_test_raw)[:, 1]

print("=== Logistic 回归基线模型评估 ===")
print(classification_report(yg_test, yg_pred_logit, target_names=["good", "bad"]))
print(f"AUC-ROC: {roc_auc_score(yg_test, yg_prob_logit):.4f}")
```

`fit` 在训练集上学习填充值、标准化参数、One-Hot 类别和 Logistic 回归系数。`predict` 输出离散类别。`predict_proba` 输出违约概率，`[:, 1]` 取正类 bad 的概率。该模型的意义不是替代后面的树模型，而是提供一个解释性基线。若复杂模型的提升很小，业务上可能更倾向使用 Logistic 回归。

### 4.1.2.2 决策树、随机森林与 XGBoost 的建模思想

决策树通过递归分裂样本空间完成分类。每个内部节点是一条判断规则，每个叶节点给出类别预测。常用分裂准则是基尼不纯度和信息熵。二分类节点的基尼不纯度为

$$Gini = 1 - p_0^2 - p_1^2$$

其中，`p_0` 和 `p_1` 分别表示节点中负类和正类的比例。基尼不纯度越低，节点越纯。

随机森林通过 Bootstrap 抽样训练多棵树，并在每次分裂时随机选择特征子集。它牺牲单棵树的完全可解释性，换取更低方差和更稳定的预测。

XGBoost 是梯度提升树模型。它按顺序训练多棵树，后续树用于修正前面模型的误差。随机森林是并行投票，主要降低方差；XGBoost 是串行加法模型，主要降低偏差并优化损失函数。


### 4.1.2.3 SVM、核函数与标准化

SVM 的思想是在特征空间中寻找最大间隔分类边界。线性 SVM 适合近似线性可分的数据。RBF 核 SVM 可以通过核函数表达非线性边界。

$$K(x_i,x_j)=\exp(-\gamma \Vert x_i-x_j\Vert^2)$$

RBF 核依赖样本距离，因此对特征尺度非常敏感。信用卡数据中额度、账单金额、还款金额和年龄的量纲差异很大。如果不标准化，大尺度变量会主导距离计算，使核函数无法合理表达客户之间的相似性。

```python
from sklearn.svm import LinearSVC, SVC


def build_svm_pipeline(
    kernel: str,
    use_scaler: bool,
    C: float = 1.0,
    gamma: str = "scale",
) -> Pipeline:
    steps = []
    if use_scaler:
        steps.append(("scaler", StandardScaler()))
    if kernel == "linear":
        steps.append(("svm", LinearSVC(
            C=C,
            class_weight="balanced",
            dual="auto",
            max_iter=5000,
        )))
    else:
        steps.append(("svm", SVC(
            kernel=kernel,
            C=C,
            gamma=gamma,
            class_weight="balanced",
        )))
    return Pipeline(steps)
```

`Pipeline` 把预处理和模型封装成统一流程，避免在测试集上提前拟合标准化器。`StandardScaler` 用训练集均值和标准差把特征转成近似零均值、单位方差。`kernel` 控制使用线性核还是 RBF 核。`C` 控制间隔违反惩罚，C 越大，模型越努力拟合训练数据，过拟合风险也越高。`gamma` 控制 RBF 核的局部影响范围。`class_weight="balanced"` 继续处理违约样本较少的问题。

```python
def evaluate_svm_model(model: Pipeline, X_eval: pd.DataFrame, y_eval: pd.Series) -> dict:
    y_pred = model.predict(X_eval)
    scores = model.decision_function(X_eval)
    return {
        "accuracy": accuracy_score(y_eval, y_pred),
        "precision": precision_score(y_eval, y_pred, zero_division=0),
        "recall": recall_score(y_eval, y_pred, zero_division=0),
        "f1": f1_score(y_eval, y_pred, zero_division=0),
        "roc_auc": roc_auc_score(y_eval, scores),
        "confusion_matrix": confusion_matrix(y_eval, y_pred),
        "y_pred": y_pred,
        "scores": scores,
    }
```

`evaluate_svm_model` 统一计算多个指标。SVM 的 `decision_function` 输出样本到分类边界的相对距离，而不是概率。ROC-AUC 只需要排序分数，因此可以直接使用 `scores`。`zero_division=0` 防止模型没有预测任何正类时指标报错。返回字典中保留混淆矩阵、预测类别和分数，便于后续画图和解释。

```python
experiments = {
    "Linear-SVM（未缩放）": build_svm_pipeline(kernel="linear", use_scaler=False),
    "Linear-SVM（标准化）": build_svm_pipeline(kernel="linear", use_scaler=True),
    "RBF-SVM（未缩放）": build_svm_pipeline(kernel="rbf", use_scaler=False),
    "RBF-SVM（标准化）": build_svm_pipeline(kernel="rbf", use_scaler=True),
}

fitted_svm_models: dict[str, Pipeline] = {}
svm_results: list[dict] = []
fit_time_rows: list[dict[str, float | str]] = []

for name, model in experiments.items():
    print(f"正在训练：{name}")
    fit_start = perf_counter()
    model.fit(Xc_train, yc_train)
    fit_seconds = perf_counter() - fit_start
    fitted_svm_models[name] = model
    metrics = evaluate_svm_model(model, Xc_test, yc_test)
    metrics["model"] = name
    svm_results.append(metrics)
    fit_time_rows.append({"model": name, "fit_seconds": fit_seconds})
```

`experiments` 定义四组对比实验，只改变核函数和是否标准化。循环中每个模型使用同一训练集和测试集。`perf_counter` 记录训练时间，使学生认识模型效果与计算成本之间的权衡。线性 SVM 训练较快，RBF SVM 表达能力更强，但在大样本上计算代价更高。


### 4.1.2.4 LightGBM 消费贷违约模型

LightGBM 是梯度提升树的高效实现。它通过直方图分裂降低计算复杂度，通过叶子优先生长提高拟合效率，并支持多种正则化和采样参数。相比随机森林，LightGBM 是提升式模型，能逐轮修正误差。相比普通单棵树，它能表达更复杂的非线性和交互关系。相比神经网络，它通常更适合中等规模表格数据，并且训练成本较低。

### 4.1.2.5 MLP 神经网络消费贷模型

MLP 是多层感知机，由全连接层、激活函数和输出层组成。它适合学习复杂非线性关系，但在表格数据上未必优于梯度提升树。选择 MLP 的教学价值在于让学生理解神经网络训练过程、标准化必要性、损失函数、类别不平衡处理和过拟合控制。


神经网络对特征尺度敏感，因此先用 `StandardScaler` 标准化。`fit_transform` 只在训练集上学习均值和标准差。`transform` 用训练集统计量转换验证集和测试集，避免数据泄漏。`torch.manual_seed` 提高实验复现性。`torch.device` 自动选择 GPU 或 CPU。`FloatTensor` 将数据转成 PyTorch 张量。`TensorDataset` 把特征和标签绑定。`DataLoader` 按批次读取样本，`shuffle=True` 在每个 epoch 打乱训练样本顺序。

```python
class MLPClassifier(nn.Module):
    """用于二分类的多层感知机，输出未经过 Sigmoid 的 logit。"""

    def __init__(
        self,
        input_dim: int,
        hidden_dims: list[int] | None = None,
        dropout: float = 0.3,
    ):
        super().__init__()
        if hidden_dims is None:
            hidden_dims = [128, 64, 32]

        layers = []
        prev_dim = input_dim
        for h_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.BatchNorm1d(h_dim),
                nn.ReLU(),
                nn.Dropout(dropout),
            ])
            prev_dim = h_dim
        layers.append(nn.Linear(prev_dim, 1))
        self.network = nn.Sequential(*layers)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.network(x).squeeze(-1)


input_dim = Xl_train_scaled.shape[1]
mlp_model = MLPClassifier(input_dim=input_dim, hidden_dims=[128, 64, 32], dropout=0.3).to(device)
print(mlp_model)
```

`MLPClassifier` 继承 `nn.Module`。`input_dim` 等于输入特征数量。`hidden_dims` 控制隐藏层宽度。`dropout=0.3` 表示训练时随机关闭 30% 神经元。`nn.Linear` 是全连接层。`nn.BatchNorm1d` 稳定每一层输入分布。`nn.ReLU` 引入非线性。`nn.Dropout` 抑制过拟合。最后一层输出 logit，而不是直接输出概率。这样可以配合 `BCEWithLogitsLoss` 提高数值稳定性，并自然加入正类权重。

```python
# Ref: https://pytorch.org/docs/stable/generated/torch.nn.BCEWithLogitsLoss.html
pos_weight_value = (yl_train == 0).sum() / (yl_train == 1).sum()
pos_weight = torch.tensor([pos_weight_value], dtype=torch.float32, device=device)

criterion = nn.BCEWithLogitsLoss(pos_weight=pos_weight)
optimizer = torch.optim.Adam(mlp_model.parameters(), lr=1e-3, weight_decay=1e-4)
scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
    optimizer,
    mode="min",
    patience=5,
    factor=0.5,
)

NUM_EPOCHS = 50
EARLY_STOPPING_PATIENCE = 8
train_losses = []
val_losses = []
val_aucs = []
best_val_auc = -np.inf
best_state = None
epochs_without_improvement = 0

for epoch in range(NUM_EPOCHS):
    mlp_model.train()
    epoch_loss = 0.0
    for X_batch, y_batch in train_loader:
        optimizer.zero_grad()
        logits_batch = mlp_model(X_batch)
        loss = criterion(logits_batch, y_batch)
        loss.backward()
        optimizer.step()
        epoch_loss += loss.item() * len(y_batch)

    epoch_loss /= len(train_dataset)
    train_losses.append(epoch_loss)

    mlp_model.eval()
    with torch.no_grad():
        val_logits = mlp_model(Xl_val_tensor)
        val_loss = criterion(val_logits, yl_val_tensor).item()
        y_prob_val = torch.sigmoid(val_logits).cpu().numpy()
        val_auc = roc_auc_score(yl_val, y_prob_val)

    val_losses.append(val_loss)
    val_aucs.append(val_auc)
    scheduler.step(val_loss)

    if val_auc > best_val_auc:
        best_val_auc = val_auc
        best_state = copy.deepcopy(mlp_model.state_dict())
        epochs_without_improvement = 0
    else:
        epochs_without_improvement += 1

    if (epoch + 1) % 10 == 0:
        print(f"Epoch {epoch+1:3d}/{NUM_EPOCHS} | Loss: {epoch_loss:.4f} | Val AUC: {val_auc:.4f}")

    if epochs_without_improvement >= EARLY_STOPPING_PATIENCE:
        print(f"验证集 AUC 连续 {EARLY_STOPPING_PATIENCE} 轮未提升，提前停止。")
        break

if best_state is not None:
    mlp_model.load_state_dict(best_state)
```

`BCEWithLogitsLoss` 把 Sigmoid 和二分类交叉熵合并在一起，数值上比先 Sigmoid 再 `BCELoss` 更稳定。`pos_weight` 提高违约样本在损失函数中的权重。`Adam` 是自适应优化器。`weight_decay=1e-4` 是 L2 正则。`ReduceLROnPlateau` 在验证损失停滞时降低学习率。每个 epoch 中，`optimizer.zero_grad()` 清空上一批次梯度，`loss.backward()` 反向传播，`optimizer.step()` 更新参数。验证阶段使用 `eval()` 和 `torch.no_grad()`，不更新模型参数。早停逻辑保存验证集 AUC 最好的参数，避免训练过久导致过拟合。

## 4.1.3 风控模型评估与解释

模型评估不能只看准确率。违约样本通常是少数类，如果模型把大多数客户都预测为正常，准确率可能仍然较高，但对风险管理几乎没有价值。风控模型评估至少应包括分类指标、排序指标、阈值分析、稳定性分析和业务解释。

令 TP 表示真实违约且预测违约，TN 表示真实未违约且预测未违约，FP 表示真实未违约但预测违约，FN 表示真实违约但预测未违约。常用指标定义为

$$Accuracy = \frac{TP + TN}{TP + TN + FP + FN}$$

$$Precision = \frac{TP}{TP + FP}$$

$$Recall = \frac{TP}{TP + FN}$$

$$F1 = \frac{2 \times Precision \times Recall}{Precision + Recall}$$

在信贷审批中，FN 表示坏客户被放行，通常带来坏账损失。FP 表示好客户被拒绝，通常带来收入损失和客户体验损失。两类错误的成本不同，因此模型阈值必须结合业务成本。

### 4.1.3.1 ROC 曲线、AUC 与模型排序能力

ROC 曲线展示不同阈值下真正率和假正率之间的关系。AUC 是 ROC 曲线下的面积，可以解释为模型把随机一个坏客户排在随机一个好客户前面的概率。AUC 不依赖某个固定阈值，适合比较模型潜在区分能力。

### 4.1.3.3 KS 统计量与阈值选择

KS 统计量是风控领域常用的区分度指标。它衡量正负样本累计分布差异的最大值。

$$KS = \max_t \left( TPR(t) - FPR(t) \right)$$

其中，`t` 是分类阈值。KS 越大，说明模型越能把好客户和坏客户拉开。KS 对模型排序能力和业务分层都很有参考意义，但它仍然不是最终审批阈值。最终阈值还需要考虑风险偏好、资本成本、人工审核资源和客户体验。

```python
def compute_ks(y_true: np.ndarray, y_prob: np.ndarray) -> tuple[float, float]:
    """计算 KS 统计量及对应阈值。"""
    fpr, tpr, thresholds = roc_curve(y_true, y_prob)
    ks_values = tpr - fpr
    ks_max = ks_values.max()
    ks_threshold = thresholds[ks_values.argmax()]
    return ks_max, ks_threshold


ks_lgb, thresh_lgb = compute_ks(yl_test.values, yl_prob_lgb)
ks_mlp, thresh_mlp = compute_ks(yl_test.values, yl_prob_mlp)

print(f"LightGBM KS: {ks_lgb:.4f}，参考阈值: {thresh_lgb:.4f}")
print(f"MLP      KS: {ks_mlp:.4f}，参考阈值: {thresh_mlp:.4f}")
```

`roc_curve` 会在多个阈值下计算 FPR 和 TPR。`ks_values` 是二者差值。`argmax` 找到最大差值对应的阈值。这个阈值不一定就是最终业务阈值，但可以作为策略分层的参考。消费贷案例的参考阈值常低于 0.5，这说明默认阈值可能严重低估坏客户识别需求。

```python
def predict_with_threshold(prob: np.ndarray, threshold: float) -> np.ndarray:
    return (prob >= threshold).astype(int)


yl_pred_lgb_ks = predict_with_threshold(yl_prob_lgb, thresh_lgb)
yl_pred_mlp_ks = predict_with_threshold(yl_prob_mlp, thresh_mlp)

print("=== LightGBM KS 阈值评估 ===")
print(classification_report(yl_test, yl_pred_lgb_ks, target_names=["正常还款", "违约/Charge-off"]))

print("=== MLP KS 阈值评估 ===")
print(classification_report(yl_test, yl_pred_mlp_ks, target_names=["正常还款", "违约/Charge-off"]))
```

`predict_with_threshold` 把连续概率转成类别。使用 KS 阈值后，Recall 往往会上升，但 Precision 可能下降。这体现了风险识别和客户通过率之间的权衡。生产系统通常不会只使用一个阈值，而会设置自动拒绝区、人工审核区和自动通过区。

```python
thresholds = np.arange(0.05, 0.60, 0.05)
threshold_rows = []

for t in thresholds:
    pred_t = (yl_prob_lgb >= t).astype(int)
    threshold_rows.append({
        "阈值": round(float(t), 2),
        "Precision": round(precision_score(yl_test, pred_t, zero_division=0), 4),
        "Recall": round(recall_score(yl_test, pred_t, zero_division=0), 4),
        "F1": round(f1_score(yl_test, pred_t, zero_division=0), 4),
        "预测违约比例": round(float(pred_t.mean()), 4),
    })

display(threshold_df)
```

阈值表让业务权衡更明确。阈值降低时，模型会把更多客户送入高风险区，Recall 通常提高，Precision 通常下降。阈值升高时，模型更谨慎地标记违约客户，Precision 可能提高，Recall 可能下降。审批策略应根据预期损失最小化，而不是机械选择 F1 最高的阈值。

### 4.1.3.4 PSI 稳定性分析

模型上线后，客户分布可能变化。宏观经济、获客渠道、审批政策、利率环境和欺诈攻击都会造成数据漂移。PSI 用于比较两个连续变量分布是否稳定。

$$PSI = \sum_{i=1}^{N} \left(P_i^{actual} - P_i^{expected}\right) \times \ln\left(\frac{P_i^{actual}}{P_i^{expected}}\right)$$

其中，`expected` 通常是训练集分布，`actual` 通常是测试集或线上新样本分布。经验上，PSI 小于 0.10 表示比较稳定，0.10 到 0.25 表示轻微漂移，大于 0.25 表示显著漂移。这个经验阈值不是法规标准，应结合业务场景、样本量和历史波动共同判断。

```python
def compute_psi(expected: np.ndarray, actual: np.ndarray, n_bins: int = 10) -> float:
    """计算 Population Stability Index，适用于连续变量或模型分数。"""
    expected = np.asarray(expected, dtype=float)
    actual = np.asarray(actual, dtype=float)
    breakpoints = np.quantile(expected, np.linspace(0, 1, n_bins + 1))
    breakpoints = np.unique(breakpoints)

    if len(breakpoints) <= 2:
        return 0.0

    breakpoints[0] = -np.inf
    breakpoints[-1] = np.inf

    psi = 0.0
    for i in range(len(breakpoints) - 1):
        mask_exp = (expected >= breakpoints[i]) & (expected < breakpoints[i + 1])
        mask_act = (actual >= breakpoints[i]) & (actual < breakpoints[i + 1])

        pct_exp = mask_exp.sum() / len(expected)
        pct_act = mask_act.sum() / len(actual)

        pct_exp = max(pct_exp, 1e-6)
        pct_act = max(pct_act, 1e-6)

        psi += (pct_act - pct_exp) * np.log(pct_act / pct_exp)

    return psi
```

`np.quantile` 根据训练集分数确定分桶边界。`np.unique` 处理大量重复值导致分位点重合的情况。第一和最后一个边界设为负无穷和正无穷，是为了覆盖所有测试样本。循环逐桶计算训练集和测试集占比。`1e-6` 用于防止除零和 `log(0)`。PSI 的优点是直观，缺点是受分桶方式影响，因此应和 AUC 分时段跟踪、坏账率跟踪、通过率监控共同使用。


```python
def compute_csi(expected: pd.Series, actual: pd.Series) -> float:
    """计算类别变量的稳定性指标。"""
    expected_pct = expected.astype(str).value_counts(normalize=True)
    actual_pct = actual.astype(str).value_counts(normalize=True)
    categories = sorted(set(expected_pct.index) | set(actual_pct.index))

    csi = 0.0
    for cat in categories:
        pct_exp = max(float(expected_pct.get(cat, 0.0)), 1e-6)
        pct_act = max(float(actual_pct.get(cat, 0.0)), 1e-6)
        csi += (pct_act - pct_exp) * np.log(pct_act / pct_exp)
    return csi


categorical_csi = {}
train_cleaned_for_csi = clean_lending_raw_features(train_lending_raw_df)
test_cleaned_for_csi = clean_lending_raw_features(test_lending_raw_df)

for feat in lending_preprocessor["categorical_columns"]:
    categorical_csi[feat] = compute_csi(
        train_cleaned_for_csi[feat].where(train_cleaned_for_csi[feat].notna(), "__missing__"),
        test_cleaned_for_csi[feat].where(test_cleaned_for_csi[feat].notna(), "__missing__"),
    )

categorical_csi_df = pd.Series(categorical_csi, name="CSI").sort_values(ascending=False).to_frame()
display(categorical_csi_df)
```

类别变量不适合直接用任意整数编码计算 PSI，因为整数大小并不代表类别之间的真实距离。`compute_csi` 按原始类别占比计算稳定性，更适合 `grade`、`purpose` 和 `home_ownership` 这类变量。数值变量使用 PSI，类别变量使用 CSI，能让稳定性监控更符合变量性质。

### 4.1.3.6 客户风险分级

模型输出的违约概率需要转化为业务策略。常见做法是把客户分为 A、B、C、D 四档。A 档风险最低，D 档风险最高。分级结果可用于差异化额度、利率、保证金、人工审核和贷后监控。

```python
def assign_risk_grade(prob: np.ndarray, n_grades: int = 4) -> np.ndarray:
    """基于预测违约概率将客户分为 A/B/C/D 四个风险等级。"""
    grade_labels = [chr(ord("A") + i) for i in range(n_grades)]
    boundaries = np.quantile(prob, np.linspace(0, 1, n_grades + 1))
    boundaries = np.unique(boundaries)
    if len(boundaries) <= 2:
        return np.array([grade_labels[0]] * len(prob))
    grades = np.digitize(prob, boundaries[1:-1])
    return np.array([grade_labels[min(g, n_grades - 1)] for g in grades])


risk_grades = assign_risk_grade(yl_prob_lgb, n_grades=4)

grade_analysis = pd.DataFrame({
    "风险等级": risk_grades,
    "实际违约": yl_test.values,
    "预测概率": yl_prob_lgb,
})

grade_summary = grade_analysis.groupby("风险等级").agg(
    客户数量=("实际违约", "count"),
    实际违约率=("实际违约", "mean"),
    平均预测概率=("预测概率", "mean"),
    最小预测概率=("预测概率", "min"),
    最大预测概率=("预测概率", "max"),
).round(4)

display(grade_summary)
```

`assign_risk_grade` 使用分位数分桶，使每个等级大致有相近数量的客户。`np.quantile` 计算分位点。`np.digitize` 根据边界把概率落入不同等级。`groupby` 汇总每个等级的客户数量、实际违约率和预测概率范围。好的分级应当满足实际违约率从 A 到 D 单调上升。如果出现倒挂，说明模型排序能力不足，或分级边界需要重新设计。

## 本章模型选择的综合讨论

Logistic 回归适合作为信用评分卡基线。它可解释、稳定、便于部署，也容易转化为评分和拒绝原因代码。它的局限在于表达非线性和变量交互的能力有限。决策树适合作为入门模型，因为它能直观展示审批规则。但单棵树容易过拟合，对数据扰动敏感，因此更适合解释教学和规则探索，不宜直接作为复杂消费贷业务的唯一模型。随机森林适合需要稳健性能和较少调参的场景。它通过样本随机和特征随机降低方差，但解释单个客户时不如单棵树清晰。XGBoost 和 LightGBM 适合结构化表格数据，是金融风控中常见的强基线。它们通常比单模型性能更好，但需要关注过拟合、参数调优、阈值选择、模型解释和上线监控。SVM 适合说明间隔、核函数和标准化的重要性。在中小规模数据上，它能提供有竞争力的结果。但在大规模消费贷数据上，RBF-SVM 计算成本较高，概率校准也需要额外处理。MLP 适合说明神经网络建模思想。它有能力拟合复杂非线性关系，但对标准化、学习率、网络结构、类别不平衡和训练稳定性敏感。在典型表格风控数据上，MLP 不一定优于梯度提升树。

实际风控项目中，模型选择不是单纯的算法竞赛。若业务强调监管解释和稳定性，Logistic 回归评分卡可能优先。若业务强调风险排序能力，LightGBM 或 XGBoost 往往更有优势。若业务需要自动化审批，还必须同时设计阈值策略、人工复核区间、拒绝原因生成、模型监控和重新训练机制。
