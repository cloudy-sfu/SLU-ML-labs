# 4.4 客户洞察与营销智能案例

客户洞察与营销智能是零售银行、财富管理、消费金融和保险机构的重要应用方向。金融机构面对的客户规模大、产品种类多、营销触点分散。如果只依靠人工经验，很难判断哪些客户应该被优先触达，哪些客户适合定期存款，哪些客户更可能对交叉销售产生响应，哪些客户应减少打扰以保护客户体验。数据驱动的客户洞察系统把客户属性、账户行为、营销互动和历史响应转化为可计算的特征，再通过分群、规则挖掘、推荐和自动化策略生成支持精准营销。

本章使用 UCI Bank Marketing 数据集展开。该数据集记录了葡萄牙某银行电话营销活动中的客户信息，目标变量是客户是否订阅定期存款产品。数据包含年龄、职业、婚姻、教育、账户余额、住房贷款、个人贷款、联系方式、联系日期、通话时长、联系次数、历史营销结果等字段。它适合说明客户分群、营销响应差异分析、关联规则挖掘和规则型推荐。

本章的核心任务分为三部分。第一，使用 K-Means 对客户进行无监督分群，并把数学簇转化为可解释的客户画像。第二，使用 Apriori 关联规则从客户行为篮子中挖掘高价值规则，并把规则用于精准营销推荐。第三，构建一个规则型 Agent，把客户分群、规则推荐、策略建议和审计记录组织成可执行的营销决策流程。

需要说明的是，生命周期价值预测通常需要客户收入、利润、交易频率、留存周期、渠道成本和产品成本等数据。本章使用的数据集没有真实收入流、利润率和客户生命周期长度。因此，本章不在该数据集上声称训练了真实 CLV 模型，而是在分群结果基础上介绍 CLV 的定义、建模条件和可落地框架，并用订阅率和账户余额构造营销优先级示例。这个处理方式符合建模原则，因为模型输出不能超出数据可支持的业务含义。

## 4.4.1 客户分群、生命周期价值预测

客户分群的目标，是把客户划分为若干内部相似、外部差异明显的人群。对银行营销而言，一个好的分群结果应满足三个条件。第一，每个群体应具有可解释的业务特征。第二，不同群体应在营销响应、产品偏好或服务成本上存在差异。第三，群体标签应能被后续策略系统使用，例如触达频率、推荐产品、渠道选择和人工跟进优先级。

### 4.4.1.4 聚类前预处理

K-Means 是无监督模型，不应把订阅结果 `y` 作为聚类特征。否则聚类会直接利用标签信息，形成“已订阅客户簇”和“未订阅客户簇”，这不再是对客户自然结构的发现。

```python
TARGET_COL = "y"


def prepare_features_for_clustering(
    df_input: pd.DataFrame,
) -> tuple[pd.DataFrame, pd.Series]:
    """完成聚类前的数据清洗，返回数值特征矩阵和目标变量。"""
    df_work = df_input.copy()

    if TARGET_COL in df_work.columns:
        y_target = df_work[TARGET_COL].copy()
        df_work = df_work.drop(columns=[TARGET_COL])
    else:
        y_target = pd.Series(dtype="object")

    cat_cols = df_work.select_dtypes(include=["category", "object"]).columns.tolist()
    num_cols = df_work.select_dtypes(include=[np.number]).columns.tolist()

    if cat_cols:
        encoder = OrdinalEncoder(handle_unknown="use_encoded_value", unknown_value=-1)
        df_work[cat_cols] = encoder.fit_transform(df_work[cat_cols].astype(str))

    all_cols = num_cols + cat_cols
    df_num = df_work[all_cols].copy()
    df_num = df_num.fillna(df_num.median(numeric_only=True))

    non_constant_cols = [c for c in df_num.columns if df_num[c].nunique() > 1]
    df_num = df_num[non_constant_cols]

    return df_num, y_target
```

`TARGET_COL` 指定目标变量列名。函数内部先复制原始 DataFrame。若存在 `y`，则把它保存到 `y_target` 并从聚类输入中删除。`select_dtypes` 分别识别分类列和数值列。`OrdinalEncoder` 将类别编码为整数，并用 `unknown_value=-1` 处理未来可能出现的新类别。`astype(str)` 统一类别类型，减少 category 类型在编码时的兼容问题。`fillna` 使用中位数填充数值缺失。`non_constant_cols` 删除只有一个取值的常量列，因为常量列不提供区分信息。

OrdinalEncoder 简单高效，适合教学演示，但它会给无序类别引入人为顺序。例如职业编码为 0、1、2 并不表示职业之间存在自然大小关系。更严格的客户分群可以使用 One-Hot 编码、Gower 距离、K-Prototypes，或只对数值特征聚类后再用类别特征画像。本章选择 OrdinalEncoder，是为了完整展示从原始混合数据到 K-Means 的基本流程，同时在解释中明确其局限。

```python
print(f"\n目标变量 ({TARGET_COL}) 分布：")
```

### 4.4.1.5 异常值检测与剔除

K-Means 使用欧氏距离并以均值作为簇中心，因此对异常值敏感。少量极端余额、极端通话时长或极高联系次数可能显著拉偏簇中心。IQR 方法是一种稳健的异常值检测方法。

$$IQR=Q_3-Q_1$$

若某个连续变量取值小于 `Q1 - 1.5 × IQR`，或大于 `Q3 + 1.5 × IQR`，则可被标记为异常。这个规则来自箱线图思想，适合快速发现极端值。但金融数据天然有长尾，删除异常样本前应确认这些样本是错误数据、极端客户，还是有业务价值的高净值客户。

```python
def detect_outliers_iqr(
    df_features: pd.DataFrame,
    continuous_cols: list[str],
    factor: float = 1.5,
) -> pd.Series:
    """基于 IQR 规则标记异常样本。"""
    is_outlier = pd.Series(False, index=df_features.index)
    for col in continuous_cols:
        q1 = df_features[col].quantile(0.25)
        q3 = df_features[col].quantile(0.75)
        iqr = q3 - q1
        lower = q1 - factor * iqr
        upper = q3 + factor * iqr
        is_outlier = is_outlier | (df_features[col] < lower) | (df_features[col] > upper)
    return is_outlier
```

`continuous_cols` 只包含原始连续数值变量，不包含经 OrdinalEncoder 编码的类别变量。函数先创建全 False 的布尔序列。循环中，`quantile(0.25)` 和 `quantile(0.75)` 计算四分位数。`lower` 和 `upper` 是异常边界。只要样本在任意一个连续变量上超出边界，就会被标记为异常。`|` 表示逐元素逻辑或。

```python
_original_num_cols = (
    df.drop(columns=[TARGET_COL], errors="ignore")
    .select_dtypes(include=[np.number])
    .columns
    .tolist()
)
continuous_cols = [c for c in _original_num_cols if c in X.columns]
print("用于异常值检测的连续特征：", continuous_cols)

outlier_mask = detect_outliers_iqr(X, continuous_cols)
n_outliers = outlier_mask.sum()
print(f"检测到异常样本数：{n_outliers}，占比 {n_outliers / len(X):.2%}")

X = X.loc[~outlier_mask].reset_index(drop=True)
y_target = y_target.loc[~outlier_mask].reset_index(drop=True)
print(f"剔除后数据维度：{X.shape}")
```

`_original_num_cols` 从编码前数据中识别原始数值列。`continuous_cols` 确保这些列在 `X` 中实际存在。`outlier_mask` 是异常样本标记。实验材料中，IQR 规则约剔除了 17018 条样本，占比约 37.64%。这个比例较高，说明该数据的余额、通话时长、联系次数等变量具有明显长尾。教学中这样做有助于展示 K-Means 对异常值的敏感性。正式项目中，可以比较删除、Winsorize 裁剪、RobustScaler、分位数变换和保留高价值长尾客户的不同方案。

### 4.4.1.6 特征标准化与数据切分

K-Means 最小化样本到簇中心的平方距离。如果某个变量的量纲很大，例如账户余额，其距离贡献会远大于职业编码或贷款状态。标准化可以把各变量放在相近尺度上。

$$z=\frac{x-\mu}{\sigma}$$

其中，`μ` 是训练集均值，`σ` 是训练集标准差。为了避免数据泄漏，标准化器只应在训练集上拟合。

```python
X_train, X_test, y_train, y_test = train_test_split(
    X,
    y_target,
    test_size=0.2,
    random_state=RANDOM_STATE,
)

print("训练集形状：", X_train.shape)
print("测试集形状：", X_test.shape)
```

`train_test_split` 把样本分成训练集和测试集。虽然聚类是无监督任务，仍然可以使用训练集拟合簇中心，再在测试集检查聚类结构是否稳定。这里没有使用 `stratify`，因为聚类输入不使用标签，且目标变量只用于事后画像。

```python
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)
X_all_scaled = scaler.transform(X)

```

`StandardScaler()` 创建标准化器。`fit_transform(X_train)` 在训练集上学习均值和标准差，并转换训练集。`transform(X_test)` 和 `transform(X)` 使用训练集统计量转换测试集和全量样本。这样做模拟了真实环境中用历史客户学习转换规则，再应用于未来客户的过程。

### 4.4.1.7 K-Means、肘部法与 Silhouette

K-Means 的目标是把样本划分到 k 个簇，使簇内平方和最小。其目标函数可写为

$$\min_{C_1,\ldots,C_k}\sum_{j=1}^{k}\sum_{x_i\in C_j}\lVert x_i-\mu_j\rVert^2$$

其中，`C_j` 是第 j 个簇，`μ_j` 是该簇中心。K-Means 迭代执行两个步骤。第一，把每个样本分配给最近的簇中心。第二，用簇内样本均值更新簇中心。算法简单、速度快、易解释，因此是客户分群的常用基线。

K-Means 的局限也很明确。它倾向发现球形簇，对异常值敏感，需要预先指定 k，并且使用欧氏距离。若客户群边界呈椭圆形、密度不均或类别变量较多，GMM、层次聚类、DBSCAN、K-Prototypes 或基于嵌入的聚类可能更合适。本章选择 K-Means，是因为它最适合作为客户分群入门方法，并且便于展示簇中心画像。

选择 k 常用肘部法和 Silhouette。肘部法观察 Inertia 随 k 增加的下降速度，下降明显变缓的位置可作为候选 k。Silhouette 衡量样本与本簇的紧密程度和与邻近簇的分离程度。

$$s(i)=\frac{b(i)-a(i)}{\max(a(i),b(i))}$$

其中，`a(i)` 是样本 i 到同簇其他样本的平均距离，`b(i)` 是样本 i 到最近其他簇样本的平均距离。Silhouette 取值范围为 -1 到 1，越大说明聚类越清晰。

```python
SILHOUETTE_SAMPLE_SIZE = 8000


def evaluate_k_candidates(
    X_scaled: np.ndarray,
    k_min: int = 2,
    k_max: int = 10,
    random_state: int = RANDOM_STATE,
) -> pd.DataFrame:
    """计算不同 k 的 Inertia 与 Silhouette 分数。"""
    rows: list[dict[str, float | int]] = []
    n_samples = X_scaled.shape[0]
    sil_sample = min(SILHOUETTE_SAMPLE_SIZE, n_samples)

    for k in range(k_min, k_max + 1):
        km = MiniBatchKMeans(
            n_clusters=k,
            random_state=random_state,
            n_init=10,
            batch_size=1024,
        )
        labels = km.fit_predict(X_scaled)
        sil = silhouette_score(
            X_scaled,
            labels,
            sample_size=sil_sample,
            random_state=random_state,
        )
        rows.append({"k": k, "inertia": km.inertia_, "silhouette": sil})
    return pd.DataFrame(rows)
```

`SILHOUETTE_SAMPLE_SIZE` 控制轮廓系数计算时的子采样规模。Silhouette 涉及样本之间距离，计算成本较高。`evaluate_k_candidates` 遍历 k 从 2 到 10。`MiniBatchKMeans` 用小批量更新簇中心，比标准 K-Means 更快。`n_init=10` 表示使用 10 次不同初始中心，取效果较好的结果。`batch_size=1024` 控制每批训练样本数量。`fit_predict` 同时训练模型并返回簇标签。`km.inertia_` 是簇内平方和。函数最终返回各 k 的指标表。

```python
def detect_elbow_k(k_values: np.ndarray, inertia_values: np.ndarray) -> int:
    """用首末点连线最大垂距方法自动检测肘点。"""
    if len(k_values) < 3:
        raise ValueError("肘部法至少需要 3 个候选 k。")

    x = k_values.astype(float)
    y = inertia_values.astype(float)
    x1, y1 = x[0], y[0]
    x2, y2 = x[-1], y[-1]
    line_norm = float(np.hypot(y2 - y1, x2 - x1))
    if line_norm == 0:
        return int(k_values[0])

    distances: list[float] = []
    for xi, yi in zip(x, y):
        numerator = abs((y2 - y1) * xi - (x2 - x1) * yi + x2 * y1 - y2 * x1)
        distances.append(float(numerator / line_norm))

    distance_array = np.array(distances, dtype=float)
    distance_array[[0, -1]] = -1.0
    return int(k_values[int(np.argmax(distance_array))])
```

`detect_elbow_k` 使用首末点连线最大垂距法检测肘点。`np.hypot` 计算连线长度。循环中计算每个候选点到首末连线的垂直距离。距离越大，说明该点越像曲线拐点。端点不应被视为肘点，因此把第一个和最后一个距离设为 -1。该方法是自动化辅助，不能替代业务判断。


### 4.4.1.8 测试集评估与最终聚类模型

为了检查聚类结构是否具有一定泛化性，可以在训练集拟合每个 k 的模型，再对测试集分配簇标签并计算测试集 Silhouette。

```python
def evaluate_k_on_test(
    X_train_scaled: np.ndarray,
    X_test_scaled: np.ndarray,
    k_min: int = 2,
    k_max: int = 10,
    random_state: int = RANDOM_STATE,
) -> pd.DataFrame:
    """在训练集拟合、在测试集上比较多 k 的 Silhouette。"""
    rows: list[dict[str, float | int]] = []
    n_test = X_test_scaled.shape[0]
    sil_sample = min(SILHOUETTE_SAMPLE_SIZE, n_test)

    for k in range(k_min, k_max + 1):
        km = MiniBatchKMeans(
            n_clusters=k,
            random_state=random_state,
            n_init=10,
            batch_size=1024,
        )
        km.fit(X_train_scaled)
        test_labels = km.predict(X_test_scaled)

        if np.unique(test_labels).size < 2:
            sil = np.nan
        else:
            sil = silhouette_score(
                X_test_scaled,
                test_labels,
                sample_size=sil_sample,
                random_state=random_state,
            )

        rows.append({
            "k": k,
            "train_inertia": km.inertia_,
            "test_silhouette": sil,
        })

    result = pd.DataFrame(rows)
    result["rank_silhouette"] = result["test_silhouette"].rank(ascending=False, method="min")
    return result
```

`evaluate_k_on_test` 与训练阶段评估函数的差别在于，模型只在训练集上 `fit`，再用 `predict` 给测试集分配标签。若测试集全部被分到一个簇，Silhouette 不可计算，因此返回 NaN。`rank_silhouette` 对测试集 Silhouette 从高到低排序。


### 4.4.1.10 客户画像与订阅率分析

聚类结果只有在被转译成业务画像后才有价值。画像应回答每个簇客户的规模、年龄、余额、贷款状态、联系方式、营销历史和订阅率。

```python
def profile_clusters(
    result_df: pd.DataFrame,
    feature_cols: list[str],
    y_target: pd.Series | None = None,
) -> pd.DataFrame:
    """输出每个簇在原始量纲下的均值和样本占比，可选附加订阅率。"""
    size = result_df.groupby("cluster").size().rename("样本数")
    means = result_df.groupby("cluster")[feature_cols].mean()
    profile = size.to_frame().join(means, how="left")
    profile["占比"] = profile["样本数"] / profile["样本数"].sum()

    if y_target is not None:
        y_str = y_target.astype(str)
        y_binary = y_str.map({"1": 0, "2": 1, "no": 0, "yes": 1}).astype(float)
        sub_df = pd.DataFrame({
            "cluster": result_df["cluster"].values,
            "subscribed": y_binary.values,
        })
        sub_rate = sub_df.groupby("cluster")["subscribed"].mean().rename("订阅率")
        profile = profile.join(sub_rate, how="left")

    return profile
```

`groupby("cluster").size()` 统计各簇样本数。`groupby("cluster")[feature_cols].mean()` 计算各簇特征均值。`占比` 表示每个簇在总体中的比例。若传入 `y_target`，函数把目标变量转为 0/1，其中 `1` 或 `no` 映射为未订阅，`2` 或 `yes` 映射为已订阅。`subscribed` 用于计算每簇订阅率。

### 4.4.1.11 生命周期价值的建模框架

客户生命周期价值通常定义为客户在未来生命周期内为企业贡献的折现净收益。一个常见形式为

$$CLV_i=\sum_{t=1}^{T}\frac{P_i(\text{active at }t)\times E[margin_{it}]-E[cost_{it}]}{(1+r)^t}$$

其中，`P(active at t)` 是客户在第 t 期仍然活跃的概率，`margin` 是预计利润贡献，`cost` 是维护和营销成本，`r` 是折现率。若有交易流水，可以用购买频率、客单价、毛利率、留存概率和渠道成本估计 CLV。若有订阅产品，可以用预期存款规模、利差收入、留存期限和营销成本估计产品级价值。

本章数据没有真实利润、产品余额变动、留存时长和营销成本，因此不能训练严格意义上的 CLV 预测模型。可以做的是构造营销优先级示例，把订阅率作为响应倾向，把账户余额作为潜在资金规模信号。这个指标不是 CLV，只是用于教学的分群策略评分。

```python
marketing_priority = cluster_profile[["样本数", "balance", "订阅率"]].copy()
marketing_priority["潜在订阅人数"] = marketing_priority["样本数"] * marketing_priority["订阅率"]

for col in ["balance", "订阅率", "潜在订阅人数"]:
    col_min = marketing_priority[col].min()
    col_max = marketing_priority[col].max()
    if col_max > col_min:
        marketing_priority[f"{col}_norm"] = (marketing_priority[col] - col_min) / (col_max - col_min)
    else:
        marketing_priority[f"{col}_norm"] = 0.0

marketing_priority["营销优先级得分"] = (
    0.5 * marketing_priority["订阅率_norm"]
    + 0.3 * marketing_priority["潜在订阅人数_norm"]
    + 0.2 * marketing_priority["balance_norm"]
)

marketing_priority = marketing_priority.sort_values("营销优先级得分", ascending=False)
display(marketing_priority)
```

`潜在订阅人数` 用样本数乘以订阅率，表示该群体中可能产生订阅响应的人数规模。归一化步骤把余额、订阅率和潜在订阅人数缩放到 0 到 1 之间。最后的加权得分给订阅率 50% 权重、潜在订阅人数 30% 权重、余额 20% 权重。权重只是教学示例，正式项目应通过收益、成本和 A/B 测试结果确定。

如果要把本节扩展为真实 CLV 项目，至少需要补充客户级交易流水、产品持有记录、营销成本、产品利润率、客户流失标签和时间窗口。模型上可以先用 Gamma-Gamma 模型、BG/NBD 模型、随机森林回归、梯度提升回归或生存分析模型作为基线。若业务目标是营销触达，应同时估计响应概率和增量效果，而不是只估计平均价值。

### 4.4.1.12 聚类稳定性检查

K-Means 对初始中心敏感。如果不同随机种子产生明显不同的结果，说明分群结构不稳，不适合直接用于资源分配。可以用多个随机种子重复训练，观察 Inertia 和 Silhouette 波动。

```python
def stability_check(X_scaled: np.ndarray, k: int, seeds: list[int]) -> pd.DataFrame:
    """使用不同随机种子训练 K-Means，记录 Inertia 与 Silhouette。"""
    rows = []
    n_samples = X_scaled.shape[0]
    sil_sample = min(SILHOUETTE_SAMPLE_SIZE, n_samples)

    for seed in seeds:
        km = KMeans(n_clusters=k, random_state=seed, n_init=10)
        labels = km.fit_predict(X_scaled)
        rows.append({
            "seed": seed,
            "inertia": km.inertia_,
            "silhouette": silhouette_score(
                X_scaled,
                labels,
                sample_size=sil_sample,
                random_state=seed,
            ),
        })
    return pd.DataFrame(rows)


stability_df = stability_check(X_train_scaled, k=FINAL_K, seeds=[1, 7, 21, 37, 77])
display(stability_df)
print("Silhouette 标准差：", round(stability_df["silhouette"].std(), 4))
```

函数循环使用不同随机种子训练 K-Means。`n_init=10` 已经降低了初始化偶然性，但稳定性检查仍然有价值。实验材料中，Silhouette 标准差约为 0.0013，说明 k=2 下的结果比较稳定。稳定并不表示业务价值一定高，它只表示模型对初始化不敏感。

## 4.4.2 推荐系统与精准营销实践

推荐系统的目标，是根据客户已有特征、历史行为和相似客户模式，生成下一步产品或服务建议。推荐系统有多种实现方式。协同过滤依赖用户和产品交互矩阵，适合电商、内容平台和多产品金融场景。内容推荐依赖客户特征和产品特征，适合新产品冷启动。监督学习可以直接预测客户对某产品的响应概率。关联规则则适合发现“具有某些特征的客户，还常常具有哪些行为”的模式。

本节使用 Apriori 关联规则。选择 Apriori 的原因是它不需要复杂模型训练，规则表达直观，能够直接生成业务人员可读的洞察。它的局限是不能很好表达连续强度、时间顺序和因果关系，也容易受阈值影响。因此，关联规则更适合作为客户洞察、营销策略启发和规则推荐基线，而不是唯一的自动决策模型。

### 4.4.2.2 行为篮子数据构造

关联规则要求输入是布尔型事务矩阵。每一行代表一位客户，每一列代表一个二元特征。`True` 表示客户具有该特征，`False` 表示不具有。

```python
def build_basket(df: pd.DataFrame) -> pd.DataFrame:
    """将 UCI Bank Marketing 原始数据转化为布尔型行为篮子数据。"""
    basket = pd.DataFrame(index=df.index)

    cat_cols = [
        "job", "marital", "education", "default", "housing",
        "loan", "contact", "poutcome", "y",
    ]
    for col in cat_cols:
        if col in df.columns:
            df[col] = df[col].astype(str).str.strip().str.lower()

    age = pd.to_numeric(df["age"], errors="coerce")
    basket["年龄:青年(≤30)"] = age <= 30
    basket["年龄:中年(31-50)"] = (age > 30) & (age <= 50)
    basket["年龄:中老年(>50)"] = age > 50

    job_map = {
        "management": "职业:管理层",
        "admin.": "职业:管理层",
        "blue-collar": "职业:蓝领",
        "technician": "职业:技术人员",
        "services": "职业:服务业",
        "entrepreneur": "职业:自雇",
        "self-employed": "职业:自雇",
        "retired": "职业:退休",
        "student": "职业:学生",
        "housemaid": "职业:其他",
        "unemployed": "职业:其他",
        "unknown": "职业:其他",
    }
    job_mapped = df["job"].map(job_map).fillna("职业:其他")
    for label in sorted(job_mapped.unique()):
        basket[label] = job_mapped == label

    basket["婚姻:已婚"] = df["marital"] == "married"
    basket["婚姻:单身"] = df["marital"] == "single"
    basket["婚姻:离异"] = df["marital"] == "divorced"

    basket["学历:高等"] = df["education"] == "tertiary"
    basket["学历:中等"] = df["education"] == "secondary"
    basket["学历:初等"] = df["education"] == "primary"

    basket["有违约记录"] = df["default"] == "yes"

    balance = pd.to_numeric(df["balance"], errors="coerce")
    q33 = balance.quantile(0.33)
    q66 = balance.quantile(0.66)
    basket["余额:低/负"] = balance <= q33
    basket["余额:中"] = (balance > q33) & (balance <= q66)
    basket["余额:高"] = balance > q66

    basket["有房贷"] = df["housing"] == "yes"
    basket["无房贷"] = df["housing"] == "no"

    basket["有个人贷款"] = df["loan"] == "yes"
    basket["无个人贷款"] = df["loan"] == "no"

    basket["前次营销:成功"] = df["poutcome"] == "success"
    basket["前次营销:失败"] = df["poutcome"] == "failure"

    y_col = df["y"]
    basket["定期存款:是"] = (y_col == "yes") | (y_col == "2") | (y_col == "yes.")
    basket["定期存款:否"] = (y_col == "no") | (y_col == "1") | (y_col == "no.")

    col_rates = basket.mean()
    keep_cols = col_rates[(col_rates >= 0.02) & (col_rates <= 0.98)].index
    basket = basket[keep_cols]

    return basket.astype(bool)


df_transactions = build_basket(df_marketing_raw.copy())
print("行为篮子数据维度：", df_transactions.shape)
print(f"{df_transactions.shape[0]} 位客户 × {df_transactions.shape[1]} 个行为特征")
```

`basket = pd.DataFrame(index=df.index)` 创建与原数据行索引一致的空表。`cat_cols` 指定需要转为小写字符串的类别列。这样可以避免 `Yes`、`yes` 和空格导致的比较错误。年龄被分为青年、中年和中老年三个区间。职业使用 `job_map` 合并为管理层、蓝领、技术人员、服务业、自雇、退休、学生和其他。婚姻、教育、违约、房贷、个人贷款和历史营销结果被转为布尔特征。余额按 33% 和 66% 分位数分为低、中、高。目标变量被转为 `定期存款:是` 和 `定期存款:否` 两个特征。

最后，代码移除覆盖率低于 2% 或高于 98% 的特征。过低覆盖率的特征容易形成偶然规则，过高覆盖率的特征往往缺少区分力。实验材料中，最终行为篮子约为 45211 行和 28 个行为特征。

### 4.4.2.4 支持度、置信度、提升度与 Apriori

支持度衡量项集在全部客户中的出现比例。

$$support(X)=\frac{包含X的客户数}{客户总数}$$

置信度衡量在具有前件 X 的客户中，同时具有后件 Y 的比例。

$$confidence(X\Rightarrow Y)=\frac{support(X\cup Y)}{support(X)}$$

提升度衡量规则相对于随机基线的提升。

$$lift(X\Rightarrow Y)=\frac{confidence(X\Rightarrow Y)}{support(Y)}$$

若 Lift 大于 1，说明 X 与 Y 正相关。若 Lift 等于 1，说明二者近似独立。若 Lift 小于 1，说明 X 与 Y 负相关。营销场景通常优先关注 Lift 显著大于 1 且 Confidence 足够高的规则。

Apriori 算法利用向下封闭性剪枝。如果一个项集不是频繁项集，那么包含它的所有更大项集也不可能频繁。这个性质可以大幅减少候选项集数量。

### 4.4.2.7 基于规则的客户推荐

关联规则可以用于简单推荐。给定一个客户当前已知的行为特征集合，如果某条规则的前件是客户特征的子集，且后件包含客户尚未具有的特征，就可以把后件作为洞察或推荐方向。

```python
def recommend_customer_behavior(
    customer_features: set[str],
    rules_df: pd.DataFrame,
    top_n: int = 5,
) -> pd.DataFrame:
    """根据客户已知行为特征，从关联规则中筛选预测或洞察。"""
    recommendations = []
    for _, row in rules_df.iterrows():
        antecedents = set(row["antecedents"])
        consequents = set(row["consequents"])

        if not antecedents.issubset(customer_features):
            continue

        for item in consequents:
            if item not in customer_features:
                recommendations.append({
                    "已知特征（前件）": ", ".join(sorted(antecedents)),
                    "预测特征（后件）": item,
                    "Confidence": round(row["confidence"], 4),
                    "Lift": round(row["lift"], 4),
                    "Support": round(row["support"], 4),
                })

    if not recommendations:
        print("未找到匹配的推荐规则。")
        return pd.DataFrame()

    rec_df = pd.DataFrame(recommendations)
    rec_df = (
        rec_df.sort_values("Lift", ascending=False)
        .drop_duplicates(subset=["预测特征（后件）"])
        .head(top_n)
        .reset_index(drop=True)
    )
    return rec_df
```

`customer_features` 是客户已知特征集合。循环逐条读取规则。`antecedents.issubset(customer_features)` 判断规则前件是否完全被客户特征覆盖。若满足，代码遍历后件中的每个预测特征。若后件特征尚未出现在客户特征中，就加入推荐列表。最后按 Lift 降序排序，并对同一个预测特征只保留 Lift 最高的规则。

```python
customer_A = {"年龄:中年(31-50)", "婚姻:已婚", "有房贷", "学历:高等"}
print(f"客户 A 已知特征：{customer_A}")

```

客户 A 是中年、已婚、有房贷、学历高等的人群。实验材料中，该客户可能得到与“职业:管理层”“无个人贷款”“定期存款:否”等相关的洞察。若推荐出现 `定期存款:否`，不应理解为一定不营销，而应理解为该客户在历史数据中与未订阅行为存在关联，需要设计更合适的产品或渠道。

### 4.4.2.8 参数敏感性分析

Apriori 的结果高度依赖 `min_support` 和 `min_confidence`。阈值过低会产生大量规则，噪声和计算成本上升。阈值过高会只留下大众特征，遗漏有价值的细分人群。

```python
support_values = [0.08, 0.10, 0.15, 0.20, 0.25, 0.30]
confidence_values = [0.30, 0.40, 0.50, 0.60, 0.70, 0.80]

results_grid = []
for s in support_values:
    fi = apriori(df_bool, min_support=s, use_colnames=True)
    for c in confidence_values:
        if len(fi) == 0:
            n_rules = 0
        else:
            try:
                r = association_rules(fi, metric="confidence", min_threshold=c)
                r = r[r["lift"] > 1.0]
                n_rules = len(r)
            except Exception:
                n_rules = 0
        results_grid.append({
            "min_support": s,
            "min_confidence": c,
            "n_rules": n_rules,
        })

grid_df = pd.DataFrame(results_grid)
pivot = grid_df.pivot(
    index="min_support",
    columns="min_confidence",
    values="n_rules",
)
```

外层循环遍历最小支持度，内层循环遍历最小置信度。每组参数先运行 `apriori`，再运行 `association_rules`，最后只统计 Lift 大于 1 的规则数量。`try...except` 防止某些参数组合下没有可生成规则而中断运行。`pivot` 把结果整理为二维表。

## 4.4.3 Agentic AI 辅助决策与自动策略生成

Agentic AI 在营销智能中不应替代统计模型或业务审批，而应编排任务。它可以读取分群结果和规则推荐，生成客户摘要，提出触达策略，检查是否需要人工复核，并保存审计记录。它不应擅自改变客户标签、伪造收益预测，或在没有授权的情况下自动发送营销信息。

本节构建一个规则型营销 Agent。它使用前两节得到的客户分群结果和关联规则结果，生成结构化建议。这里不调用外部大语言模型，目的是让教学案例可复现、可审计、可解释。

### 4.4.3.1 分群策略表

首先把分群画像转化为策略表。策略表可以被 Agent 读取，也可以由业务人员审核。

```python
def build_cluster_strategy_table(cluster_profile: pd.DataFrame) -> pd.DataFrame:
    """根据簇画像生成营销策略表。"""
    strategy = cluster_profile[["样本数", "占比", "订阅率", "balance", "campaign"]].copy()
    overall_sub_rate = (
        (strategy["订阅率"] * strategy["样本数"]).sum()
        / strategy["样本数"].sum()
    )

    def assign_strategy(row: pd.Series) -> str:
        if row["订阅率"] >= overall_sub_rate and row["balance"] >= strategy["balance"].median():
            return "优先触达，推荐定期存款或稳健理财"
        if row["订阅率"] >= overall_sub_rate:
            return "保持触达，重点优化话术和渠道"
        if row["campaign"] > strategy["campaign"].median():
            return "降低电话频次，改用低打扰渠道"
        return "观察维护，结合规则推荐选择产品"

    strategy["相对订阅率"] = strategy["订阅率"] / overall_sub_rate
    strategy["建议策略"] = strategy.apply(assign_strategy, axis=1)
    return strategy.sort_values("订阅率", ascending=False)


cluster_strategy_table = build_cluster_strategy_table(cluster_profile)
display(cluster_strategy_table)
```

`overall_sub_rate` 是全体客户的加权订阅率。`assign_strategy` 根据订阅率、余额和联系次数生成策略。若某簇订阅率高且余额高，可以优先推荐定期存款或稳健理财。若订阅率低且联系次数高，应考虑减少电话打扰，改用短信、App 或内容运营。`相对订阅率` 表示该簇订阅率相对于总体平均水平的倍数。

### 4.4.3.3 营销策略 Agent

```python
class MarketingStrategyAgent:
    """客户洞察与营销策略规则 Agent。"""

    def __init__(
        self,
        rules_df: pd.DataFrame,
        cluster_strategy: pd.DataFrame,
        policy_version: str = "marketing_strategy_policy_v1",
    ) -> None:
        self.rules_df = rules_df
        self.cluster_strategy = cluster_strategy
        self.policy_version = policy_version

    def recommend_by_rules(self, features: set[str], top_n: int = 3) -> pd.DataFrame:
        return recommend_customer_behavior(features, self.rules_df, top_n=top_n)

    def summarize_cluster(self, cluster_id: int) -> str:
        if cluster_id not in self.cluster_strategy.index:
            return "未找到该客户所属簇的策略配置。"
        row = self.cluster_strategy.loc[cluster_id]
        return (
            f"客户所属簇为 {cluster_id}，该簇订阅率为 {row['订阅率']:.2%}，"
            f"相对总体订阅率为 {row['相对订阅率']:.2f} 倍。"
            f"建议策略是：{row['建议策略']}。"
        )

    def build_customer_report(self, customer_row: pd.Series) -> dict:
        customer_id = customer_row["customer_id"]
        cluster_id = int(customer_row["cluster"])
        features = set(customer_row["features"])
        rec_df = self.recommend_by_rules(features, top_n=3)
        cluster_summary = self.summarize_cluster(cluster_id)

        if rec_df.empty:
            rule_summary = "未发现高置信度规则推荐，建议使用分群策略作为主依据。"
            top_recommendation = "无"
        else:
            first = rec_df.iloc[0]
            top_recommendation = first["预测特征（后件）"]
            rule_summary = (
                f"最高 Lift 规则预测特征为 {top_recommendation}，"
                f"Confidence={first['Confidence']:.4f}，"
                f"Lift={first['Lift']:.4f}，"
                f"Support={first['Support']:.4f}。"
            )

        manual_review = False
        if "定期存款:否" == top_recommendation:
            manual_review = True
        if "有违约记录" in features:
            manual_review = True

        return {
            "customer_id": customer_id,
            "cluster": cluster_id,
            "已知特征": ", ".join(sorted(features)),
            "分群摘要": cluster_summary,
            "规则摘要": rule_summary,
            "首要推荐或洞察": top_recommendation,
            "是否需要人工复核": manual_review,
            "策略版本": self.policy_version,
        }

    def run(self, customers: pd.DataFrame) -> tuple[pd.DataFrame, pd.DataFrame]:
        reports = [self.build_customer_report(row) for _, row in customers.iterrows()]
        report_df = pd.DataFrame(reports)
        audit_log = report_df[[
            "customer_id", "cluster", "首要推荐或洞察", "是否需要人工复核", "策略版本",
        ]].copy()
        return report_df, audit_log
```

`MarketingStrategyAgent` 把规则推荐和分群策略封装在一个对象中。`__init__` 保存规则表、分群策略表和策略版本。`recommend_by_rules` 调用前一节的推荐函数。`summarize_cluster` 读取客户所属簇的订阅率和策略建议。`build_customer_report` 为单个客户生成报告，包含客户编号、簇、已知特征、分群摘要、规则摘要、首要推荐或洞察、人工复核标记和策略版本。`run` 对客户表逐行生成报告，并输出审计日志。

人工复核规则只是示例。若首要洞察为 `定期存款:否`，系统不应自动放弃客户，而应让业务人员判断是否改用其他产品。若客户已有违约记录，也应避免激进营销或不合规推荐。

```python
marketing_agent = MarketingStrategyAgent(
    rules_df=rules,
    cluster_strategy=cluster_strategy_table,
)

final_marketing_report, marketing_audit_log = marketing_agent.run(sample_customer_profiles)
display(final_marketing_report)
display(marketing_audit_log)
```

这段代码实例化 Agent 并运行。`final_marketing_report` 面向客户经理，提供可读摘要。`marketing_audit_log` 面向模型治理和合规审计，记录关键决策字段。正式系统中，审计日志还应包含运行时间、模型版本、数据版本、特征版本、阈值版本和营销授权状态。

### 4.4.3.4 Agentic AI 的边界

营销 Agent 可以承担重复、规范和可审计的任务。它可以整理客户画像，匹配关联规则，生成营销摘要，提醒人工复核，生成策略版本记录。它不应擅自修改分群标签，不应把弱相关规则包装成确定结论，不应在客户未授权的情况下触达客户，也不应绕过金融产品适当性要求。

在金融营销中，最稳健的架构是模型、规则、Agent 和人工协作。分群模型提供客户结构。关联规则提供行为洞察。规则系统提供可审计阈值和动作。Agent 负责把这些信息编排成工作流。人工负责关键客户、高影响动作和合规边界的最终判断。

## 本章模型选择的综合讨论

K-Means 适合作为客户分群的基线。它速度快、结果易解释、簇中心可以直接用于画像。但它要求数值输入，依赖欧氏距离，对异常值和特征尺度敏感，并且需要预先指定 k。对于类别特征较多的客户数据，K-Prototypes、Gower 距离聚类或 One-Hot 后聚类可能更合适。对于非球形客户结构，GMM、谱聚类或层次聚类可能更合适。

MiniBatchKMeans 适合大样本快速比较 k。它牺牲少量精度换取明显速度提升，因此适合调参阶段。最终模型若样本规模允许，可以使用标准 KMeans 重新训练，以获得更稳定的簇中心。

Silhouette 和肘部法是互补工具。Silhouette 偏向选择几何分离更清晰的 k。肘部法关注继续增加簇数是否显著降低簇内平方和。二者不一致时，应结合客户经理能否理解、策略是否能执行、每个簇样本量是否足够、线上 A/B 测试是否带来收益来决定。

Apriori 关联规则适合生成可解释的客户洞察。Support、Confidence 和 Lift 让业务人员能够理解规则覆盖面、命中率和相对提升。它的局限在于不能表达因果关系，不能自然处理连续变量，强依赖离散化方式和阈值设定。若目标是最大化营销收益，最终仍应引入响应模型、增量模型、成本收益模型和实验评估。

规则推荐适合冷启动和解释性要求高的场景。它不需要复杂用户产品交互矩阵，也容易落地到 CRM 系统。但规则推荐覆盖有限，无法捕捉复杂非线性关系。若银行拥有多产品持有记录和交易流水，可以进一步使用协同过滤、矩阵分解、双塔召回模型、梯度提升响应模型或因果 uplift 模型。

CLV 是营销智能的长期目标，但它对数据要求更高。没有收入、利润、成本和留存数据时，不应把订阅率或余额包装成 CLV。合理做法是先建立分群和响应分析，再逐步接入产品利润、客户留存和渠道成本，构建真实生命周期价值模型。

## 拓展方向

1. 使用 One-Hot 编码替代 OrdinalEncoder，比较 K-Means 聚类结果、Silhouette 和业务画像差异。
2. 使用 K-Prototypes 处理数值变量和类别变量混合数据，并与 K-Means 比较。
3. 在聚类前使用 RobustScaler 或 Winsorize 替代 IQR 删除，观察高余额客户是否形成独立高价值群体。
4. 对 `duration` 进行评分时点分析，比较包含和不包含通话时长时的客户画像差异。
5. 把 k=2 和 k=6 的分群同时交给业务人员命名，比较哪个分群更便于制定营销动作。
6. 专门筛选后件为 `定期存款:是` 的关联规则，建立定期存款推荐清单。
7. 用 FP-Growth 替代 Apriori，比较大规模数据下的运行速度和规则结果。
8. 引入响应预测模型，例如 Logistic 回归、随机森林或 LightGBM，预测客户订阅概率，并与关联规则推荐比较。
9. 设计营销 A/B 测试，把客户随机分配到规则推荐组、分群策略组和常规策略组，比较订阅率、投诉率和净收益。
10. 接入真实交易流水和产品利润数据，构建 CLV 模型，并把 CLV、响应概率和触达成本合并为期望收益排序。
11. 将 Agent 接入 CRM 工单系统，使其生成客户摘要和触达建议，但保留人工确认、营销授权检查和审计日志。
