# 5.3 综合实训项目3：金融新闻舆情驱动的智能决策系统

金融市场中的价格变化不仅来自财务报表、宏观指标和交易行为，也来自新闻、公告、研究报告和社交媒体所形成的信息流。新闻文本往往先以非结构化形式出现，随后被投资者、风控人员和交易系统解释为利好、利空或中性信息。金融新闻舆情驱动的智能决策系统，就是把这种文本信息转化为可计算、可验证、可审计的风险信号，并把信号进一步接入决策模拟和风险预警流程。

本节构建一个完整的综合实训项目。项目以 Financial PhraseBank 作为带标签金融情绪语料，以 FNSPID 中的 AAPL 新闻作为新闻事件数据，以 Yahoo Finance 行情作为市场反应数据。模型层面包含三种文本情绪识别路线。第一种路线是 TF-IDF 与朴素贝叶斯，它提供速度快、成本低、可解释的基线。第二种路线是 FinBERT，它利用金融领域预训练模型进行三分类推理。第三种路线是大语言模型文本嵌入与逻辑回归，它体现云端大模型语义能力与轻量监督模型结合的工程范式。系统层面把单条新闻情绪聚合为日度情绪指标，并进一步完成股价联动分析、事件窗口分析、策略决策模拟和 Agentic AI 风险预警。

本项目不是为了证明新闻情绪一定可以稳定预测股票收益，也不是为了构造自动交易系统。它的训练重点在于掌握从非结构化文本到结构化风险信号的建模流程，理解文本模型、时间对齐、样本外验证、交易成本、风险阈值和审计日志之间的关系。学生应始终注意，金融文本模型只能提供辅助信息，不能替代投资研究、合规审查和风险管理流程。

## 5.3.1 文本挖掘、情感分析、决策模拟

### 5.3.1.1 项目问题与系统结构

本项目解决的问题可以表述为：给定某一只股票的新闻流和历史行情，系统如何自动识别新闻情绪，如何把新闻级别情绪转化为交易日级别信号，如何观察情绪与收益率之间的关系，如何设计一个只用于教学和验证的决策模拟框架。

该问题可以分成五个环节。第一个环节是数据获取。带标签语料用于训练和测试情绪模型，新闻流用于构造实际舆情指标，行情数据用于观察市场反应。第二个环节是文本预处理。词袋模型需要较强清洗，BERT 和嵌入模型需要保留更多语义信息。第三个环节是情绪建模。系统比较传统机器学习、领域预训练模型和通用大模型嵌入三条路线。第四个环节是事件与决策模拟。新闻情绪被聚合为日度指标，并与收益率、异常收益和模拟策略收益连接起来。第五个环节是风险预警。Agent 使用规则、阈值、滚动统计和审计日志，把模型输出转化为可复核的风险提示。

选择这一结构的原因在于，金融舆情项目很容易被误解为单一的文本分类任务。事实上，分类模型只是系统的一部分。若缺少时间对齐，新闻情绪可能与错误的交易日合并。若缺少样本外验证，策略收益可能存在前视偏差。若缺少交易成本，模拟收益会被高估。若缺少审计日志，系统无法满足金融机构对可追踪性的要求。

### 5.3.1.3 数据来源与数据角色

Financial PhraseBank 是带标签金融句子数据。本项目使用其中标注者一致的子集，用于训练和测试情绪分类器。其标签通常包括 positive、neutral 和 negative。这个数据集规模不大，但标签一致性较好，适合作为教学项目中的监督学习样本。

FNSPID 是金融新闻与股票价格相关数据集。本项目使用其 Nasdaq-100 过滤版本，并筛选 AAPL 新闻作为案例。完整新闻数据规模较大，采用 streaming 方式可以边读取边过滤，不必一次性下载完整文件。

Yahoo Finance 行情数据用于计算收益率。收益率是观察新闻情绪与市场反应关系的核心变量。直接建模价格通常会受到非平稳性影响，因此本项目主要使用普通日收益率和异常收益率。

```python
from huggingface_hub import hf_hub_download
from datasets import load_dataset
import yfinance as yf


def load_financial_phrasebank() -> tuple[pd.Series, pd.Series]:
    """加载 Financial PhraseBank 的 Sentences_AllAgree 子集。"""
    # Ref: https://huggingface.co/datasets/financial_phrasebank
    zip_path = hf_hub_download(
        repo_id="financial_phrasebank",
        repo_type="dataset",
        filename="data/FinancialPhraseBank-v1.0.zip",
    )
    with zipfile.ZipFile(zip_path) as zf:
        raw_lines = zf.read(
            "FinancialPhraseBank-v1.0/Sentences_AllAgree.txt"
        ).decode("latin-1").splitlines()

    records = []
    for line in raw_lines:
        sentence, label = line.rsplit("@", 1)
        records.append({"sentence": sentence.strip(), "label": label.strip()})

    df = pd.DataFrame(records)
    return df["sentence"], df["label"]


X_fpb, y_fpb = load_financial_phrasebank()
print(f"Financial PhraseBank 样本数: {len(X_fpb)}")
print(y_fpb.value_counts())
```

`hf_hub_download` 从 HuggingFace 数据集仓库下载压缩文件。`zipfile.ZipFile` 打开压缩包。文件中的每行由句子和标签组成，中间用 `@` 分隔。代码使用 `rsplit("@", 1)` 从右侧分割一次，是为了避免句子中出现其他 `@` 字符时错误切分。函数返回文本序列和标签序列。

标签分布通常不均衡，neutral 类最多，negative 类最少。金融风控场景中，少数负面新闻往往具有较高业务价值，因此后续评估不能只看 Accuracy，还要观察 macro F1 和负面类召回率。

```python
FNSPID_NEWS_CACHE = DATA_DIR / f"fnspid_news_{TARGET_TICKER.lower()}.parquet"


def load_fnspid_news_streaming(
    ticker: str = TARGET_TICKER,
    cache_path: Path = FNSPID_NEWS_CACHE,
    max_rows: int | None = None,
) -> pd.DataFrame:
    """通过 HuggingFace streaming 加载单只股票的 FNSPID 新闻子集。"""
    # Ref: https://huggingface.co/docs/datasets/en/stream
    # Ref: https://huggingface.co/datasets/benstaf/FNSPID-filtered-nasdaq-100
    if cache_path.exists():
        print(f"从本地缓存加载: {cache_path}")
        df = pd.read_parquet(cache_path)
        if max_rows is not None:
            df = df.head(max_rows)
        return df

    print(f"正在流式加载 {ticker} 新闻数据。")
    ds = load_dataset(
        "benstaf/FNSPID-filtered-nasdaq-100",
        split="train",
        streaming=True,
    )
    ticker_upper = ticker.upper()
    filtered = ds.filter(
        lambda row: str(row.get("Stock_symbol", "")).upper() == ticker_upper
    )

    rows = []
    for i, row in enumerate(filtered):
        rows.append(row)
        if (i + 1) % 1000 == 0:
            print(f"已收集 {i + 1} 条", end="\r")

    if not rows:
        raise ValueError(f"未找到 ticker={ticker} 的新闻记录。")

    df = pd.DataFrame(rows)
    cache_path.parent.mkdir(parents=True, exist_ok=True)
    df.to_parquet(cache_path, index=False)
    print(f"已缓存到: {cache_path}")

    if max_rows is not None:
        df = df.head(max_rows)
    return df
```

函数先检查本地 Parquet 缓存。若缓存存在，直接读取，避免重复扫描远程数据。若缓存不存在，则用 `load_dataset(..., streaming=True)` 创建流式数据集，再按照 `Stock_symbol` 过滤目标股票。流式加载适合大数据集教学，因为它降低了内存压力。最后把结果写入本地缓存，使后续运行更稳定。

### 5.3.1.4 文本清洗与训练测试划分

文本清洗要根据模型类型决定。词袋模型把文本拆成词项并统计频率，因而通常需要小写化、删除 URL、删除 HTML 标签和规范特殊字符。BERT 和嵌入模型能够利用大小写、标点和上下文，过度清洗可能损失语义。因此项目保留两套清洗函数。

```python
def clean_text_for_embedding(text: str) -> str:
    """轻量清洗，适用于 BERT 和大模型嵌入。"""
    if not isinstance(text, str):
        return ""
    text = re.sub(r"<[^>]+>", " ", text)
    text = re.sub(r"http\S+", " ", text)
    text = re.sub(r"\s+", " ", text).strip()
    return text


def clean_text_bow(text: str) -> str:
    """较强清洗，适用于词袋和 TF-IDF 模型。"""
    text = clean_text_for_embedding(text)
    text = re.sub(r"[^a-zA-Z0-9\s.,!?'-]", " ", text)
    text = re.sub(r"\s+", " ", text).strip().lower()
    return text


X_train, X_test, y_train, y_test = train_test_split(
    X_fpb,
    y_fpb,
    test_size=TEST_SIZE,
    random_state=RANDOM_STATE,
    stratify=y_fpb,
)

X_train_bow = X_train.map(clean_text_bow)
X_test_bow = X_test.map(clean_text_bow)
X_train_embed = X_train.map(clean_text_for_embedding)
X_test_embed = X_test.map(clean_text_for_embedding)

print(f"训练集: {len(X_train)}  测试集: {len(X_test)}")
print("清洗前:", X_test.iloc[0][:100])
print("词袋清洗后:", X_test_bow.iloc[0][:100])
print("嵌入清洗后:", X_test_embed.iloc[0][:100])
```

`stratify=y_fpb` 保证训练集和测试集中的类别比例接近原始数据。若不分层抽样，少数 negative 类可能在测试集中数量过少，导致评估不稳定。`clean_text_bow` 删除较多非英文符号，并统一小写。`clean_text_for_embedding` 只去除明显噪声，尽量保留原句语义。

这种清洗策略体现了模型适配思想。不存在一种清洗方式适合所有模型。对于词袋模型，规范化有助于降低稀疏性。对于上下文模型，保留语序和标点通常更重要。

### 5.3.1.5 朴素贝叶斯情绪分类基线

朴素贝叶斯是金融文本分类的低成本基线。它的多项式形式可写为

$$P(c\mid x)\propto P(c)\prod_{j=1}^{d}P(w_j\mid c)^{x_j}$$

其中，`c` 是情绪类别，`w_j` 是词项，`x_j` 是该词项在文本中的频率或权重。模型假设在给定类别后各词项条件独立。自然语言并不真正满足这个假设，但该模型训练速度快、可解释性强，适合建立初始对照。

```python
def build_nb_pipeline(
    vectorizer_type: str = "tfidf",
    alpha: float = 0.1,
    max_features: int = 10000,
) -> Pipeline:
    """构建朴素贝叶斯文本分类管道。"""
    if vectorizer_type == "tfidf":
        vectorizer = TfidfVectorizer(
            max_features=max_features,
            stop_words="english",
        )
    else:
        vectorizer = CountVectorizer(
            max_features=max_features,
            stop_words="english",
        )

    return Pipeline([
        ("vectorizer", vectorizer),
        ("nb", MultinomialNB(alpha=alpha)),
    ])


nb_model = build_nb_pipeline(vectorizer_type="tfidf", alpha=0.1)
t0 = perf_counter()
nb_model.fit(X_train_bow, y_train)
nb_fit_time = perf_counter() - t0

nb_preds = nb_model.predict(X_test_bow)
nb_probs = nb_model.predict_proba(X_test_bow)

print(f"朴素贝叶斯训练耗时: {nb_fit_time:.4f}s")
print(classification_report(y_test, nb_preds, digits=4))
```

`TfidfVectorizer` 把文本转化为 TF-IDF 特征。TF-IDF 同时考虑词频和逆文档频率，使常见但区分度低的词权重下降。`max_features=10000` 限制词表规模，避免特征过多。`MultinomialNB(alpha=0.1)` 使用平滑项，避免某些词在某个类别中未出现而使概率为零。

朴素贝叶斯的优势是快。它可以在普通 CPU 上迅速完成训练和推理。它的局限是难以理解否定、转折和金融语境。例如，`profit rose but missed expectations` 同时含有增长和不及预期，单纯词项统计可能难以判断其负面含义。

### 5.3.1.6 FinBERT 金融情绪推理

FinBERT 是面向金融文本的 BERT 类模型。它能够根据上下文理解词语含义，并直接输出 positive、negative 和 neutral 三类概率。本项目使用预训练模型推理，不在本项目数据上微调。这样做可以降低训练成本，也能展示领域预训练模型的迁移能力。

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification

FINBERT_MODEL_NAME = "ProsusAI/finbert"
# Ref: https://huggingface.co/ProsusAI/finbert

try:
    print(f"加载 FinBERT 到 {DEVICE}")
    finbert_tokenizer = AutoTokenizer.from_pretrained(FINBERT_MODEL_NAME)
    finbert_model = AutoModelForSequenceClassification.from_pretrained(
        FINBERT_MODEL_NAME
    ).to(DEVICE)
    finbert_model.eval()
    HAS_FINBERT = True
except Exception as exc:
    HAS_FINBERT = False
    finbert_tokenizer = None
    finbert_model = None
    print(f"FinBERT 加载失败: {exc}")

FINBERT_LABELS = ["positive", "negative", "neutral"]
```

`AutoTokenizer` 把文本分词为模型可接受的 token，并生成 attention mask。`AutoModelForSequenceClassification` 加载带分类头的模型。`.to(DEVICE)` 把模型移动到 GPU 或 CPU。`eval()` 切换到推理模式，关闭 dropout 等训练行为。

```python
def finbert_predict_batch(
    texts: list[str],
    tokenizer=finbert_tokenizer,
    model=finbert_model,
    device=DEVICE,
    batch_size: int = 64,
) -> tuple[list[str], list[dict[str, float]]]:
    """批量运行 FinBERT 推理。"""
    if tokenizer is None or model is None:
        raise RuntimeError("FinBERT 模型尚未加载。")

    all_labels = []
    all_scores = []

    for start in range(0, len(texts), batch_size):
        batch_texts = texts[start : start + batch_size]
        inputs = tokenizer(
            batch_texts,
            padding=True,
            truncation=True,
            max_length=512,
            return_tensors="pt",
        ).to(device)

        with torch.no_grad():
            outputs = model(**inputs)
            probs = torch.softmax(outputs.logits, dim=-1)

        for i in range(len(batch_texts)):
            prob_dict = {
                FINBERT_LABELS[j]: float(probs[i, j])
                for j in range(len(FINBERT_LABELS))
            }
            pred_idx = int(probs[i].argmax())
            all_labels.append(FINBERT_LABELS[pred_idx])
            all_scores.append(prob_dict)

    return all_labels, all_scores
```

函数按批次处理文本。`padding=True` 把同一批文本补齐到相同长度。`truncation=True` 表示超过 512 token 的文本被截断。`torch.no_grad()` 关闭梯度计算，减少显存和时间消耗。`softmax` 把模型输出的 logits 转化为概率。概率字典不仅用于分类，也用于计算连续情绪分数。

```python
if HAS_FINBERT:

        X_test_embed.tolist(),
        batch_size=64,
```

FinBERT 通常比朴素贝叶斯更能识别金融表达，尤其对 negative 类召回率更好。它的代价是推理速度较慢，模型文件较大，对 GPU 或高性能 CPU 更敏感。生产系统中是否使用 FinBERT，需要结合精度、延迟、吞吐量和部署成本判断。

### 5.3.1.7 大语言模型文本嵌入与本地缓存

大语言模型嵌入把文本映射为连续向量。语义接近的文本在向量空间中距离更近。与直接让大模型输出情绪标签不同，嵌入方法把语义表征和业务分类分开。大模型负责生成特征，逻辑回归负责学习当前标签体系下的分类边界。

本项目使用阿里云百炼的 OpenAI 兼容接口。API Key 必须存放在环境变量或 `.env` 文件中，不能写入代码仓库。

在批量嵌入前，先构建缓存机制。嵌入 API 有费用和延迟，同一文本不应反复请求。缓存使用文本哈希作为键，并把向量保存为 `.npz` 文件。

```python
CACHE_DIR = DATA_DIR / "embedding_cache"
CACHE_DIR.mkdir(parents=True, exist_ok=True)


def _text_hash(text: str) -> str:
    """使用 SHA256 生成文本指纹。"""
    return hashlib.sha256(text.encode("utf-8")).hexdigest()


def _load_cache(cache_file: Path) -> dict[str, np.ndarray]:
    """从 npz 文件读取嵌入缓存。"""
    if not cache_file.exists():
        return {}
    data = np.load(cache_file, allow_pickle=False)
    return {key: data[key] for key in data.files}


def _save_cache(cache: dict[str, np.ndarray], cache_file: Path) -> None:
    """把嵌入缓存写入本地文件。"""
    np.savez(cache_file, **cache)
```

`_text_hash` 保证相同文本映射到相同缓存键。`np.savez` 简单易用，适合单机教学场景。它不适合多个进程同时写入。生产环境可以用 SQLite、对象存储、Redis 或向量数据库管理缓存。

```python
@retry(stop=stop_after_attempt(5), wait=wait_exponential(min=1, max=30))
def _embed_batch_api(texts: list[str]) -> list[np.ndarray]:
    """调用一次嵌入 API。"""
    # Ref: https://www.alibabacloud.com/help/zh/model-studio/embedding-interfaces-compatible-with-openai
    if embedding_client is None:
        raise RuntimeError("嵌入 API 客户端尚未初始化。")

    resp = embedding_client.embeddings.create(
        model=EMBED_MODEL_NAME,
        input=texts,
        dimensions=EMBED_DIM,
        encoding_format="float",
    )
    return [np.array(item.embedding, dtype=np.float32) for item in resp.data]
```

`tenacity.retry` 提供失败重试。API 调用可能遇到网络抖动、限流或临时服务错误。指数退避可以避免短时间内连续重试造成更严重限流。

```python
def embed_texts(
    texts: list[str],
    cache_name: str,
    max_chars: int = 8000,
    verbose: bool = True,
) -> np.ndarray:
    """批量生成文本嵌入并使用本地缓存。"""
    if not HAS_EMBEDDING_API:
        raise RuntimeError("未配置嵌入 API，无法生成文本向量。")

    cache_file = CACHE_DIR / f"{cache_name}.npz"
    cache = _load_cache(cache_file)

    processed = [(text or " ")[:max_chars] for text in texts]
    keys = [_text_hash(text) for text in processed]
    todo_idx = [i for i, key in enumerate(keys) if key not in cache]

    if verbose:
        print(
            f"[{cache_name}] 总计 {len(texts)} 条，"
            f"命中缓存 {len(texts) - len(todo_idx)} 条，"
            f"待请求 {len(todo_idx)} 条"
        )

    for start in range(0, len(todo_idx), EMBED_BATCH_SIZE):
        batch_idx = todo_idx[start : start + EMBED_BATCH_SIZE]
        batch_texts = [processed[i] for i in batch_idx]
        vecs = _embed_batch_api(batch_texts)
        for i, vec in zip(batch_idx, vecs):
            cache[keys[i]] = vec
        if verbose and (start // EMBED_BATCH_SIZE) % 20 == 0:
            print(f"进度 {start + len(batch_idx)}/{len(todo_idx)}", end="\r")

    if todo_idx:
        _save_cache(cache, cache_file)
        if verbose:
            print(f"\n已写入缓存: {cache_file}")

    return np.stack([cache[key] for key in keys], axis=0)
```

`max_chars=8000` 控制文本长度，降低 token 成本并避免超过接口限制。对新闻标题而言通常不会截断；对正文而言可能截断关键信息。若项目使用长新闻正文，应考虑标题、摘要、正文分段或层次化汇总。

### 5.3.1.10 新闻情绪指标构建

单条新闻的情绪分数可定义为

$$s_i=P(positive\mid x_i)-P(negative\mid x_i)$$

若 `s_i` 接近 1，文本偏正面。若接近 -1，文本偏负面。若接近 0，文本可能中性，也可能正负信息抵消。日度平均情绪可定义为

$$S_t=\frac{1}{n_t}\sum_{i=1}^{n_t}s_{t,i}$$

其中，`n_t` 是第 `t` 日新闻数量。新闻数量本身也应保留，因为大量新闻集中出现往往表示市场关注度上升。

```python
def score_from_probabilities(
    probs: np.ndarray,
    class_order: list[str],
) -> np.ndarray:
    """由三分类概率计算情绪分数。"""
    pos_idx = class_order.index("positive")
    neg_idx = class_order.index("negative")
    return probs[:, pos_idx] - probs[:, neg_idx]


def score_news_with_nb(texts: list[str]) -> np.ndarray:
    """使用朴素贝叶斯模型给新闻打分。"""
    clean = [clean_text_bow(text) for text in texts]
    probs = nb_model.predict_proba(clean)
    class_order = list(nb_model.classes_)
    return score_from_probabilities(probs, class_order)


def score_news_with_embedding(texts: list[str], cache_name: str) -> np.ndarray:
    """使用嵌入分类器给新闻打分。"""
    if clf_pipeline is None:
        raise RuntimeError("Embedding 分类器尚未训练。")
    clean = [clean_text_for_embedding(text) for text in texts]
    news_emb = embed_texts(clean, cache_name=cache_name)
    probs = clf_pipeline.predict_proba(news_emb)
    class_order = list(clf_pipeline.classes_)
    return score_from_probabilities(probs, class_order)


def score_news_with_finbert(texts: list[str]) -> np.ndarray:
    """使用 FinBERT 给新闻打分。"""
    clean = [clean_text_for_embedding(text) for text in texts]
    _, scores = finbert_predict_batch(clean)
    return np.array([
        item["positive"] - item["negative"]
        for item in scores
    ], dtype=float)
```

这三个函数统一输出连续情绪分数。统一接口的好处是，后续事件分析和决策模拟不必关心底层模型来自朴素贝叶斯、FinBERT 还是嵌入分类器。系统架构上，这属于模型服务与业务逻辑解耦。

```python
def build_daily_sentiment(
    news: pd.DataFrame,
    model_name: str = SENTIMENT_MODEL,
    max_news_for_embedding: int | None = 1000,
) -> pd.DataFrame:
    """把新闻数据转化为日度情绪指标。"""
    if news.empty:
        raise ValueError("新闻数据为空，无法构建日度情绪。")

    if "Article_title" in news.columns:
        text_col = "Article_title"
    elif "Article" in news.columns:
        text_col = "Article"
    else:
        raise KeyError("新闻数据缺少 Article_title 或 Article 列。")

    if "Date" not in news.columns:
        raise KeyError("新闻数据缺少 Date 列。")

    work = news.dropna(subset=[text_col, "Date"]).copy()
    if model_name == "embedding" and max_news_for_embedding is not None:
        work = work.head(max_news_for_embedding).copy()

    texts = work[text_col].astype(str).tolist()
    if model_name == "finbert":
        scores = score_news_with_finbert(texts)
    elif model_name == "embedding":
        scores = score_news_with_embedding(
            texts,
            cache_name=f"fnspid_{TARGET_TICKER.lower()}_{text_col.lower()}",
        )
    else:
        scores = score_news_with_nb(texts)

    work["sentiment_score"] = scores
    work["date"] = pd.to_datetime(work["Date"]).dt.date

    daily = (
        work.groupby("date")["sentiment_score"]
        .agg(avg_sentiment="mean", news_count="count", sentiment_std="std")
        .reset_index()
    )
    daily["date"] = pd.to_datetime(daily["date"])
    daily["sentiment_std"] = daily["sentiment_std"].fillna(0.0)
    return daily.sort_values("date").reset_index(drop=True)


if HAS_FNSPID_NEWS:
    daily_sentiment = build_daily_sentiment(news_df, SENTIMENT_MODEL)
    print(daily_sentiment.head())
    print(f"日度情绪天数: {len(daily_sentiment)}")
```

代码优先使用新闻标题。标题短、噪声少、计算成本低，适合教学演示。使用正文可以包含更多信息，但可能遇到模型输入长度限制和更高推理成本。对嵌入模型，代码默认最多取 1000 条新闻，是为了控制 API 成本。若课堂已准备缓存或预算充足，可以取消这个限制。

`sentiment_std` 表示同一天新闻情绪分歧程度。平均情绪相同的两天，可能有不同含义。若一天中所有新闻都轻微正面，分歧较低。若一天同时出现强正面和强负面新闻，平均值可能接近 0，但分歧很高。

### 5.3.1.11 情绪与收益率合并

价格数据需要先计算日收益率。普通收益率定义为

$$R_t=\frac{P_t-P_{t-1}}{P_{t-1}}$$

其中，`P_t` 是第 `t` 日收盘价。若使用对数收益率，则定义为

$$r_t=\log(P_t)-\log(P_{t-1})$$

本项目使用普通收益率，便于解释策略收益和异常收益。短期日频分析中，两者差异通常较小。

```python
def prepare_price_returns(price: pd.DataFrame) -> pd.DataFrame:
    """整理价格数据并计算收益率。"""
    price = price.copy()
    price["Date"] = pd.to_datetime(price["Date"])
    price = price.sort_values("Date").reset_index(drop=True)
    close_col = "Close" if "Close" in price.columns else "Adj Close"
    price["return"] = price[close_col].pct_change()
    price["log_return"] = np.log(price[close_col]).diff()
    return price


price_ret_df = prepare_price_returns(price_df)
print(price_ret_df[["Date", "Close", "return", "log_return"]].tail())
```

收益率第一行为空值，因为没有上一交易日价格。后续分析可保留或删除该空值。若使用复权价格，应明确使用 `Adj Close` 还是 `Close`。本项目主要强调流程，因此使用 `Close`，但在严肃回测中通常应考虑复权价格和现金分红。

```python
def merge_sentiment_and_returns(

```

`inner` 合并只保留同时存在新闻和交易收益率的日期。周末和节假日新闻会被排除。更严格的做法是把非交易日新闻映射到下一交易日，把盘后新闻映射到下一交易日，把盘前新闻映射到当日交易。教学项目先使用自然日期合并，但必须在报告中说明这一限制。

### 5.3.1.12 相关性与事件窗口分析

情绪与收益率的 Pearson 相关系数定义为

$$\rho_{S,R}=\frac{Cov(S_t,R_t)}{\sigma_S\sigma_R}$$

它只能衡量线性相关关系，不能证明因果。新闻可能影响价格，也可能只是报道价格已经发生的变化。市场也可能在新闻发布前已经通过其他渠道吸收信息。

事件窗口分析关注极端情绪日前后若干天的收益变化。若把样本平均收益作为正常收益基准，则异常收益为

$$AR_t=R_t-\bar{R}$$

累计异常收益为

$$CAR(a,b)=\sum_{\tau=a}^{b} AR_{\tau}$$

```python
def event_window_analysis(
    merged_df: pd.DataFrame,
    window: int = 2,
    quantile: float = 0.05,
) -> pd.DataFrame | None:
    """对极端情绪日进行事件窗口分析。"""
    if len(merged_df) < 50:
        print("样本量不足，跳过事件窗口分析。")
        return None

    df = merged_df.sort_values("date").reset_index(drop=True).copy()
    avg_ret = df["return"].mean()
    pos_threshold = df["avg_sentiment"].quantile(1 - quantile)
    neg_threshold = df["avg_sentiment"].quantile(quantile)

    results = []
    for event_type, mask in [
        ("极正面", df["avg_sentiment"] >= pos_threshold),
        ("极负面", df["avg_sentiment"] <= neg_threshold),
    ]:
        event_indices = df[mask].index.tolist()
        for event_idx in event_indices:
            car = 0.0
            for offset in range(-window, window + 1):
                target_idx = event_idx + offset
                if 0 <= target_idx < len(df):
                    ar = df.loc[target_idx, "return"] - avg_ret
                    car += ar
                    results.append({
                        "事件类型": event_type,
                        "事件日": df.loc[event_idx, "date"],
                        "偏移天数": offset,
                        "异常收益率": ar,
                        "累计异常收益率": car,
                    })

    event_df = pd.DataFrame(results)
    summary = (
        event_df.groupby(["事件类型", "偏移天数"])
        .agg(
            平均异常收益率=("异常收益率", "mean"),
            平均累计异常收益率=("累计异常收益率", "mean"),
            事件数=("事件日", "nunique"),
        )
        .reset_index()
    )
    return summary


if HAS_FNSPID_NEWS:
    event_summary = event_window_analysis(merged, window=2, quantile=0.05)
    if event_summary is not None:
        print(event_summary.to_string(index=False))
```

该函数把情绪最高 5% 的交易日定义为极正面事件，把情绪最低 5% 的交易日定义为极负面事件。它计算每个事件日前后两天的异常收益率和累计异常收益率，并汇总为平均曲线。

本方法是简化事件研究。它没有使用市场模型估计正常收益，没有控制行业收益，没有处理事件重叠，也没有处理盘前盘后发布时间。因此它适合教学和初步探索，不足以支持正式投资结论。

### 5.3.1.13 决策模拟框架

决策模拟是为了检验舆情指标是否能形成可执行的规则，而不是为了给出真实投资建议。模拟必须避免前视偏差。若第 `t` 日收盘后才能获得完整新闻和情绪分数，则只能把该信号用于第 `t+1` 日。若新闻在盘中实时获取，才可以讨论日内或当日收盘前决策。

本项目采用保守设计。系统用第 `t` 日情绪生成第 `t+1` 日持仓信号。情绪显著正面且新闻数量不低时持有，情绪显著负面时空仓，其余情况保持上一日状态。策略收益扣除简单交易成本。

```python
def build_decision_table(
    daily_sent: pd.DataFrame,
    price_returns: pd.DataFrame,
    sentiment_window: int = 60,
) -> pd.DataFrame:
    """构造用于决策模拟的交易日级别数据表。"""
    base = price_returns[["Date", "return"]].copy()
    base = base.rename(columns={"Date": "date"})
    base["date"] = pd.to_datetime(base["date"])

    sent = daily_sent.copy()
    sent["date"] = pd.to_datetime(sent["date"])

    decision = pd.merge(base, sent, on="date", how="left")
    decision["has_news"] = decision["avg_sentiment"].notna()
    decision["avg_sentiment"] = decision["avg_sentiment"].fillna(0.0)
    decision["news_count"] = decision["news_count"].fillna(0).astype(int)
    decision["sentiment_std"] = decision["sentiment_std"].fillna(0.0)

    rolling_mean = decision["avg_sentiment"].rolling(
        sentiment_window,
        min_periods=20,
    ).mean().shift(1)
    rolling_std = decision["avg_sentiment"].rolling(
        sentiment_window,
        min_periods=20,
    ).std().shift(1)
    decision["sentiment_z"] = (
        (decision["avg_sentiment"] - rolling_mean)
        / rolling_std.replace(0, np.nan)
    )

    news_ma = decision["news_count"].rolling(
        sentiment_window,
        min_periods=20,
    ).mean().shift(1)
    decision["news_intensity"] = decision["news_count"] / news_ma.replace(0, np.nan)
    return decision


if HAS_FNSPID_NEWS:
    decision_df = build_decision_table(daily_sentiment, price_ret_df)
    print(decision_df.tail())
```

`rolling(...).shift(1)` 是关键。它保证第 `t` 日的标准化基准只使用第 `t` 日之前的信息。如果不使用 `shift(1)`，滚动均值和标准差会包含当日情绪，虽然影响较小，但严格来说仍降低了时点一致性。

无新闻日期的 `avg_sentiment` 被填为 0。这是简化处理，表示没有可用新闻时不提供正负方向。更严谨的做法是区分“中性新闻”和“没有新闻”。本项目保留 `has_news` 字段，便于后续改进。

```python
def simulate_sentiment_strategy(
    decision: pd.DataFrame,
    positive_z: float = 0.5,
    negative_z: float = -1.0,
    min_news_intensity: float = 1.0,
    transaction_cost: float = 0.0005,
) -> pd.DataFrame:
    """模拟基于新闻情绪的多头或空仓策略。"""
    df = decision.sort_values("date").reset_index(drop=True).copy()
    signal = []
    current_position = 1

    for _, row in df.iterrows():
        z = row.get("sentiment_z", np.nan)
        intensity = row.get("news_intensity", np.nan)

        if pd.notna(z) and pd.notna(intensity):
            if z >= positive_z and intensity >= min_news_intensity:
                current_position = 1
            elif z <= negative_z:
                current_position = 0

        signal.append(current_position)

    df["signal_for_next_day"] = signal
    df["position"] = df["signal_for_next_day"].shift(1).fillna(1)
    df["turnover"] = df["position"].diff().abs().fillna(0)
    df["strategy_return"] = (
        df["position"] * df["return"].fillna(0)
        - transaction_cost * df["turnover"]
    )
    df["benchmark_return"] = df["return"].fillna(0)
    df["strategy_nav"] = (1 + df["strategy_return"]).cumprod()
    df["benchmark_nav"] = (1 + df["benchmark_return"]).cumprod()
    return df


if HAS_FNSPID_NEWS:
    sim_df = simulate_sentiment_strategy(decision_df)
    print(sim_df[["date", "position", "strategy_nav", "benchmark_nav"]].tail())
```

`signal_for_next_day` 表示第 `t` 日产生的信号。`position` 是第 `t` 日实际持仓，它来自前一日信号。这样可以避免用当天收盘后才知道的信息交易当天收益。`transaction_cost` 是单边交易成本的简化表达。真实回测还应考虑买卖价差、冲击成本、停牌、滑点、税费和成交量约束。

该策略只允许多头或空仓，不允许做空。这样做更符合一般资产管理教学场景，也降低了对融券、保证金和强平机制的要求。若研究对象是期货或可卖空资产，可以扩展为多空策略，但风险控制要求更高。

```python
def summarize_strategy(sim: pd.DataFrame) -> pd.Series:
    """汇总策略绩效指标。"""
    ret = sim["strategy_return"].dropna()
    bench = sim["benchmark_return"].dropna()
    annual_factor = 252

    total_return = sim["strategy_nav"].iloc[-1] - 1
    benchmark_return = sim["benchmark_nav"].iloc[-1] - 1
    annual_return = (1 + total_return) ** (annual_factor / max(len(ret), 1)) - 1
    annual_vol = ret.std() * np.sqrt(annual_factor)
    sharpe = annual_return / annual_vol if annual_vol > 0 else np.nan
    max_drawdown = (sim["strategy_nav"] / sim["strategy_nav"].cummax() - 1).min()
    benchmark_max_drawdown = (
        sim["benchmark_nav"] / sim["benchmark_nav"].cummax() - 1
    ).min()

    return pd.Series({
        "策略累计收益": total_return,
        "基准累计收益": benchmark_return,
        "策略年化收益": annual_return,
        "策略年化波动率": annual_vol,
        "策略夏普比率": sharpe,
        "策略最大回撤": max_drawdown,
        "基准最大回撤": benchmark_max_drawdown,
        "交易次数": sim["turnover"].sum(),
        "平均持仓": sim["position"].mean(),
    })


if HAS_FNSPID_NEWS:
    perf = summarize_strategy(sim_df)
    print(perf.apply(lambda x: round(float(x), 4)))
```

### 5.3.1.14 为什么不直接用新闻情绪预测价格

直接用新闻情绪回归价格不是本项目选择的主要路线。股票价格通常是非平稳序列，价格水平受长期增长、拆股、分红、宏观环境和市场趋势影响。若直接拟合价格，模型可能只是学到时间趋势，而不是新闻效应。收益率或异常收益率更适合短期事件分析。

简单关键词词典也不是主方法。金融文本中的含义依赖上下文。例如，`costs declined` 往往偏正面，`revenue declined` 往往偏负面。`profit rose but missed expectations` 不能简单地因 `rose` 判断为利好。关键词方法适合可解释的粗筛，但不适合作为唯一模型。

本项目没有让 LLM 直接输出买入或卖出建议。直接建议存在稳定性、可审计性和合规风险。更合理的方式是用模型产生可验证的情绪概率、用规则构建风险提示、用人工流程确认处置方案。

## 5.3.2 Agentic AI 辅助市场风险预测

### 5.3.2.1 Agent 在本项目中的职责

本项目中的 Agentic AI 不是自动交易机器人。它承担四类职责。第一，监控新闻情绪、新闻数量和价格波动。第二，基于滚动基准识别异常舆情。第三，根据可审计规则生成风险等级和触发原因。第四，输出近期事件摘要和审计日志，供人工复核。

这种定位符合金融风险管理的基本要求。模型可以提高信息处理效率，但不能绕过风控政策、合规审查和人工责任。尤其在新闻舆情场景中，模型可能误解语义、新闻可能重复转载、市场可能已经提前反应。如果让 Agent 直接下单，系统风险会被放大。

### 5.3.2.2 风险监控特征

Agent 的输入不是原始文本，而是经过模型和时间聚合后的监控表。核心字段包括平均情绪、情绪 Z 分数、新闻强度、收益率绝对值和是否有新闻。情绪 Z 分数定义为

$$Z_t=\frac{S_t-\mu_{t,w}}{\sigma_{t,w}}$$

其中，`S_t` 是当日平均情绪，`\mu_{t,w}` 和 `\sigma_{t,w}` 是过去窗口内情绪均值和标准差。使用滚动标准化的原因在于，不同股票和不同时期的新闻基准不同。固定阈值难以适应所有场景。

```python
def build_sentiment_monitoring_table(
    decision: pd.DataFrame,
    sentiment_window: int = 20,
    volume_window: int = 20,
) -> pd.DataFrame:
    """构造新闻舆情风险监控表。"""
    monitor = decision.sort_values("date").reset_index(drop=True).copy()

    monitor["sentiment_ma"] = monitor["avg_sentiment"].rolling(
        sentiment_window,
        min_periods=5,
    ).mean().shift(1)
    monitor["sentiment_std_roll"] = monitor["avg_sentiment"].rolling(
        sentiment_window,
        min_periods=5,
    ).std().shift(1)
    monitor["sentiment_z_agent"] = (
        (monitor["avg_sentiment"] - monitor["sentiment_ma"])
        / monitor["sentiment_std_roll"].replace(0, np.nan)
    )

    monitor["news_count_ma"] = monitor["news_count"].rolling(
        volume_window,
        min_periods=5,
    ).mean().shift(1)
    monitor["news_intensity_agent"] = (
        monitor["news_count"] / monitor["news_count_ma"].replace(0, np.nan)
    )
    monitor["abs_return"] = monitor["return"].abs()
    return monitor


if HAS_FNSPID_NEWS:
    monitor_df = build_sentiment_monitoring_table(decision_df)
    print(monitor_df.tail())
```

这里再次使用 `shift(1)`，确保监控基准不包含当日数据。`news_intensity_agent` 衡量当天新闻数量相对近期平均水平是否放大。若负面情绪显著偏低且新闻数量放大，风险等级应高于单条负面新闻。

### 5.3.2.3 规则型风险 Agent

规则型 Agent 的优点是透明、可解释、可复现。每个风险等级都可以追踪到具体规则。其局限是规则需要人工设计，可能无法覆盖新型事件。因此它适合作为金融舆情系统的第一版，也适合作为复杂 Agent 的安全底座。

```python
class NewsSentimentRiskAgent:
```

构造函数保存监控表和策略参数。`z_warning` 是一般情绪异常阈值，`z_critical` 是严重异常阈值。`intensity_warning` 表示新闻数量相对均值放大到多少倍时触发关注。`return_warning` 表示当日绝对收益率超过多少时认为价格波动同步放大。

```python
    def score_latest_day(self) -> dict:
        """对最新可用日期生成风险评分。"""
        required = ["avg_sentiment", "sentiment_z_agent"]
        clean = self.monitor_table.dropna(subset=required)
        if clean.empty:
            return {
                "风险等级": "无法判断",
                "原因": "可用情绪历史不足",
                "建议": "继续收集新闻并检查数据管道",
            }

        latest = clean.iloc[-1]
        reasons = []
        risk_score = 0

        z_value = latest["sentiment_z_agent"]
        intensity = latest.get("news_intensity_agent", np.nan)
        abs_return = latest.get("abs_return", np.nan)

        if z_value <= -self.z_critical:
            risk_score += 3
            reasons.append("负面情绪显著低于近期基准")
        elif z_value <= -self.z_warning:
            risk_score += 2
            reasons.append("负面情绪低于近期基准")
        elif z_value >= self.z_critical:
            risk_score += 1
            reasons.append("正面情绪显著高于近期基准，需关注过度乐观")

        if pd.notna(intensity) and intensity >= self.intensity_warning:
            risk_score += 1
            reasons.append("新闻数量明显放大")

        if pd.notna(abs_return) and abs_return >= self.return_warning:
            risk_score += 1
            reasons.append("价格波动同步放大")

        if risk_score >= 4:
            level = "高风险"
        elif risk_score >= 2:
            level = "中风险"
        elif risk_score >= 1:
            level = "低风险关注"
        else:
            level = "正常"
            reasons.append("未触发主要舆情风险规则")

        return {
            "日期": latest["date"],
            "目标股票": self.target_ticker,
            "风险等级": level,
            "风险分数": int(risk_score),
            "平均情绪": round(float(latest["avg_sentiment"]), 4),
            "情绪Z分数": round(float(z_value), 4),
            "新闻强度": round(float(intensity), 4) if pd.notna(intensity) else None,
            "当日收益率": round(float(latest["return"]), 4) if pd.notna(latest["return"]) else None,
            "原因": "；".join(reasons),
            "建议": "复核新闻标题、来源、发布时间、是否重复转载以及是否存在重大公司事件。",
        }
```

评分逻辑把显著负面情绪赋予更高权重，因为风险管理更关注下行冲击。显著正面情绪只加少量分数，是因为它可能代表利好，也可能代表过度乐观或拥挤预期。新闻数量和价格波动是辅助信号。

```python
    def summarize_recent_events(self, lookback_days: int = 10) -> pd.DataFrame:
        """输出最近若干交易日的舆情监控状态。"""
        recent = self.monitor_table.dropna(subset=["sentiment_z_agent"]).tail(
            lookback_days
        ).copy()
        if recent.empty:
            return pd.DataFrame()

        recent["是否负面异常"] = recent["sentiment_z_agent"] <= -self.z_warning
        recent["是否新闻放量"] = recent["news_intensity_agent"] >= self.intensity_warning
        recent["是否价格异动"] = recent["abs_return"] >= self.return_warning

        cols = [
            "date",
            "avg_sentiment",
            "sentiment_z_agent",
            "news_count",
            "news_intensity_agent",
            "return",
            "是否负面异常",
            "是否新闻放量",
            "是否价格异动",
        ]
        return recent[cols].reset_index(drop=True)
```

近期事件摘要帮助判断风险是否持续。单日异常可能是噪声，连续多日负面情绪、新闻放量和价格波动更值得关注。

```python
    def build_audit_log(self) -> pd.DataFrame:
        """生成风险判断审计日志。"""
        decision = self.score_latest_day()
        audit = {
            "策略版本": self.policy_version,
            "生成时间": pd.Timestamp.now(),
            "目标股票": self.target_ticker,
            "新闻打分模型": SENTIMENT_MODEL,
            "是否允许自动交易": False,
            "风险等级": decision.get("风险等级"),
            "风险分数": decision.get("风险分数"),
            "触发原因": decision.get("原因"),
            "人工复核要求": True,
        }
        return pd.DataFrame([audit])
```

审计日志记录策略版本、生成时间、目标股票、模型来源、风险等级和触发原因。`是否允许自动交易` 固定为 False，明确本项目 Agent 的职责是辅助预警，而不是执行交易。正式系统还应记录数据版本、模型版本、特征生成代码版本、阈值审批记录、人工处置结果和事后复盘结论。

### 5.3.2.4 Agent 报告生成

风险 Agent 的输出可以进一步组织成自然语言报告。报告必须基于结构化指标生成，不能让大语言模型自由编造事实。下面函数使用模板方式生成报告，保证内容可追踪。

```python
def build_risk_report(
    latest_risk: dict,
    recent_events: pd.DataFrame,
    audit_log: pd.DataFrame,
) -> str:
    """基于结构化结果生成风险报告文本。"""
    lines = []
    lines.append(f"目标股票：{latest_risk.get('目标股票')}")
    lines.append(f"风险等级：{latest_risk.get('风险等级')}")
    lines.append(f"风险分数：{latest_risk.get('风险分数')}")
    lines.append(f"平均情绪：{latest_risk.get('平均情绪')}")
    lines.append(f"情绪 Z 分数：{latest_risk.get('情绪Z分数')}")
    lines.append(f"新闻强度：{latest_risk.get('新闻强度')}")
    lines.append(f"当日收益率：{latest_risk.get('当日收益率')}")
    lines.append(f"触发原因：{latest_risk.get('原因')}")
    lines.append(f"处置建议：{latest_risk.get('建议')}")

    if not recent_events.empty:
        abnormal_days = int(recent_events["是否负面异常"].sum())
        high_volume_days = int(recent_events["是否新闻放量"].sum())
        volatile_days = int(recent_events["是否价格异动"].sum())
        lines.append(
            f"最近观察窗口内，负面异常日期 {abnormal_days} 天，"
            f"新闻放量日期 {high_volume_days} 天，价格异动日期 {volatile_days} 天。"
        )

    if not audit_log.empty:
        lines.append(f"策略版本：{audit_log.loc[0, '策略版本']}")
        lines.append("系统结论不构成自动交易指令，必须经过人工复核。")

    return "\n".join(lines)


if HAS_FNSPID_NEWS:
    report_text = build_risk_report(latest_risk, recent_events, audit_log)
    print(report_text)
```

模板报告的可控性强。若引入 LLM 生成更流畅的报告，也应把 LLM 限制在改写和摘要角色，不允许它新增事实、修改风险等级或生成交易指令。推荐做法是将结构化指标作为唯一事实来源，并要求 LLM 只根据这些指标改写为报告语言。

### 5.3.2.5 模型治理与风险控制

金融舆情 Agent 的风险主要来自四个方面。第一，数据风险。新闻可能重复转载，日期可能错位，标题可能夸张，来源可信度不同。第二，模型风险。文本模型可能误判否定、转折、讽刺和新型金融事件。第三，回测风险。若时间对齐错误或阈值过度调参，策略表现会被高估。第四，合规风险。自动化系统若输出交易建议，可能超出辅助分析范围。

本项目采用三项控制。其一，所有模型输出都转化为可解释指标，例如情绪分数、新闻数量、情绪 Z 分数和收益率。其二，Agent 使用规则生成风险等级，并记录触发原因。其三，系统明确不允许自动交易，所有高风险提示需要人工复核。

这些控制并不会消除模型错误，但能降低错误被无监督放大的概率。学生在项目报告中应说明数据来源、模型选择、阈值依据、回测假设、交易成本、限制条件和人工复核流程。

### 5.3.2.6 项目交付建议

本项目的交付物应包括四部分。第一部分是数据说明，说明 Financial PhraseBank、FNSPID 新闻和 Yahoo Finance 行情的用途、字段和时间范围。第二部分是模型比较，报告朴素贝叶斯、FinBERT 和 LLM Embedding 加逻辑回归的 Accuracy、macro F1、混淆矩阵和运行成本。第三部分是舆情决策分析，展示日度情绪、收益率相关性、事件窗口结果和决策模拟净值曲线。第四部分是 Agent 风险预警，展示最新风险等级、近期异常摘要、审计日志和人工复核建议。

项目报告不应只展示最好结果。若某模型表现差，应解释原因。若策略没有跑赢基准，也不代表项目失败，因为金融数据本身噪声很大。更重要的是，报告是否清楚说明了建模流程、时间一致性、误差来源和风险控制。

## 本节方法选择的综合讨论

朴素贝叶斯适合作为基线。它对算力要求低，训练速度快，便于快速建立可运行系统。它的不足是语义理解弱，难以处理复杂金融表达。

FinBERT 适合高精度金融情绪识别。它利用金融领域语料训练，通常能更好识别专业文本。它的不足是推理成本高，长文本需要截断，且模型仍可能受到训练语料和标签体系限制。

LLM Embedding 加逻辑回归适合工程化系统。它把大模型语义能力封装为可缓存向量，再由轻量模型完成业务分类。它的不足是依赖外部 API，存在费用、延迟、隐私和版本漂移问题。

事件窗口分析适合观察新闻冲击，但不能证明因果。决策模拟适合理解决策规则与回测假设，但不能直接转化为投资建议。Agentic AI 适合做监控、预警、摘要和审计，不适合在缺少治理的情况下自动交易。

## 拓展方向

1. 将新闻从自然日期映射到交易日期，并区分盘前、盘中和盘后新闻。
2. 引入新闻来源权重，对公告、主流媒体、转载新闻和分析师评论使用不同权重。
3. 对重复新闻进行文本相似度去重，降低同一事件被多次计数的影响。
4. 使用市场模型或行业指数计算异常收益，而不是只用样本平均收益。
5. 在多只股票上构建面板数据，检验情绪指标是否具有跨股票稳定性。
6. 用 Walk-Forward Validation 选择阈值，避免在全样本上调参造成前视偏差。
7. 对 FinBERT 进行小规模微调，比较微调前后的负面类召回率。
8. 将新闻嵌入存入向量数据库，用于相似历史事件检索和 RAG 风险报告。
9. 在 Agent 中加入人工反馈，把误报和漏报作为后续模型更新数据。
10. 建立模型监控看板，持续观察情绪分布、新闻数量、模型置信度和策略暴露。
11. 引入合规审计模块，记录数据版本、模型版本、阈值版本、触发规则和人工处置。
12. 设计压力测试，观察极端负面新闻、数据缺失和 API 失败时系统如何降级运行。
