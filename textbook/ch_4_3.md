# 4.3 欺诈检测与异常行为识别案例

金融欺诈检测是风险管理中最典型的低频高损失问题。信用卡盗刷、账户接管、虚假交易、羊毛党套利和洗钱交易都具有相似特征，即正常行为数量巨大，异常行为数量极少，且异常模式会随着规则、模型和拦截策略不断变化。一个反欺诈模型的价值不在于获得很高的总体准确率，而在于尽可能早地发现少数高风险交易，同时把误报控制在人工复核和客户体验可以接受的范围内。

本章基于 Kaggle Credit Card Fraud Detection 数据集展开。该数据集包含 284807 笔信用卡交易，其中 492 笔为欺诈交易，欺诈比例约为 0.173%。数据中的 V1 到 V28 是经过 PCA 脱敏后的主成分特征，Time 和 Amount 保留了原始含义，Class 是目标变量，其中 0 表示正常交易，1 表示欺诈交易。由于欺诈样本极少，如果模型把所有交易都预测为正常，也能得到约 99.83% 的 Accuracy。因此，本章把 Recall、Precision、F1、PR-AUC、混淆矩阵和阈值分析作为核心评估工具。

本章围绕三个问题展开。第一，异常检测算法如何在标签稀缺或欺诈样本极少的场景中识别异常交易。第二，极端不平衡数据如何通过 SMOTE、随机欠采样、类别权重和正类权重进行处理。第三，模型输出的欺诈概率如何转化为风险识别、阈值策略和智能告警。

反欺诈建模应遵循三个原则。第一，测试集必须保持真实业务分布，不能对测试集进行过采样或欠采样。第二，任何标准化、采样和阈值选择都不能使用未来测试信息。第三，模型不应只追求单一指标，而要结合漏报成本、误报成本、人工复核能力、交易金额和客户体验共同确定策略。

## 4.3.1 异常检测算法应用

异常检测的目标是识别与大多数样本显著不同的行为。在信用卡交易中，异常可能表现为金额异常、时间异常、交易模式异常，也可能表现为多个脱敏主成分组合上的偏离。与有监督分类不同，无监督异常检测可以不使用标签，因此适合冷启动阶段、新型欺诈样本不足阶段和标签确认延迟较长的场景。它的局限也很明显，因为它并不知道哪些异常是真正欺诈，哪些异常只是正常客户的少见行为。

### 4.3.1.2 数据集加载与目标变量定义

目标变量 `Class` 的正类是欺诈交易。这个定义决定了后续 Precision 和 Recall 的业务解释。Precision 表示被模型判为欺诈的交易中有多少是真的欺诈。Recall 表示所有真实欺诈交易中有多少被模型识别出来。在反欺诈业务中，漏报欺诈通常对应直接经济损失，因此 Recall 是非常重要的指标。

### 4.3.1.4 训练集内标准化与数据切分

V1 到 V28 已经过 PCA 变换，尺度相对接近。Time 和 Amount 的尺度与其他特征差异较大，尤其 SVM、SMOTE 和基于距离的分析会受到尺度影响，因此需要标准化。严格的做法是先划分训练集和测试集，再只用训练集学习标准化参数。若先在全量数据上标准化再切分，测试集均值和标准差会提前进入训练流程，形成轻微数据泄漏。

```python
feature_columns = [c for c in df.columns if c != "Class"]
X_raw = df[feature_columns].copy()
y = df["Class"].astype(int)

X_train_raw, X_test_raw, y_train, y_test = train_test_split(
    X_raw,
    y,
    test_size=0.2,
    random_state=42,
    stratify=y,
)

print(f"训练集大小：{X_train_raw.shape[0]}，欺诈比例：{y_train.mean():.5f}")
print(f"测试集大小：{X_test_raw.shape[0]}，欺诈比例：{y_test.mean():.5f}")
```

`feature_columns` 排除目标变量。`X_raw` 保存原始特征，`y` 保存目标变量。`train_test_split` 使用 80% 样本训练，20% 样本测试。`stratify=y` 使训练集和测试集保持相近欺诈比例。对于正类只有 0.173% 的数据，分层切分非常重要，否则测试集中正类数量可能过少，评估结果会不稳定。

```python
scaler = StandardScaler()

X_train = X_train_raw.copy()
X_test = X_test_raw.copy()

scaled_columns = ["Time", "Amount"]
scaled_feature_names = ["Time_scaled", "Amount_scaled"]

X_train[scaled_feature_names] = scaler.fit_transform(X_train_raw[scaled_columns])
X_test[scaled_feature_names] = scaler.transform(X_test_raw[scaled_columns])

X_train = X_train.drop(columns=scaled_columns)
X_test = X_test.drop(columns=scaled_columns)

ordered_columns = [c for c in X_train.columns if c.startswith("V")] + scaled_feature_names
X_train = X_train[ordered_columns]
X_test = X_test[ordered_columns]

print(f"特征矩阵维度：{X_train.shape}")
print(f"特征列表：{X_train.columns.tolist()}")
```

`StandardScaler` 把变量转换为近似零均值和单位标准差。`fit_transform` 只用于训练集，它学习训练集中的 Time 和 Amount 均值及标准差，并完成转换。`transform` 用训练集学到的参数转换测试集。`drop(columns=scaled_columns)` 删除原始 Time 和 Amount，避免同一信息重复出现。`ordered_columns` 让 V1 到 V28 排在前面，标准化后的时间和金额排在后面。这样得到的训练集和测试集列顺序完全一致。

### 4.3.1.5 Isolation Forest 异常检测基线

Isolation Forest 是一种无监督异常检测算法。它的基本思想是，异常点通常更少见、更孤立，因此在随机切分树中更容易被少量分裂隔离出来。模型反复随机选择特征和分裂点，把样本空间切成更小区域。若某个样本在许多树中平均路径长度较短，说明它更容易被孤立，可以被视为异常。

Isolation Forest 的优势是无需标签，可以用于欺诈标签缺失、标签延迟或新业务冷启动场景。它的局限是无法直接学习“欺诈”这个标签的业务含义。异常交易不一定是欺诈，正常客户的大额旅行消费、跨境消费或突发医疗支出也可能是异常。因此，在有可靠标签时，有监督模型通常会比纯异常检测模型更准确。

```python
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.ensemble.IsolationForest.html
iso_clf = IsolationForest(
    n_estimators=200,
    contamination=0.00173,
    random_state=42,
    n_jobs=-1,
)
```

`n_estimators=200` 表示构建 200 棵孤立树。树越多，异常分数越稳定，但训练时间也更长。`contamination=0.00173` 告诉模型预期异常比例约为 0.173%。如果真实欺诈比例未知，可以先根据历史告警率、人工复核能力或业务经验设置初值，再通过验证集评估和人工反馈调整。`random_state=42` 保证结果可复现。`n_jobs=-1` 使用所有可用 CPU 核心。

```python
iso_clf.fit(X_train)

iso_pred_raw = iso_clf.predict(X_test)
y_pred_iso = (iso_pred_raw == -1).astype(int)

iso_scores = -iso_clf.decision_function(X_test)

print("=== Isolation Forest 模型评估 ===")
print(classification_report(y_test, y_pred_iso, target_names=["正常", "欺诈"]))
print(f"ROC-AUC: {roc_auc_score(y_test, iso_scores):.4f}")
print(f"PR-AUC:  {average_precision_score(y_test, iso_scores):.4f}")
```

`fit(X_train)` 只使用特征，不使用 `y_train`。`predict` 的原始输出为 1 和 -1，其中 1 表示正常，-1 表示异常。`(iso_pred_raw == -1).astype(int)` 把异常转换为欺诈正类 1，把正常转换为 0。`decision_function` 的值越小越异常，因此使用负值 `iso_scores` 作为欺诈分数，使分数越大表示越可疑。最后输出分类报告、ROC-AUC 和 PR-AUC。

## 4.3.2 数据不平衡问题处理

欺诈检测最困难的地方不是二分类本身，而是极端不平衡。在训练集中，欺诈样本非常少，模型很容易偏向多数类。处理不平衡数据的常见方法包括随机欠采样、SMOTE 过采样、类别权重调整和代价敏感学习。它们不是互相排斥的方案，而是根据数据规模、模型类型、计算成本和业务目标进行选择。

随机欠采样从正常交易中抽取一部分样本，使正负样本数量更接近。它简单快速，可以显著降低训练时间，但会丢失大量正常交易信息。SMOTE 在少数类样本之间插值生成合成欺诈样本，其基本形式为

$$x_{new}=x_i+\lambda(x_{nn}-x_i),\quad \lambda\in[0,1].$$

其中，`x_i` 是一个少数类样本，`x_nn` 是它的少数类近邻，`λ` 是随机插值系数。SMOTE 的优点是不丢弃多数类信息，并增加少数类样本多样性。它的局限是可能在噪声样本附近生成不合理的合成样本，也可能模糊正常和欺诈边界。类别权重不改变数据分布，而是在损失函数中给欺诈样本更高权重，适合随机森林、SVM 和 XGBoost 等模型。

### 4.3.2.1 SMOTE 过采样

过采样只能应用于训练集，不能应用于测试集。测试集必须保持原始欺诈比例，这样评估结果才接近真实交易场景。若先对全量数据做 SMOTE 再划分训练集和测试集，合成样本可能与测试样本共享近邻信息，导致严重数据泄漏和过于乐观的评估结果。

```python
# Ref: https://imbalanced-learn.org/stable/references/generated/imblearn.over_sampling.SMOTE.html
smote = SMOTE(random_state=42)
X_train_smote, y_train_smote = smote.fit_resample(X_train, y_train)

print(f"SMOTE 前训练集大小：{X_train.shape[0]}，欺诈数量：{y_train.sum()}")
print(f"SMOTE 后训练集大小：{X_train_smote.shape[0]}，欺诈数量：{y_train_smote.sum()}")
print(f"SMOTE 后类别分布：\n{pd.Series(y_train_smote).value_counts()}")
```

`SMOTE(random_state=42)` 创建可复现的过采样器。`fit_resample` 只在训练集上学习少数类近邻并生成合成样本。`X_train_smote` 是过采样后的训练特征，`y_train_smote` 是对应标签。打印语句用于比较过采样前后的样本数和欺诈数量。SMOTE 后的训练集通常会接近平衡，但测试集仍保持原始极端不平衡分布。

### 4.3.2.2 随机欠采样

随机欠采样从多数类正常交易中随机抽取样本，使多数类数量接近少数类数量。它能让训练速度大幅提升，尤其适合快速原型实验。它的主要问题是会丢失大量正常交易模式。对于欺诈检测，正常交易内部本身也非常复杂，删除太多正常样本会使模型误报上升。

```python
# Ref: https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.RandomUnderSampler.html
rus = RandomUnderSampler(random_state=42)
X_train_under, y_train_under = rus.fit_resample(X_train, y_train)

print(f"欠采样后训练集大小：{X_train_under.shape[0]}，欺诈数量：{y_train_under.sum()}")
print(f"欠采样后类别分布：\n{pd.Series(y_train_under).value_counts()}")
```

`RandomUnderSampler` 创建随机欠采样器。`fit_resample` 从训练集多数类中抽样，并保留全部少数类样本。`X_train_under` 和 `y_train_under` 可以用于训练快速模型。由于欠采样后的数据量很小，模型训练很快，但可能无法充分学习正常交易边界。本章后续有监督模型以 SMOTE 和类别权重为主，欠采样作为对比方法保留。

### 4.3.2.3 线性 SVM 与概率校准

SVM 的核心思想是在特征空间中寻找最大间隔分类超平面。对于线性 SVM，模型寻找参数 `w` 和截距 `b`，使分类边界 `w^T x + b = 0` 尽可能把两类样本分开，并使距离边界最近的样本到边界的间隔最大。软间隔 SVM 允许部分样本被误分，并使用参数 C 控制误分类惩罚。C 越大，模型越努力拟合训练数据，过拟合风险也越高。C 越小，模型允许更多间隔违反，边界更平滑。

标准 `SVC` 在大规模数据上计算成本较高。常见核 SVM 的训练复杂度可能接近 `O(n²)` 到 `O(n³)`，在 28 万条交易和 SMOTE 后更大的训练集上非常缓慢。因此，本章采用 `LinearSVC`。它基于线性优化实现，适合大规模高维稀疏或连续特征数据。`LinearSVC` 不直接输出概率，需要使用 `CalibratedClassifierCV` 做概率校准，才能计算 PR-AUC 并进行阈值分析。

```python
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.svm.LinearSVC.html
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.calibration.CalibratedClassifierCV.html
linear_svc = LinearSVC(
    C=1.0,
    max_iter=2000,
    random_state=42,
    dual="auto",
)
```

`C=1.0` 是正则化强度的控制参数。`max_iter=2000` 增加最大迭代次数，降低不收敛风险。`random_state=42` 保证复现性。`dual="auto"` 让 scikit-learn 根据样本量和特征数自动选择原始问题或对偶问题求解。

```python
svm_clf = CalibratedClassifierCV(linear_svc, method="sigmoid", cv=3)
svm_clf.fit(X_train_smote, y_train_smote)

y_pred_svm = svm_clf.predict(X_test)
y_prob_svm = svm_clf.predict_proba(X_test)[:, 1]

print("=== SVM，线性核加 SMOTE，模型评估 ===")
print(classification_report(y_test, y_pred_svm, target_names=["正常", "欺诈"]))
print(f"ROC-AUC: {roc_auc_score(y_test, y_prob_svm):.4f}")
print(f"PR-AUC:  {average_precision_score(y_test, y_prob_svm):.4f}")
```

`CalibratedClassifierCV` 把 SVM 决策值映射为概率。`method="sigmoid"` 表示使用 Platt Scaling。`cv=3` 表示用三折交叉验证进行校准。`fit` 使用 SMOTE 后的训练集。`predict` 输出离散类别。`predict_proba` 输出欺诈概率，`[:, 1]` 取正类欺诈概率。SVM 的优点是理论清晰，适合高维线性边界。它的局限是对尺度敏感，非线性表达有限，概率输出需要额外校准。

### 4.3.2.4 随机森林与类别权重

随机森林通过 Bootstrap 抽样训练多棵决策树，并在每次分裂时随机选择部分特征。每棵树都可能学习到不同的局部结构，最终通过投票或概率平均给出结果。随机森林可以处理非线性和特征交互，对异常值和单调变换相对稳健，也能给出特征重要性。

在本案例中，随机森林直接使用原始训练集，并通过 `class_weight="balanced"` 处理类别不平衡。这样做的原因是，随机森林能够利用大量正常交易信息，而不需要像 SMOTE 那样增加训练集规模。类别权重会让少数类欺诈交易在分裂和损失计算中获得更高关注。

```python
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.ensemble.RandomForestClassifier.html
rf_clf = RandomForestClassifier(
    n_estimators=200,
    max_depth=10,
    max_features="sqrt",
    random_state=42,
    class_weight="balanced",
    n_jobs=-1,
)
```

`n_estimators=200` 表示训练 200 棵树。`max_depth=10` 限制树深度，降低过拟合风险。`max_features="sqrt"` 表示每次分裂只考虑特征数平方根个候选特征，这是降低树间相关性的重要机制。`class_weight="balanced"` 根据类别频率自动设置权重。`n_jobs=-1` 启用并行训练。

```python
rf_clf.fit(X_train, y_train)

y_pred_rf = rf_clf.predict(X_test)
y_prob_rf = rf_clf.predict_proba(X_test)[:, 1]

print("=== 随机森林，class_weight=balanced，模型评估 ===")
print(classification_report(y_test, y_pred_rf, target_names=["正常", "欺诈"]))
print(f"ROC-AUC: {roc_auc_score(y_test, y_prob_rf):.4f}")
print(f"PR-AUC:  {average_precision_score(y_test, y_prob_rf):.4f}")
```

`fit` 使用原始训练集训练随机森林。`predict` 使用默认阈值 0.5 输出类别。`predict_proba` 输出欺诈概率。随机森林通常能提供较强基线，但在极端不平衡场景中，默认阈值未必能获得高 Recall。因此，后续仍需要阈值分析。

### 4.3.2.5 XGBoost 与正类权重

XGBoost 是梯度提升树模型。它按顺序训练多棵树，每一棵新树都试图修正前面模型的误差。与随机森林不同，随机森林是并行集成，主要降低方差；XGBoost 是串行提升，主要降低偏差，并通过正则化、学习率和树结构参数控制过拟合。对于结构化表格数据，XGBoost 通常是强基线模型。

在极端不平衡场景中，XGBoost 常使用 `scale_pos_weight` 调整正类权重。经验初值通常取负类样本数除以正类样本数。

```python
neg_count = (y_train == 0).sum()
pos_count = (y_train == 1).sum()
scale_ratio = neg_count / pos_count

print(f"负类：{neg_count}, 正类：{pos_count}, 比例：{scale_ratio:.1f}")
```

`neg_count` 是正常交易数量。`pos_count` 是欺诈交易数量。`scale_ratio` 是负类和正类数量比。若训练集中正常交易约为欺诈交易的 578 倍，则模型如果不加权，很容易偏向预测正常。正类权重能让欺诈样本在损失函数中产生更大影响。

```python
# Ref: https://xgboost.readthedocs.io/en/stable/parameter.html
xgb_clf = xgb.XGBClassifier(
    n_estimators=200,
    max_depth=6,
    learning_rate=0.1,
    scale_pos_weight=scale_ratio,
    eval_metric="aucpr",
    random_state=42,
    n_jobs=-1,
)
```

`n_estimators=200` 表示提升轮数。`max_depth=6` 限制每棵树深度。`learning_rate=0.1` 控制每棵树对最终模型的贡献。`scale_pos_weight=scale_ratio` 提高欺诈样本权重。`eval_metric="aucpr"` 使用 PR-AUC 相关指标作为训练评价口径，更符合不平衡场景。`random_state=42` 保证复现性。旧版本 XGBoost 代码中常见 `use_label_encoder=False`，在较新版本中通常不再需要。

```python
xgb_clf.fit(X_train, y_train)

y_pred_xgb = xgb_clf.predict(X_test)
y_prob_xgb = xgb_clf.predict_proba(X_test)[:, 1]

print("=== XGBoost，scale_pos_weight，模型评估 ===")
print(classification_report(y_test, y_pred_xgb, target_names=["正常", "欺诈"]))
print(f"ROC-AUC: {roc_auc_score(y_test, y_prob_xgb):.4f}")
print(f"PR-AUC:  {average_precision_score(y_test, y_prob_xgb):.4f}")
```

`fit` 训练 XGBoost 模型。`predict` 使用默认阈值 0.5。`predict_proba` 输出欺诈概率。XGBoost 的优势是非线性表达能力强，能处理特征交互，并支持类别不平衡权重。它的局限是参数较多，解释性弱于线性模型和单棵树，且需要配合阈值、监控和解释工具才能进入生产系统。

## 4.3.3 风险识别与智能告警

反欺诈系统最终要把模型分数转化为行动。行动可以是实时拦截、短信验证、3D Secure 验证、人工复核、用户通知、延迟清算或放行。不同动作的成本不同，因此模型阈值不应固定为 0.5。低阈值可以捕获更多欺诈，但会带来更多误报。高阈值可以减少误报，但会漏掉更多欺诈。风险识别的核心是把模型排序能力、业务成本和复核能力结合起来。

### 4.3.3.1 PR 曲线与 ROC 曲线

ROC 曲线绘制的是真正率和假正率之间的关系。PR 曲线绘制的是 Precision 和 Recall 之间的关系。在极端不平衡数据中，ROC-AUC 可能显得过于乐观。原因是正常交易数量巨大，即使出现不少误报，假正率的分母仍然很大，FPR 可能仍然较低。PR-AUC 更关注被模型判为正类的样本质量，因此更适合欺诈检测。

Precision、Recall 和 F1 的定义如下。

$$Precision=\frac{TP}{TP+FP}$$

$$Recall=\frac{TP}{TP+FN}$$

$$F1=\frac{2\times Precision\times Recall}{Precision+Recall}$$

其中，TP 表示真实欺诈且被识别为欺诈，FP 表示正常交易被误报为欺诈，FN 表示欺诈交易被漏掉。反欺诈业务通常特别关注 FN，因为漏报会造成真实损失。

```python
fig, axes = plt.subplots(1, 2, figsize=(16, 6))

ax = axes[0]
for name, y_score in [
    ("SVM，线性核", y_prob_svm),
    ("随机森林", y_prob_rf),
    ("XGBoost", y_prob_xgb),
    ("Isolation Forest", iso_scores),
]:
    precision_vals, recall_vals, _ = precision_recall_curve(y_test, y_score)
    ap = average_precision_score(y_test, y_score)
    ax.plot(recall_vals, precision_vals, label=f"{name} (AP={ap:.4f})")

baseline = y_test.mean()
ax.axhline(y=baseline, color="gray", linestyle="--", label=f"随机基线 ({baseline:.4f})")
ax.set_xlabel("Recall，召回率")
ax.set_ylabel("Precision，精确率")
ax.set_title("PR 曲线对比")
ax.legend(loc="upper right")
ax.set_xlim([0, 1])
ax.set_ylim([0, 1.05])
```

这段代码先创建左右两个子图。第一部分绘制 PR 曲线。循环中的 `precision_recall_curve` 计算不同阈值下的 Precision 和 Recall。`average_precision_score` 计算平均精确率，常被作为 PR-AUC 的实现形式。`baseline = y_test.mean()` 是随机分类器在 PR 图中的基线，等于测试集正类比例。若模型 PR 曲线明显高于基线，说明模型确实捕捉到了欺诈信号。


第二部分绘制 ROC 曲线。`roc_curve` 返回 FPR、TPR 和阈值。`roc_auc_score` 计算 ROC 曲线下面积。虚线表示随机猜测。学生应同时观察 ROC-AUC 和 PR-AUC。如果 ROC-AUC 很高但 PR-AUC 不高，说明模型排序能力在负类数量巨大的情况下被 ROC 指标放大，实际拦截效果仍需谨慎评估。

### 4.3.3.2 指标汇总与混淆矩阵

所有模型的 Accuracy 都可能超过 99%，但这并不说明模型可以用于反欺诈。如果一个模型完全不识别欺诈，也能因为正常交易占绝大多数而得到很高准确率。因此，指标表中必须同时展示 Precision、Recall、F1、ROC-AUC 和 PR-AUC。

```python
results = []
for name, y_pred, y_score in [

print("=== 四模型评估指标汇总 ===")

```

循环逐个整理模型结果。`Accuracy` 表示总体分类正确率。`Precision` 表示被判为欺诈的交易中真实欺诈占比。`Recall` 表示真实欺诈交易中被识别出的比例。`F1 Score` 是 Precision 和 Recall 的调和平均。`ROC-AUC` 衡量整体排序能力。`PR-AUC` 更能反映极端不平衡场景下的正类识别质量。

混淆矩阵把错误结构显示出来。左上角是正常交易被正确放行。右上角是正常交易被误报为欺诈。左下角是欺诈交易被漏报为正常。右下角是欺诈交易被成功识别。反欺诈系统中，左下角的 FN 是高风险错误，右上角的 FP 是客户体验和运营成本问题。模型选择必须在这两类错误之间进行权衡。

### 4.3.3.3 阈值分析与优化

默认分类阈值 0.5 往往不是反欺诈最优阈值。模型输出的是欺诈概率或风险分数，业务系统需要选择阈值。若阈值降低，更多交易会被标记为欺诈，Recall 通常提高，FN 减少，但 FP 增加。若阈值升高，模型只拦截最可疑交易，Precision 可能提高，但 Recall 下降。

本节以 XGBoost 为例进行阈值分析。实际项目中，应先根据验证集选择阈值，再在测试集进行最终评估。这里的代码用于说明阈值与指标之间的关系。

```python
best_model_name = "XGBoost"
best_y_prob = y_prob_xgb

precisions, recalls, thresholds_pr = precision_recall_curve(y_test, best_y_prob)
thresholds_pr = np.append(thresholds_pr, 1.0)

f1_scores = 2 * precisions * recalls / (precisions + recalls + 1e-8)
```

`best_model_name` 保存用于阈值分析的模型名称。`best_y_prob` 保存该模型输出的欺诈概率。`precision_recall_curve` 返回不同阈值下的 Precision、Recall 和阈值数组。由于 Precision 和 Recall 数组比阈值数组多一个元素，代码用 `np.append` 补充 1.0，使三个数组长度一致。`f1_scores` 根据公式计算每个阈值对应的 F1，`1e-8` 用于避免除零。

```python
best_f1_idx = np.argmax(f1_scores)
best_f1_threshold = thresholds_pr[best_f1_idx]

recall_mask = recalls >= 0.95
if recall_mask.any():
    valid_indices = np.where(recall_mask)[0]
    valid_thresholds = thresholds_pr[valid_indices]
    recall95_position = valid_indices[np.argmax(valid_thresholds)]
    recall95_threshold = thresholds_pr[recall95_position]
else:
    recall95_position = 0
    recall95_threshold = 0.0

print(f"最优 F1 阈值：{best_f1_threshold:.4f}，对应 F1={f1_scores[best_f1_idx]:.4f}")
print(f"Recall≥0.95 的最高可用阈值：{recall95_threshold:.4f}")
```

`np.argmax(f1_scores)` 找到 F1 最大的位置。`best_f1_threshold` 是对应阈值。`recall_mask` 找出 Recall 大于等于 0.95 的位置。若存在满足条件的位置，代码选择其中最高的阈值。选择最高阈值的原因是，在保证 Recall 目标的同时尽量提高 Precision，减少误报。若没有任何阈值满足 Recall 目标，则把参考阈值设为 0。

```python
fig, axes = plt.subplots(1, 2, figsize=(16, 5))

ax = axes[0]
ax.plot(thresholds_pr, precisions, label="Precision", color="steelblue")
ax.plot(thresholds_pr, recalls, label="Recall", color="crimson")
ax.plot(thresholds_pr, f1_scores, label="F1 Score", color="green", linestyle="--")
ax.set_xlabel("分类阈值")
ax.set_ylabel("指标值")
ax.set_title(f"{best_model_name}：阈值与指标关系")
ax.set_xlim([0, 1])
ax.axvline(x=0.5, color="gray", linestyle=":", label="默认阈值 0.5")
ax.axvline(x=best_f1_threshold, color="green", linestyle="--", alpha=0.7)
ax.legend()

ax = axes[1]
ax.plot(recalls, precisions, color="steelblue")
ax.scatter(
    recalls[best_f1_idx],
    precisions[best_f1_idx],
    color="green",
    s=100,
    zorder=5,
    label=f"最优 F1 点，阈值={best_f1_threshold:.3f}",
)
ax.set_xlabel("Recall")
ax.set_ylabel("Precision")
ax.set_title(f"{best_model_name}：PR 曲线关键点")
ax.legend()

plt.tight_layout()
plt.show()
```

左图展示阈值变化时 Precision、Recall 和 F1 的变化。灰色竖线是默认阈值 0.5。绿色竖线是 F1 最大对应阈值。右图在 PR 曲线上标注最优 F1 点。学生应注意，最优 F1 阈值不一定是业务最优阈值。如果银行更关注防损，可能选择更低阈值以提高 Recall。如果银行更关注客户体验，可能选择更高阈值以减少误报。

```python
threshold_candidates = [0.3, 0.4, 0.5, best_f1_threshold, recall95_threshold]
threshold_results = []

for t in sorted(set(threshold_candidates)):
    y_pred_t = (best_y_prob >= t).astype(int)
    threshold_results.append({
        "阈值": round(float(t), 4),
        "Precision": round(precision_score(y_test, y_pred_t, zero_division=0), 4),
        "Recall": round(recall_score(y_test, y_pred_t), 4),
        "F1": round(f1_score(y_test, y_pred_t), 4),
        "误报数 FP": int(((y_pred_t == 1) & (y_test == 0)).sum()),
        "漏报数 FN": int(((y_pred_t == 0) & (y_test == 1)).sum()),
    })

threshold_df = pd.DataFrame(threshold_results)
print(f"=== {best_model_name} 不同阈值下的表现 ===")
display(threshold_df)
```

`threshold_candidates` 包含常见阈值、最优 F1 阈值和 Recall 目标阈值。循环中，`best_y_prob >= t` 把概率转成类别。`FP` 是正常交易被误报数量，`FN` 是欺诈交易被漏报数量。这个表比单一指标更接近业务讨论，因为风控团队可以直接看到降低阈值会增加多少误报、减少多少漏报。

若每笔漏检欺诈的平均损失为 L，每笔误报的处理成本为 C，则阈值 t 下的简化总成本可以写为

$$Cost(t)=FN(t)\times L+FP(t)\times C.$$

当 L 远大于 C 时，较低阈值可能更合理。当人工复核资源有限或客户体验成本很高时，较高阈值可能更合理。实际系统还应把交易金额、客户等级、商户风险、地理位置、设备指纹和历史行为纳入成本函数。

### 4.3.3.4 特征重要性与脱敏特征解释

V1 到 V28 是 PCA 脱敏后的主成分，不能直接解释为商户类别、地理位置或设备特征。因此，特征重要性只能说明哪些主成分方向对模型区分欺诈与正常交易贡献较大，不能直接生成业务拒绝原因。Time_scaled 和 Amount_scaled 仍有明确含义，可以作为解释线索之一，但不能单独作为欺诈判定依据。

```python
fig, axes = plt.subplots(1, 2, figsize=(16, 6))
```

### 4.3.3.5 风险等级与智能告警

模型分数需要转化为风险等级，才能进入业务系统。反欺诈场景通常不只有拦截和放行两种动作。高风险交易可以直接拦截或强验证，中风险交易可以进入人工复核或短信确认，低风险交易可以放行但保留监控记录。这样的分层策略比单一阈值更灵活。

```python
def assign_fraud_risk_level(prob: float) -> str:
    """根据欺诈概率划分交易风险等级。"""
    if prob >= 0.80:
        return "红色告警"
    if prob >= 0.50:
        return "橙色告警"
    if prob >= 0.20:
        return "黄色关注"
    return "绿色放行"


def suggest_fraud_action(level: str) -> str:
    """根据风险等级生成交易处置建议。"""
    action_map = {
        "红色告警": "实时拦截交易，触发强身份验证，并进入人工复核队列",
        "橙色告警": "暂缓交易或触发短信确认，复核设备、商户和历史行为",
        "黄色关注": "允许交易继续，但提高后续监控频率并记录风险原因",
        "绿色放行": "正常放行，保留常规风控日志",
    }
    return action_map[level]
```

`assign_fraud_risk_level` 把欺诈概率映射为四个风险等级。阈值 0.80、0.50 和 0.20 是教学示例，正式项目应通过验证集、历史回测、误报成本和人工复核容量确定。`suggest_fraud_action` 把风险等级映射为处置建议，使模型输出可以被业务人员理解和执行。

```python
rng = np.random.default_rng(42)
sample_positions = rng.choice(X_test.shape[0], size=10, replace=False)
sample_prob = y_prob_xgb[sample_positions]

alert_report = pd.DataFrame({
    "样本位置": sample_positions,
    "欺诈概率": np.round(sample_prob, 4),
    "真实状态": ["欺诈" if y_test.iloc[pos] == 1 else "正常" for pos in sample_positions],
})
alert_report["风险等级"] = alert_report["欺诈概率"].apply(assign_fraud_risk_level)
alert_report["建议动作"] = alert_report["风险等级"].apply(suggest_fraud_action)

display(alert_report)
```

`np.random.default_rng(42)` 创建可复现随机数生成器。`choice` 从测试集中抽取 10 笔交易用于展示。`sample_prob` 是 XGBoost 输出的欺诈概率。`alert_report` 保存样本位置、概率和真实状态。在线上系统中，真实状态在交易当时不可知，通常要等拒付、客户申诉、人工调查或后续标签回流后才能确认。`apply` 把概率转换为风险等级和建议动作。

智能告警系统应保存审计记录。记录内容应包括交易编号、模型版本、评分时间、欺诈概率、阈值版本、告警等级、处置动作和人工复核结果。这样，当客户投诉或监管检查发生时，可以回放当时的决策依据。

```python
class FraudAlertAgent:
    """反欺诈规则 Agent，用于生成告警解释和审计记录。"""

    def __init__(
        self,
        review_levels: tuple[str, ...] = ("红色告警", "橙色告警"),
        policy_version: str = "fraud_alert_policy_v1",
    ) -> None:
        self.review_levels = review_levels
        self.policy_version = policy_version

    def explain(self, row: pd.Series) -> str:
        explanation = (
            f"样本 {int(row['样本位置'])} 的模型欺诈概率为 {row['欺诈概率']:.4f}，"
            f"风险等级为 {row['风险等级']}。"
            f"建议动作是：{row['建议动作']}。"
        )
        return explanation

    def run(self, report: pd.DataFrame) -> tuple[pd.DataFrame, pd.DataFrame]:
        result = report.copy()
        result["是否进入人工复核"] = result["风险等级"].isin(self.review_levels)
        result["文字解释"] = result.apply(self.explain, axis=1)

        audit_log = result[["样本位置", "欺诈概率", "风险等级", "是否进入人工复核"]].copy()
        audit_log["策略版本"] = self.policy_version
        return result, audit_log
```

`FraudAlertAgent` 是一个规则型 Agent。它不改变模型分数，也不自动修改阈值，只负责根据结构化报告生成解释和审计记录。`__init__` 指定哪些等级进入人工复核，并保存策略版本。`explain` 把一条交易记录转化为自然语言说明。`run` 生成完整告警报告和审计日志。审计日志中的策略版本很重要，因为阈值和动作规则可能随时间调整。

```python
fraud_agent = FraudAlertAgent()
final_alert_report, fraud_audit_log = fraud_agent.run(alert_report)

display(final_alert_report[["样本位置", "风险等级", "是否进入人工复核", "文字解释"]])
display(fraud_audit_log)
```

`fraud_agent` 实例化规则 Agent。`run` 执行告警流程。`final_alert_report` 面向业务人员，包含风险等级、复核标记和文字解释。`fraud_audit_log` 面向审计和模型治理，记录关键决策字段。生产系统中，Agent 可以进一步对接工单系统、短信系统、交易拦截系统和人工复核平台，但最终高影响决策仍应受到规则审批、模型监控和人工监督约束。

## 本章模型选择的综合讨论

Isolation Forest 适合无标签或标签极少的冷启动阶段。它不需要欺诈标签，可以快速发现与多数交易不同的异常行为。但它无法区分业务异常和欺诈异常，通常只能作为基线、预警补充或新模式探索工具。

SVM 适合展示最大间隔和线性分类思想。由于本数据量较大，本章选择 LinearSVC，而不是 RBF 核 SVC。线性 SVM 训练较快，适合高维脱敏特征。它的局限是非线性表达有限，而且需要概率校准才能用于 PR-AUC 和阈值策略。

随机森林适合作为稳健的非线性基线。它能捕捉特征交互，对异常值相对稳健，并能输出特征重要性。它的不足是模型体积较大，极端不平衡下默认阈值可能过于保守，单笔交易解释也不如单棵树直观。

XGBoost 在结构化表格数据上通常表现较强。它能通过 `scale_pos_weight` 处理类别不平衡，并通过树集成学习复杂非线性关系。本章把 XGBoost 作为阈值分析和告警策略的示例模型。它的不足是调参复杂，解释性有限，且上线后需要持续监控 PR-AUC、Recall、误报率、漏报率和数据漂移。

SMOTE、欠采样和类别权重各有适用条件。SMOTE 保留多数类信息并扩充少数类，但可能生成噪声样本。欠采样训练快，但会丢失大量正常交易信息。类别权重不改变样本分布，部署简单，但权重设置需要验证。实际反欺诈系统常把规则引擎、监督模型、异常检测模型和人工复核结合起来，而不是依赖单一模型。

## 拓展方向

1. 比较在训练集内标准化和全量标准化两种流程下的指标差异，说明数据泄漏为什么会使评估偏乐观。
2. 分别使用 SMOTE、随机欠采样、类别权重和 SMOTE 加欠采样的混合策略训练同一模型，比较 PR-AUC、Recall 和误报数。
3. 把 XGBoost 的阈值选择从测试集移到验证集，再使用测试集做最终评估，构建更严格的模型选择流程。
4. 使用业务成本函数选择阈值。可以设漏检欺诈损失为 L，误报处理成本为 C，计算不同阈值下的总成本。
5. 在交易金额上引入金额加权 Recall，使大额欺诈漏报获得更高惩罚。
6. 使用时间切分替代随机切分，观察欺诈模式随时间变化时模型表现是否下降。
7. 使用 SHAP 解释 XGBoost 单笔交易的风险来源，但需要说明 V1 到 V28 是脱敏主成分，不能直接解释为具体业务字段。
8. 构建规则加模型的混合系统。例如，对超大金额、异地设备和模型高分交易执行强验证，对模型中分交易进入人工复核。
9. 比较 Isolation Forest、One-Class SVM、Local Outlier Factor 和自编码器在无监督欺诈检测中的表现。
10. 将告警 Agent 接入工单流程，让 Agent 生成复核摘要、检查缺失字段和记录策略版本，但不允许 Agent 擅自改变模型分数和拦截规则。
