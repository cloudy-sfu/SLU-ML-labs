# 4.2 企业破产预警与财务风险分析案例

企业破产预警是金融风险管理中的重要任务。银行、投资机构和监管部门都需要在企业真正进入破产程序之前，识别盈利能力下降、偿债压力上升、现金流恶化和资本结构失衡等早期信号。预警模型的价值不在于给出绝对确定的结论，而在于把高风险企业提前排到人工复核名单前列，使有限的尽调资源用于最需要关注的对象。

本章以 Polish Companies Bankruptcy 数据为案例，围绕企业财务比率构建破产预测模型，并比较 Logistic 回归、支持向量机、随机森林和 XGBoost 四类方法。案例从数据读取、缺失值处理、异常值裁剪和类别不平衡处理开始，逐步进入模型训练、模型评估、关键财务指标解释和自动化预警策略设计。

企业破产预警模型的任务，是根据企业财务报表中的盈利能力、偿债能力、营运效率和资本结构指标，估计企业未来进入破产、违约或财务困境状态的概率。与个人信用评分相比，企业破产预测有两个更突出的困难。第一，破产样本极少，类别不平衡严重。第二，财务比率容易出现极端值、缺失值和口径差异。因此，本章重点讨论数据清洗、异常值处理、类别不平衡处理、模型解释和预警阈值。

## 4.2.1 财务比率与行业指标提取

企业财务风险分析通常从财务报表出发。资产负债表提供企业资产、负债和所有者权益结构。利润表提供收入、成本、费用和利润信息。现金流量表提供经营、投资和筹资活动的现金流信息。单个报表数值往往受企业规模影响，因此风险建模更常使用财务比率。例如，总负债除以总资产可以衡量杠杆水平，净利润除以总资产可以衡量资产收益能力，流动资产除以流动负债可以衡量短期偿债能力。

本章使用的 Polish Companies Bankruptcy 数据集中，特征命名为 Attr1 到 Attr64。每个特征代表一个财务比率。目标变量 class 中，0 表示企业存续，1 表示企业破产。破产样本约占 4.7%，这是典型的低频高损失风险事件。

### 4.2.1.3 财务比率含义与行业相对指标

财务比率把报表项目转换为可比较的风险指标。盈利能力指标回答企业能否稳定创造利润。偿债能力指标回答企业能否按期履行债务。营运效率指标回答企业资产、存货和应收账款周转是否顺畅。资本结构指标回答企业是否过度依赖债务融资。企业破产通常不是由单一指标决定，而是多类指标共同恶化的结果。

```python
financial_ratio_notes = {
    "Attr1": "净利润 / 总资产，常用于衡量资产收益能力",

for key, value in financial_ratio_notes.items():
    print(f"{key}: {value}")
```

这个字典用于把模型字段翻译为财务语言。机器学习模型处理的是数值矩阵，风险管理人员需要的是财务含义。若一个企业的盈利能力指标长期偏低、杠杆率偏高、流动性指标偏弱，模型通常会给出更高的破产概率。字段解释必须依赖数据字典和会计口径，不能只根据变量名作过度推断。

行业指标用于解决跨行业可比性问题。同样的资产负债率，在重资产行业可能接近行业常态，在轻资产服务业可能已经表示较高风险。同样的存货周转速度，在制造业、零售业和软件企业中的含义也不同。因此，企业风险模型常把绝对财务比率转化为相对指标。常见做法包括企业指标减去行业中位数、企业指标除以行业四分位距、企业在行业内的百分位位置，以及行业历史破产基准率。

设企业 i 属于行业 g，某一财务比率为 x_{ig}。行业中位数相对偏离可以写成

$$d_{ig}=x_{ig}-\operatorname{median}(x_g).$$

行业四分位距标准化指标可以写成

$$z_{ig}^{IQR}=\frac{x_{ig}-\operatorname{median}(x_g)}{Q_{0.75}(x_g)-Q_{0.25}(x_g)}.$$

第一个指标保留了企业偏离同业中心的方向。第二个指标考虑了不同行业波动范围不同的问题。若某行业本身财务比率分散度很高，单纯偏离中位数并不一定代表异常；若某行业财务结构高度稳定，同样大小的偏离就更值得关注。

下面的函数给出行业相对指标的构造方法。`group_column` 用于指定行业字段。若业务数据中已经有行业代码，函数按行业分组计算相对指标。若教师希望在公开数据上演示相对指标思想，也可以把 `group_column` 设为 `None`，此时函数会用整个样本作为组合基准，生成组合中位数偏离、组合四分位距得分和组合百分位。组合基准不能替代真实行业基准，但可以帮助学生理解相对指标的计算过程。

```python
def add_relative_benchmark_features(
    data: pd.DataFrame,
    ratio_columns: list[str],
    group_column: str | None = None,
    prefix: str = "benchmark",
) -> pd.DataFrame:
    """构造财务比率相对基准指标。"""
    X_base = data[ratio_columns].copy()

    if group_column is None:
        center = pd.DataFrame(
            np.tile(X_base.median().to_numpy(), (len(X_base), 1)),
            columns=ratio_columns,
            index=X_base.index,
        )
        q75 = X_base.quantile(0.75)
        q25 = X_base.quantile(0.25)
        iqr = pd.DataFrame(
            np.tile((q75 - q25).replace(0, np.nan).to_numpy(), (len(X_base), 1)),
            columns=ratio_columns,
            index=X_base.index,
        )
        pct = X_base.rank(pct=True)
    else:
        center = data.groupby(group_column)[ratio_columns].transform("median")
        q75 = data.groupby(group_column)[ratio_columns].transform(lambda s: s.quantile(0.75))
        q25 = data.groupby(group_column)[ratio_columns].transform(lambda s: s.quantile(0.25))
        iqr = (q75 - q25).replace(0, np.nan)
        pct = X_base.copy()
        for col in ratio_columns:
            pct[col] = data.groupby(group_column)[col].rank(pct=True)

    diff_from_center = X_base - center
    diff_from_center.columns = [f"{col}_{prefix}_median_gap" for col in ratio_columns]

    scaled_gap = diff_from_center / iqr
    scaled_gap.columns = [f"{col}_{prefix}_iqr_score" for col in ratio_columns]

    pct.columns = [f"{col}_{prefix}_percentile" for col in ratio_columns]

    return pd.concat([X_base, diff_from_center, scaled_gap, pct], axis=1)
```

`X_base` 保存原始财务比率。`center` 是基准中位数，既可以来自行业分组，也可以来自整体组合。`iqr` 是四分位距，用于衡量基准组内部的离散程度。`pct` 是百分位特征，用于回答企业在同业或组合中的相对位置。函数最后把原始比率、中位数偏离、四分位距得分和百分位特征拼接在一起。

```python
ratio_cols = X_raw.select_dtypes(include=[np.number]).columns.tolist()
benchmark_feature_preview = add_relative_benchmark_features(
    data=df,
    ratio_columns=ratio_cols,
    group_column=None,
    prefix="portfolio",
)

print(f"原始财务比率数：{len(ratio_cols)}")
print(f"加入相对基准后的示范特征数：{benchmark_feature_preview.shape[1]}")
display(benchmark_feature_preview.head())

X_raw = df[ratio_cols].copy()
```

这段代码先识别数值型财务比率，再生成一份相对基准特征示范表。`group_column=None` 表示用当前组合计算相对指标。若业务数据中存在 `industry_code`、`sector`、`sic` 或 `naics` 等行业字段，只需把该字段名传入 `group_column`，同一函数即可按行业生成相对指标。最后一行把后续模型输入设回原始财务比率，目的是使后续模型结果与本案例的基准结果保持一致。若正式项目要把行业相对指标纳入模型，应把相对指标的中位数、四分位距和百分位计算放入训练集内部完成，避免测试集信息提前进入训练流程。

行业相对指标并不总是优于原始财务比率。若行业样本数量很少，行业中位数和分位数会不稳定。若行业代码过粗，同一行业内部企业差异仍然很大。若行业代码过细，每组样本可能不足以支持稳定估计。因此，行业特征需要配合最小样本数、时间窗口和稳健统计量使用。
真实财务数据中经常出现缺失。原因可能是企业没有披露某些报表项目，也可能是比率计算时分母为零，还可能是不同企业会计科目口径不完全一致。缺失值本身可能携带风险信息，但在本章基础模型中，先使用中位数填充进行处理。

### 4.2.1.5 防止数据泄漏的预处理流程

一种常见但不够严谨的预处理顺序，是先在全量数据上做中位数填充和 Winsorize 异常值裁剪，然后再划分训练集和测试集。这种写法便于快速演示，但会让测试集的信息参与训练阶段的统计量计算，存在轻微数据泄漏。本章采用更严格的写法，先划分训练集和测试集，再只用训练集学习填充值和异常值边界。

```python
X_train_raw, X_test_raw, y_train, y_test = train_test_split(
    X_raw,
    y,
    test_size=0.2,
    random_state=42,
    stratify=y,
)

print(f"训练集大小：{X_train_raw.shape[0]}")
print(f"测试集大小：{X_test_raw.shape[0]}")
print(f"训练集破产比例：{y_train.mean():.4f}")
print(f"测试集破产比例：{y_test.mean():.4f}")
```

`train_test_split` 将数据划分为训练集和测试集。`test_size=0.2` 表示测试集占 20%。`random_state=42` 保证结果可复现。`stratify=y` 保证训练集和测试集具有相近的破产比例。在破产样本只有约 5% 的情况下，分层抽样很重要，否则测试集中可能破产样本过少，评估结果不稳定。

```python
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.impute.SimpleImputer.html
imputer = SimpleImputer(strategy="median")

X_train_imputed = pd.DataFrame(
    imputer.fit_transform(X_train_raw),
    columns=X_train_raw.columns,
    index=X_train_raw.index,
)
X_test_imputed = pd.DataFrame(
    imputer.transform(X_test_raw),
    columns=X_test_raw.columns,
    index=X_test_raw.index,
)

print(f"训练集填充后缺失值总数：{X_train_imputed.isnull().sum().sum()}")
print(f"测试集填充后缺失值总数：{X_test_imputed.isnull().sum().sum()}")
```

`SimpleImputer(strategy="median")` 使用中位数填充缺失值。`fit_transform` 只用于训练集，它先学习每个特征的中位数，再完成填充。`transform` 用训练集学到的中位数处理测试集。这样做符合真实预测流程，因为模型上线时只能使用历史训练数据学到的填充规则。

```python
def fit_winsor_bounds(
    X_train_data: pd.DataFrame,
    lower_q: float = 0.01,
    upper_q: float = 0.99,
) -> tuple[pd.Series, pd.Series]:
    """仅基于训练集计算 Winsorize 上下界。"""
    lower_bounds = X_train_data.quantile(lower_q)
    upper_bounds = X_train_data.quantile(upper_q)
    return lower_bounds, upper_bounds


def apply_winsor_bounds(
    X_data: pd.DataFrame,
    lower_bounds: pd.Series,
    upper_bounds: pd.Series,
) -> pd.DataFrame:
    """使用训练集上下界裁剪任意数据集。"""
    X_clipped = X_data.copy()
    for col in X_clipped.columns:
        X_clipped[col] = X_clipped[col].clip(lower_bounds[col], upper_bounds[col])
    return X_clipped
```

`fit_winsor_bounds` 计算训练集每个变量的 1% 和 99% 分位数。`apply_winsor_bounds` 使用这些边界对训练集和测试集进行裁剪。Winsorize 不删除样本，而是把极端值压到边界上。财务比率常有极端值，例如负资产、极端负利润或分母接近零的比率。裁剪可以降低极端值对 Logistic 回归和 SVM 的影响。

```python
lower_bounds, upper_bounds = fit_winsor_bounds(X_train_imputed)

X_train = apply_winsor_bounds(X_train_imputed, lower_bounds, upper_bounds)
X_test = apply_winsor_bounds(X_test_imputed, lower_bounds, upper_bounds)

print("异常值裁剪完成。")
print(f"处理后训练集维度：{X_train.shape}")
print(f"处理后测试集维度：{X_test.shape}")
display(X_train.describe().T.head(10))
```

这段代码完成异常值裁剪。`lower_bounds` 和 `upper_bounds` 只来自训练集。`X_train` 和 `X_test` 具有相同特征列。`describe().T.head(10)` 展示前 10 个特征的描述统计量，学生可以检查极端值是否被压缩。

## 4.2.2 违约风险预测模型设计

企业破产预警可以被看作二分类问题。给定企业财务比率向量 `x`，模型输出破产概率 `P(y=1|x)`。其中，1 表示破产，0 表示存续。

本节比较四类常见模型，并将它们放入统一的风险建模框架中。Logistic 回归强调可解释性。SVM 强调最大间隔和非线性边界。随机森林强调稳健性和特征随机采样。XGBoost 强调梯度提升和预测性能。选择这些模型的原因，是它们分别代表了线性概率模型、核方法、Bagging 集成和 Boosting 集成四种重要思想。

### 4.2.2.1 Logistic 回归模型

Logistic 回归把线性函数映射为 0 到 1 之间的概率。

$$P(y=1 \mid x)=\frac{1}{1+e^{-(\beta_0+\beta_1x_1+\cdots+\beta_px_p)}}$$

如果 `β_i` 大于 0，说明在其他变量不变时，该变量增加会提高破产对数几率。如果 `β_i` 小于 0，说明该变量增加会降低破产对数几率。`exp(β_i)` 是优势比，表示特征增加一个单位时破产赔率的倍数变化。由于本章使用标准化后的特征，系数大小可以更公平地比较变量影响强弱。

```python
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.linear_model.LogisticRegression.html
logit_clf = make_pipeline(
    StandardScaler(),
    LogisticRegression(
        max_iter=1000,
        class_weight="balanced",
        C=1.0,
        solver="lbfgs",
        random_state=42,
    ),
)
```

`make_pipeline` 将标准化和模型训练封装为一个整体。`StandardScaler` 用训练集均值和标准差对特征标准化。`LogisticRegression` 是二分类 Logistic 回归模型。`max_iter=1000` 提高最大迭代次数，减少不收敛风险。`class_weight="balanced"` 根据类别频率调整损失权重，使破产样本在训练中获得更高权重。`C=1.0` 是正则化强度的倒数，C 越小，正则化越强。`solver="lbfgs"` 是适合中小规模连续特征的优化算法。

```python
logit_clf.fit(X_train, y_train)

y_pred_logit = logit_clf.predict(X_test)
y_prob_logit = logit_clf.predict_proba(X_test)[:, 1]

print("=== Logistic 回归模型评估 ===")
print(classification_report(y_test, y_pred_logit, target_names=["存续", "破产"]))
print(f"AUC-ROC: {roc_auc_score(y_test, y_prob_logit):.4f}")
```


```python
logit_model = logit_clf.named_steps["logisticregression"]
coef_series = pd.Series(logit_model.coef_[0], index=X_train.columns)

top_features = coef_series.abs().sort_values(ascending=False).head(15)

```

`named_steps["logisticregression"]` 从 Pipeline 中取出 Logistic 回归模型。`coef_[0]` 是二分类模型的系数向量。`pd.Series` 将系数和特征名绑定。`abs().sort_values` 按系数绝对值排序。`head(15)` 取影响最大的 15 个指标。`top_coefs` 保留原始正负号，用于判断风险方向。

```python
plt.figure(figsize=(10, 6))

```

红色条表示该指标越大，模型判断破产风险越高。绿色条表示该指标越大，模型判断破产风险越低。Logistic 回归的优势在于这种方向解释较直观，适合监管报告和管理层沟通。局限在于它主要表达线性关系，对变量交互和非线性结构的表达能力较弱。

### 4.2.2.2 SVM 模型

SVM 的核心思想是寻找最大间隔分类边界。在线性不可分时，RBF 核函数可以把样本映射到更高维空间，使模型学习非线性边界。RBF 核的形式可以写为

$$K(x_i,x_j)=\exp(-\gamma \Vert x_i-x_j\Vert^2)$$

其中，`γ` 控制样本影响范围。`γ` 过大时，模型过于关注局部细节，容易过拟合。`γ` 过小时，边界过于平滑，容易欠拟合。由于 RBF 核依赖距离，特征标准化是必要步骤。

```python
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.svm.SVC.html
svm_clf = make_pipeline(
    StandardScaler(),
    SVC(
        kernel="rbf",
        C=1.0,
        gamma="scale",
        class_weight="balanced",
        probability=True,
        random_state=42,
    ),
)
```

`StandardScaler` 先统一特征尺度。`SVC` 是支持向量机分类器。`kernel="rbf"` 表示使用径向基函数核。`C=1.0` 控制误分类惩罚。`gamma="scale"` 让 scikit-learn 根据特征数量和方差自动设置 gamma。`class_weight="balanced"` 缓解破产样本稀少问题。`probability=True` 启用概率估计，便于计算 AUC 和生成风险评分。需要注意，启用概率估计会增加训练时间。

```python
svm_clf.fit(X_train, y_train)

y_pred_svm = svm_clf.predict(X_test)
y_prob_svm = svm_clf.predict_proba(X_test)[:, 1]

print("=== SVM 模型评估 ===")
print(classification_report(y_test, y_pred_svm, target_names=["存续", "破产"]))
print(f"AUC-ROC: {roc_auc_score(y_test, y_prob_svm):.4f}")
```

SVM 在高维小样本场景中常有较好理论性质，但在上万样本规模下训练可能较慢。在一次基准运行中，SVM 的 AUC-ROC 约为 0.7234，破产类 Recall 约为 0.5253。它能识别部分破产企业，但整体排序能力弱于随机森林和 XGBoost。

### 4.2.2.3 随机森林模型

随机森林通过 Bootstrap 抽样训练多棵决策树，并在每次分裂时随机选择部分特征。它的思想是让每棵树都具有一定差异，再通过集成投票降低方差。与 Logistic 回归和 SVM 相比，随机森林对单调变换和异常值更稳健，也能捕捉非线性和特征交互。

```python
# Ref: https://scikit-learn.org/stable/modules/generated/sklearn.ensemble.RandomForestClassifier.html
rf_clf = RandomForestClassifier(
    n_estimators=300,
    max_depth=10,
    max_features="sqrt",
    random_state=42,
    class_weight="balanced",
    n_jobs=-1,
)
```

`n_estimators=300` 表示训练 300 棵树。树的数量越多，模型越稳定，但训练时间也越长。`max_depth=10` 限制树深度，降低过拟合风险。`max_features="sqrt"` 表示每次分裂只考虑特征数平方根个候选变量，这可以降低树之间的相关性。`class_weight="balanced"` 使少数类破产企业获得更高训练权重。`n_jobs=-1` 使用所有可用 CPU 核心。

```python
rf_clf.fit(X_train, y_train)

y_pred_rf = rf_clf.predict(X_test)
y_prob_rf = rf_clf.predict_proba(X_test)[:, 1]

print("=== 随机森林模型评估 ===")
print(classification_report(y_test, y_pred_rf, target_names=["存续", "破产"]))
print(f"AUC-ROC: {roc_auc_score(y_test, y_prob_rf):.4f}")
```

在一次基准运行中，随机森林的 AUC-ROC 约为 0.8583，明显高于 Logistic 回归和 SVM。它的 Precision 较高，说明被判为破产的企业更可能是真的破产。但默认阈值下 Recall 不高，说明仍有不少破产企业被判为存续。这个现象说明，排序能力好并不等于默认阈值下的召回率一定高。

### 4.2.2.4 XGBoost 模型

XGBoost 是梯度提升树模型。它按顺序训练多棵树，每一棵新树都试图修正前面模型的误差。随机森林是并行集成，主要降低方差；XGBoost 是串行提升，主要降低偏差并优化损失函数。对于结构化表格数据，XGBoost 通常是强基线模型。

破产样本只有约 5%，因此需要设置正类权重。

```python
neg_count = (y_train == 0).sum()
pos_count = (y_train == 1).sum()
scale_ratio = neg_count / pos_count

print(f"存续企业数：{neg_count}")
print(f"破产企业数：{pos_count}")
print(f"正负样本权重比例：{scale_ratio:.2f}")
```

`neg_count` 是存续企业数量。`pos_count` 是破产企业数量。`scale_ratio` 等于负类数量除以正类数量。在分层划分后的训练集中，存续企业通常约为 8006 家，破产企业约为 396 家，比例约为 20.22。这意味着如果不处理类别不平衡，模型很容易偏向预测存续。

```python
# Ref: https://xgboost.readthedocs.io/en/stable/parameter.html
xgb_clf = xgb.XGBClassifier(
    n_estimators=300,
    max_depth=6,
    learning_rate=0.1,
    scale_pos_weight=scale_ratio,
    eval_metric="logloss",
    random_state=42,
)
```


```python
xgb_clf.fit(X_train, y_train)

y_pred_xgb = xgb_clf.predict(X_test)
y_prob_xgb = xgb_clf.predict_proba(X_test)[:, 1]

print("=== XGBoost 模型评估 ===")
print(classification_report(y_test, y_pred_xgb, target_names=["存续", "破产"]))
print(f"AUC-ROC: {roc_auc_score(y_test, y_prob_xgb):.4f}")
```

在一次基准运行中，XGBoost 的 AUC-ROC 可达到约 0.9205，是四个模型中最高的。它在 Precision 和 F1 上也较好，但默认 0.5 阈值下破产类 Recall 约为 0.4040。银行若更重视不漏掉破产企业，需要降低阈值或建立分级预警策略。

### 4.2.2.5 四模型统一评估

模型评估需要区分分类能力、排序能力和业务可用性。准确率在本场景中参考价值有限，因为即使把所有企业预测为存续，也能得到约 95% 的准确率。更重要的是破产类 Recall、Precision、F1 和 AUC-ROC。

### 4.2.2.7 阈值选择与业务成本

模型输出的是破产概率，业务系统需要把概率转化为行动。默认阈值 0.5 并不一定适合破产预警。若银行希望尽量不漏掉破产企业，应降低阈值以提高 Recall。若银行人工审核资源有限，应提高阈值以提高 Precision。

```python
thresholds = np.arange(0.1, 0.9, 0.05)
threshold_results = []

for t in thresholds:
    y_pred_t = (y_prob_xgb >= t).astype(int)
    threshold_results.append({
        "阈值": round(t, 2),
        "Precision": round(precision_score(y_test, y_pred_t, zero_division=0), 4),
        "Recall": round(recall_score(y_test, y_pred_t, zero_division=0), 4),
        "F1": round(f1_score(y_test, y_pred_t, zero_division=0), 4),
    })

thr_df = pd.DataFrame(threshold_results)
display(thr_df)
```

`np.arange(0.1, 0.9, 0.05)` 生成多个候选阈值。循环中，若 XGBoost 破产概率大于等于阈值，就预测为破产。`zero_division=0` 防止某些阈值下没有正类预测而导致指标报错。该表可以帮助风控部门观察阈值变化对 Precision、Recall 和 F1 的影响。

```python
plt.figure(figsize=(10, 5))

plt.plot(thr_df["阈值"], thr_df["Recall"], "r-s", label="Recall", markersize=4)
```

这张图展示阈值选择的业务权衡。阈值升高通常会提升 Precision，但降低 Recall。阈值降低通常会提升 Recall，但带来更多误报。企业破产预警系统通常不只设置一个阈值，而是设置多个风险区间。例如，高风险企业进入人工尽调，中风险企业进入观察名单，低风险企业正常处理。

## 4.2.3 Agentic AI 自动化预警策略

企业破产预警系统不仅要训练模型，还要把模型输出转化为可以执行、可以复核、可以追踪的风险处置流程。Agentic AI 在这里不是用生成式模型替代财务风控模型，而是把数据检查、风险评分、阈值判断、解释生成和人工复核编排成一个连续任务链。这样的设计能够减少人工重复操作，同时保留审计线索和人工最终判断。

自动化预警策略应遵循三个原则。第一，模型负责估计概率，不直接决定授信结论。第二，规则负责把概率映射为预警等级，不隐藏阈值依据。第三，Agent 负责编排任务和生成报告，不擅自修改模型分数、阈值和审批结果。

### 4.2.3.1 破产风险评分输出

风险评分是自动化预警的入口。经过训练的 XGBoost 模型可以对每一家企业输出破产概率。概率越高，企业在测试样本中越接近历史破产企业的财务特征组合。概率不是确定性结论，而是排序信号和预警依据。

```python
rng = np.random.default_rng(42)
sample_idx = rng.choice(X_test.shape[0], size=10, replace=False)
sample_scores = xgb_clf.predict_proba(X_test.iloc[sample_idx, :])[:, 1]

warning_report = pd.DataFrame(
    data={
        "破产概率": [f"{s:.4f}" for s in sample_scores],
        "真实状态": ["破产" if yi == 1 else "存续" for yi in y_test[sample_idx]],
    },
    index=sample_idx,
)

display(warning_report)
```

`np.random.default_rng(42)` 用于生成可复现的样本抽取结果。`choice` 从测试集位置中抽取 10 家企业。`predict_proba` 计算每家企业属于破产类的概率。`[:, 1]` 表示取目标变量为 1 的概率。在模型验证阶段，报告可以保留真实状态以便核对模型表现；在线上评分阶段，未来是否破产尚不可知，报告中应使用企业编号、评分日期、客户经理、行业代码和模型版本等业务字段。

### 4.2.3.2 预警等级与业务动作

风险评分需要转化为预警等级。分级的好处是把连续概率转换为可执行动作，使客户经理、审批人员和风险管理部门能够使用同一套语言沟通。阈值不是纯技术参数，而是由历史回测、风险偏好、尽调资源和误判成本共同决定。

```python
def assign_warning_level(prob: float) -> str:
    """将破产概率映射为预警等级。"""
    if prob >= 0.50:
        return "红色预警"
    if prob >= 0.25:
        return "橙色预警"
    if prob >= 0.10:
        return "黄色预警"
    return "绿色观察"


def suggest_action(level: str) -> str:
    """根据预警等级给出处置建议。"""
    action_map = {
        "红色预警": "暂停新增授信，启动人工尽调，复核现金流与债务结构",
        "橙色预警": "降低授信额度，要求补充材料，进入重点监控名单",
        "黄色预警": "维持业务关系，缩短复核周期，跟踪核心财务指标",
        "绿色观察": "正常处理，按常规频率监控",
    }
    return action_map[level]
```

`assign_warning_level` 将概率分为红色、橙色、黄色和绿色四个等级。红色预警表示模型认为企业财务特征与历史破产企业高度相似。橙色预警表示风险已经进入重点关注区间。黄色预警表示企业需要缩短复核周期。绿色观察表示按常规频率监控。`suggest_action` 把等级映射为处置建议，使模型输出能够进入业务流程。

### 4.2.3.3 面向 Agent 的结构化报告

Agentic AI 的输入不应只是一段自然语言，而应包含结构化字段。结构化字段包括企业编号、破产概率、预警等级、建议动作、关键财务指标和模型版本。结构化字段可以被审计和回放，也便于后续接入工单系统、邮件系统或审批系统。

```python
def build_warning_report(
    X_eval: pd.DataFrame,
    y_eval: np.ndarray,
    prob: np.ndarray,
    feature_rank: pd.DataFrame,
    sample_positions: np.ndarray,
) -> pd.DataFrame:
    """构造企业破产预警报告。"""
    key_features = feature_rank.head(3).index.tolist()
    rows = []

    for pos in sample_positions:
        p = float(prob[pos])
        level = assign_warning_level(p)
        row = {
            "样本编号": int(X_eval.index[pos]),
            "破产概率": round(p, 4),
            "预警等级": level,
            "建议动作": suggest_action(level),
            "真实状态": "破产" if y_eval[pos] == 1 else "存续",
        }
        for feat in key_features:
            row[f"关键指标_{feat}"] = round(float(X_eval.iloc[pos][feat]), 4)
        rows.append(row)

    return pd.DataFrame(rows)
```

`build_warning_report` 是预警系统的报告生成函数。`feature_rank.head(3)` 取综合排名最高的三个财务指标。循环中的 `pos` 是测试集中的样本位置。`p` 是破产概率。`level` 是预警等级。`row` 保存一条企业预警记录。关键指标列用于解释模型分数，避免报告只给出一个难以理解的概率。

```python
full_xgb_prob = y_prob_xgb
agent_ready_report = build_warning_report(
    X_eval=X_test,
    y_eval=y_test,
    prob=full_xgb_prob,
    feature_rank=rank_df,
    sample_positions=sample_idx,
)

display(agent_ready_report)
```

`agent_ready_report` 是后续 Agent 编排的标准输入。它已经包含结构化事实、模型分数、预警等级和建议动作。这样的表格可以进一步传递给报告生成模块、人工复核队列或模型监控模块。

### 4.2.3.4 规则 Agent 的编排逻辑

为了说明 Agentic AI 的实现思想，可以把预警系统写成一个简单的规则 Agent。该 Agent 接收结构化预警表，生成文字解释，筛选需要人工复核的企业，并保存审计记录。这里的 Agent 不调用外部系统，因此结果可复现，适合课堂教学。

```python
class BankruptcyWarningAgent:
    """企业破产预警规则 Agent。"""

    def __init__(self, review_levels: tuple[str, ...] = ("红色预警", "橙色预警")) -> None:
        self.review_levels = review_levels

    def explain(self, row: pd.Series) -> str:
        key_parts = []
        for col in row.index:
            if col.startswith("关键指标_"):
                key_parts.append(f"{col.replace('关键指标_', '')}={row[col]}")

        explanation = (
            f"样本 {row['样本编号']} 的预测破产概率为 {row['破产概率']:.4f}，"
            f"预警等级为 {row['预警等级']}。"
            f"建议动作是：{row['建议动作']}。"
        )
        if key_parts:
            explanation += " 需要重点复核的财务指标包括：" + "，".join(key_parts) + "。"
        return explanation

    def run(self, report: pd.DataFrame) -> tuple[pd.DataFrame, pd.DataFrame]:
        result = report.copy()
        result["文字解释"] = result.apply(self.explain, axis=1)
        result["是否进入人工复核"] = result["预警等级"].isin(self.review_levels)

        audit_log = result[["样本编号", "破产概率", "预警等级", "是否进入人工复核"]].copy()
        audit_log["策略版本"] = "bankruptcy_warning_rule_v1"
        return result, audit_log


warning_agent = BankruptcyWarningAgent()
final_warning_report, audit_log = warning_agent.run(agent_ready_report)

display(final_warning_report[["样本编号", "预警等级", "是否进入人工复核", "文字解释"]])
display(audit_log)
```

`BankruptcyWarningAgent` 将预警任务封装为可重复执行的对象。`__init__` 设置哪些等级需要进入人工复核。`explain` 根据一行结构化记录生成自然语言解释。`run` 生成完整报告和审计日志。审计日志保存样本编号、概率、等级、复核标记和策略版本。当模型、阈值或规则发生变化时，策略版本应同步更新，便于日后追踪同一企业为什么在不同时间得到不同处置建议。

### 4.2.3.5 Agentic AI 的边界

Agentic AI 适合承担重复、规范、可审计的辅助任务。例如，它可以读取模型评分表，生成风险摘要，整理尽调问题，检查关键字段是否缺失，并把红色和橙色预警企业推送给人工复核队列。它不适合在没有授权的情况下自动拒绝授信，也不适合绕过模型验证、阈值审批和合规审查。

在企业破产预警场景中，最稳健的架构是模型、规则、Agent 和人工复核的分工协作。模型提供概率。规则提供等级。Agent 提供编排和报告。人工负责最终判断。这样既能利用自动化提高效率，也能避免把高影响金融决策完全交给黑箱系统。
## 本章模型选择的综合讨论

Logistic 回归适合需要监管解释和管理层沟通的场景。它可以清楚说明某个财务指标增加会提高还是降低破产风险。但它表达非线性和交互项的能力有限，面对复杂财务结构时可能欠拟合。

SVM 适合展示最大间隔和核方法思想。在高维、样本量适中的任务中，它可能表现良好。但 RBF-SVM 在大样本上训练成本较高，概率输出需要额外校准，模型解释也不直观。

随机森林是解释性和预测性的折中方案。它能处理非线性和特征交互，对异常值较稳健，调参成本相对较低。但随机森林给出的特征重要性仍然不是因果解释，也不能直接告诉业务人员某个变量的单调影响方向。

XGBoost 通常在结构化表格任务上表现很强。本案例中它的 AUC-ROC 最高，适合作为内部风险排序模型。但 XGBoost 的复杂度更高，需要配合阈值管理、特征解释、模型监控和人工复核，不能只凭一个分数做最终授信决策。

行业指标的作用，是把企业自身财务状态放回经营环境中解释。绝对杠杆率、流动比率和利润率可以描述企业内部风险，但不同行业的资产结构、周转速度和债务使用习惯差异很大。行业相对指标能够回答企业是否显著偏离同业水平。若业务数据中包含行业代码，应优先构造行业中位数差值、行业标准化偏离度、行业分位数和行业历史破产基准率。在建模流程中，原始财务比率提供企业自身画像，行业相对特征提供同业比较视角，两类特征可以共同服务于破产风险排序和预警解释。

## 拓展方向

1.   比较全量预处理与训练集内预处理的评估结果，观察在全量数据上学习填充值和异常值边界是否会造成指标偏乐观。
2.   为缺失率较高的变量增加缺失指示变量，检验缺失本身是否具有破产预警价值。
3.   使用 Logistic 回归的 L1 正则进行特征选择，并与树模型特征重要性比较。
4.   使用交叉验证搜索 XGBoost 的 `max_depth`、`learning_rate`、`n_estimators` 和 `scale_pos_weight`。
5.   使用 PR-AUC 替代 ROC-AUC 作为补充指标。破产样本只有约 5%，PR 曲线往往比 ROC 曲线更能反映少数类识别质量。
6.   为不同阈值计算业务成本。可以设漏判破产企业成本为 L，误报存续企业成本为 C，选择期望成本最低的阈值。
7. 引入 SHAP 解释 XGBoost 的单样本预测，把高风险企业的主要风险因素转化为管理层可读的说明。
8. 补充行业字段后，构造行业分位数特征，比较加入行业相对指标前后的 AUC、Recall 和稳定性。
9. 把本章的结构化预警报告接入后续 Agentic AI 项目，让 LLM 只负责生成报告草稿和尽调问题，不负责改变模型分数和审批结论。
