# 4.6 金融新闻文本分析案例

金融新闻文本分析把非结构化文本转化为可用于投资研究、风险监测和智能决策的结构化信号。新闻标题、正文、摘要、发布机构和发布时间都可能影响市场对上市公司未来现金流、风险暴露和估值水平的判断。与价格序列相比，新闻文本更接近事件本身，但它也更难处理。文本存在同义表达、否定结构、行业术语、夸张标题、重复转载、发布时间滞后和语境依赖等问题。如果直接用关键词计数判断利好或利空，很容易把“亏损收窄”“未达到预期”“监管调查结束”等复杂表达误读。

本章围绕金融新闻情绪识别、事件窗口联动分析和智能 Agent 风险预警展开。数据部分使用 Financial PhraseBank 作为带标签情绪分类样本，使用 FNSPID 金融新闻数据集的 AAPL 子集作为新闻事件来源，并使用 Yahoo Finance 行情作为股价联动分析的数据来源。模型部分比较三条路线。第一条路线是 TF-IDF 特征加朴素贝叶斯分类器。第二条路线是 FinBERT 金融领域预训练模型推理。第三条路线是大语言模型文本嵌入加逻辑回归分类器。事件分析部分把新闻情绪聚合为日度指标，并与股票收益率进行相关性和事件窗口分析。最后，本章构造一个规则型新闻情绪风险 Agent，用于把模型输出转化为可审计的风险提示。

本章的核心目标不是把新闻情绪直接等同于可交易信号，而是建立一套从文本到情绪、从情绪到事件、从事件到风险提示的规范流程。学生需要理解，金融新闻与股价之间往往是相关关系而非直接因果关系。新闻可能引导市场，也可能只是报道市场已经发生的变化。若缺乏严格的时间戳控制、样本外验证和交易成本分析，文本模型很容易被误用为看似智能的投资建议系统。

## 4.6.1 新闻文本分类与舆情分析

### 4.6.1.1 金融新闻情绪分类问题

金融新闻情绪分类通常把文本划分为 positive、neutral 和 negative 三类。positive 表示新闻内容可能传递利好信息，例如利润增长、营收超预期、上调指引或重大合作。negative 表示新闻内容可能传递利空信息，例如亏损扩大、监管调查、评级下调或供应链风险。neutral 表示文本主要是事实陈述、市场综述或难以判断方向的信息。

设文本为 `x_i`，情绪标签为 `y_i`。三分类模型学习条件概率

$$P(y_i=c\mid x_i), \quad c\in\{positive, neutral, negative\}$$

若模型输出三类概率，可以把情绪转化为连续分数

$$s_i=P(positive\mid x_i)-P(negative\mid x_i)$$

其中，`s_i` 越接近 1，表示文本越偏正面；越接近 -1，表示文本越偏负面；接近 0，表示中性或正负情绪相互抵消。

金融文本分类与普通商品评论分类不同。第一，金融语义高度依赖上下文。例如“成本下降”通常是利好，但“收入下降”通常是利空。第二，金融文本中大量表达是相对预期而非绝对数值。例如“profit rose but missed expectations”同时包含增长和不及预期。第三，新闻发布时间与市场反应之间存在交易日、盘前盘后和时区问题。第四，新闻情绪并不一定对应未来收益，因为市场可能已经提前定价。

### 4.6.1.2 为什么同时使用朴素贝叶斯、FinBERT 和 LLM Embedding

朴素贝叶斯是文本分类的经典基线。它把文本表示为词频或 TF-IDF 特征，然后用条件独立假设建模词与类别之间的关系。它的优点是训练速度快、实现简单、可解释性强，并且适合在 CPU 环境下快速建立基线。它的局限是无法理解上下文、语序、否定和复杂金融语义。

FinBERT 是面向金融语境的 BERT 类模型。它能够基于上下文编码文本，并直接输出金融情绪类别。它的优点是对金融语言理解更强，不需要在本章数据上重新训练即可推理。它的局限是模型较大、推理成本高、依赖 GPU 或较长 CPU 推理时间，并且输出仍受预训练语料和标签定义影响。

LLM Embedding 加逻辑回归代表“大模型做特征，小模型做决策”的工程范式。嵌入模型把文本映射到连续向量空间，逻辑回归在这些向量上学习情绪分类边界。它的优点是不需要本地部署大模型，嵌入可以缓存，轻量分类器训练快，适合快速构建业务系统。它的局限是依赖外部 API、存在费用和延迟，模型版本变化可能导致向量分布漂移。

本章选择这三类方法，是为了让学生看到文本建模从传统词袋模型、领域预训练模型到通用大模型嵌入的演进。模型越复杂并不必然越适合生产环境。最终选择要综合考虑精度、速度、成本、隐私、可解释性、可维护性和监管要求。

### 4.6.1.4 带标签数据 Financial PhraseBank

Financial PhraseBank 是金融情绪分类中常用的标注语料。本章使用 `Sentences_AllAgree` 子集，即标注者意见一致的句子。这个子集样本量较小，但标签可靠性较高，适合教学中比较不同文本模型。

```python
import zipfile
from huggingface_hub import hf_hub_download


def load_financial_phrasebank() -> tuple[pd.Series, pd.Series]:
    """加载 Financial PhraseBank 的 Sentences_AllAgree 子集。"""
    # Ref: https://huggingface.co/datasets/financial_phrasebank
    zip_path = hf_hub_download(
        repo_id="financial_phrasebank",
        repo_type="dataset",
        filename="data/FinancialPhraseBank-v1.0.zip",
    )
    with zipfile.ZipFile(zip_path) as zf:
        raw = zf.read(
            "FinancialPhraseBank-v1.0/Sentences_AllAgree.txt"
        ).decode("latin-1").splitlines()

    records = []
    for line in raw:
        sentence, label = line.rsplit("@", 1)
        records.append({"sentence": sentence.strip(), "label": label.strip()})
    df = pd.DataFrame(records)
    return df["sentence"], df["label"]


X_fpb, y_fpb = load_financial_phrasebank()
print(f"Financial PhraseBank 样本数: {len(X_fpb)}")
print(y_fpb.value_counts())
```

`hf_hub_download` 从 HuggingFace 下载压缩包。`zipfile.ZipFile` 读取压缩文件中的 `Sentences_AllAgree.txt`。每一行文本由句子和标签组成，中间用 `@` 分隔。由于句子中可能包含其他符号，代码使用 `rsplit("@", 1)` 从右侧只分割一次。函数返回文本序列 `X_fpb` 和标签序列 `y_fpb`。

运行结果中，样本数通常为 2264 条，其中 neutral 最多，positive 次之，negative 最少。这说明数据存在类别不平衡。若只看 Accuracy，模型可能因为偏向 neutral 而得到较高分数，因此还要关注 macro F1。

### 4.6.1.5 FNSPID 新闻数据与股价数据

FNSPID 是用于金融新闻与股票价格联动研究的大规模数据集。完整新闻数据规模很大，本章使用 Nasdaq-100 过滤子集，并通过 HuggingFace streaming 方式只保留 AAPL 新闻。流式加载的意义在于，程序可以边读边过滤，不必一次性下载和加载完整文件。

```python
from datasets import load_dataset

FNSPID_NEWS_CACHE = Path(f"data/fnspid_news_{TARGET_TICKER.lower()}.parquet")


def load_fnspid_news_streaming(
    ticker: str = TARGET_TICKER,
    cache_path: Path = FNSPID_NEWS_CACHE,
    max_rows: int | None = None,
) -> pd.DataFrame:
    """通过 HuggingFace streaming 加载单只股票的 FNSPID 新闻子集。"""
    # Ref: https://huggingface.co/docs/datasets/en/stream
    # Ref: https://huggingface.co/datasets/benstaf/FNSPID-filtered-nasdaq-100
    if cache_path.exists():
        print(f"从本地缓存加载: {cache_path}")
        df = pd.read_parquet(cache_path)
        if max_rows is not None:
            df = df.head(max_rows)
        return df

    print(f"正在从 HuggingFace 流式加载 {ticker} 的新闻数据...")
    ds = load_dataset(
        "benstaf/FNSPID-filtered-nasdaq-100",
        split="train",
        streaming=True,
    )
    ticker_upper = ticker.upper()
    filtered = ds.filter(
        lambda x: str(x.get("Stock_symbol", "")).upper() == ticker_upper
    )

    rows = []
    for i, row in enumerate(filtered):
        rows.append(row)
        if (i + 1) % 1000 == 0:
            print(f"  已收集 {i + 1} 条...", end="\r")

    if not rows:
        raise ValueError(
            f"未找到 ticker={ticker} 的新闻。"
            "请检查 Stock_symbol 列的取值。"
        )

    df = pd.DataFrame(rows)
    print(f"\n  共获取 {len(df)} 条 {ticker} 新闻")

    cache_path.parent.mkdir(parents=True, exist_ok=True)
    df.to_parquet(cache_path, index=False)
    print(f"  已缓存到: {cache_path}")

    if max_rows is not None:
        df = df.head(max_rows)
    return df
```

函数首先检查本地 Parquet 缓存是否存在。若存在，直接读取缓存，避免重复远程扫描。若不存在，则使用 `load_dataset(..., streaming=True)` 创建流式数据集。`filter` 按 `Stock_symbol` 筛选目标股票。循环把匹配行保存到列表。若没有匹配新闻，则抛出错误。最后将数据写入本地缓存，后续运行会更快。

### 4.6.1.6 文本清洗

文本清洗要根据模型类型选择强度。朴素贝叶斯和 TF-IDF 对大小写、HTML、URL 和稀有符号较敏感，因此通常要做小写化和符号规范化。嵌入模型和 BERT 模型能够处理更多原始语言信息，过度清洗反而可能损失语义。为了与实验保持一致，FinBERT 和朴素贝叶斯示例使用基础小写清洗；后续 LLM Embedding 部分会使用更轻量的清洗方式。

```python
def clean_text(text: str) -> str:
    """基础文本清洗，去 HTML 标签、URL、多余空白并转小写。"""
    if not isinstance(text, str):
        return ""
    text = re.sub(r"<[^>]+>", " ", text)
    text = re.sub(r"http\S+", " ", text)
    text = re.sub(r"[^a-zA-Z0-9\s.,!?'-]", " ", text)
    text = re.sub(r"\s+", " ", text).strip()
    return text.lower()


X_train_clean = X_train.map(clean_text)
X_test_clean = X_test.map(clean_text)

print("清洗前:", X_test.iloc[0][:80])
print("清洗后:", X_test_clean.iloc[0][:80])
```

`re.sub(r"<[^>]+>", " ", text)` 删除 HTML 标签。`re.sub(r"http\S+", " ", text)` 删除 URL。第三个正则表达式保留英文、数字、空白和少量常见标点。最后把连续空白合并为一个空格，并转换为小写。若输入不是字符串，函数返回空字符串，避免后续向量化器报错。

清洗的局限也要明确。删除符号可能损失公司代码、百分号和货币符号中的信息。转小写通常无害，但在某些 ticker、缩写和专有名词中可能减少区分度。因此，在生产系统中应保留原始文本，并为不同模型维护不同的清洗版本。

### 4.6.1.7 朴素贝叶斯文本分类基线

朴素贝叶斯把文本看作词的集合或词频向量。多项式朴素贝叶斯的基本形式为

$$P(c\mid x)\propto P(c)\prod_{j=1}^{d}P(w_j\mid c)^{x_j}$$

其中，`c` 是类别，`w_j` 是词项，`x_j` 是词项在文本中的频率或权重。朴素贝叶斯的“朴素”之处在于假设给定类别后各词条件独立。这个假设在自然语言中显然不完全成立，但它使模型非常高效。

```python
def build_nb_pipeline(
    vectorizer_type: str = "tfidf",
    alpha: float = 0.1,
    max_features: int = 10000,
) -> Pipeline:
    """构建朴素贝叶斯文本分类管道。"""
    if vectorizer_type == "tfidf":
        vec = TfidfVectorizer(max_features=max_features, stop_words="english")
    else:
        vec = CountVectorizer(max_features=max_features, stop_words="english")
    return Pipeline([
        ("vectorizer", vec),
        ("nb", MultinomialNB(alpha=alpha)),
    ])


nb_model = build_nb_pipeline("tfidf", alpha=0.1)
t0 = perf_counter()
nb_model.fit(X_train_clean, y_train)
nb_fit_time = perf_counter() - t0
print(f"朴素贝叶斯训练耗时: {nb_fit_time:.4f}s")

nb_preds = nb_model.predict(X_test_clean)
print("\n=== 朴素贝叶斯 分类报告 ===")
print(classification_report(y_test, nb_preds, digits=4))
```

`TfidfVectorizer` 把文本转为 TF-IDF 特征。TF-IDF 由词频和逆文档频率组成，它降低高频普通词的权重，提高更具区分度词项的影响。`max_features=10000` 限制词表规模，避免特征过多。`stop_words="english"` 删除英文停用词。`MultinomialNB(alpha=0.1)` 使用平滑参数 `alpha`，防止某些词在某类别中未出现而导致概率为零。

实验结果中，朴素贝叶斯通常训练极快，Accuracy 约为 0.82，macro F1 约为 0.72。它对 neutral 类识别较好，但对 negative 类召回率较低。这说明词袋模型能抓住一些明显情绪词，但对复杂负面金融表达的覆盖不足。

### 4.6.1.8 FinBERT 推理

FinBERT 直接输入文本，输出 positive、negative 和 neutral 三类概率。本章只做推理，不做微调。这样可以把 FinBERT 理解为一个已经训练好的金融情绪识别器。

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification

FINBERT_MODEL_NAME = "ProsusAI/finbert"
# Ref: https://huggingface.co/ProsusAI/finbert

print(f"加载 FinBERT 模型到 {DEVICE}...")
finbert_tokenizer = AutoTokenizer.from_pretrained(FINBERT_MODEL_NAME)
finbert_model = AutoModelForSequenceClassification.from_pretrained(
    FINBERT_MODEL_NAME
).to(DEVICE)
finbert_model.eval()

FINBERT_LABELS = ["positive", "negative", "neutral"]
```

`AutoTokenizer` 负责把文本切分为 BERT 可处理的 token，并生成 attention mask。`AutoModelForSequenceClassification` 加载带分类头的 FinBERT 模型。`.to(DEVICE)` 把模型移动到 GPU 或 CPU。`eval()` 切换到推理模式，关闭 dropout 等训练行为。`FINBERT_LABELS` 记录模型输出维度与标签名称的对应关系。

```python
def finbert_predict_batch(
    texts: list[str],
    tokenizer=finbert_tokenizer,
    model=finbert_model,
    device=DEVICE,
    batch_size: int = 64,
) -> tuple[list[str], list[dict]]:
    """批量 FinBERT 推理，返回预测标签和三类概率。"""
    all_labels = []
    all_scores = []

    for i in range(0, len(texts), batch_size):
        batch_texts = texts[i : i + batch_size]
        inputs = tokenizer(
            batch_texts,
            padding=True,
            truncation=True,
            max_length=512,
            return_tensors="pt",
        ).to(device)

        with torch.no_grad():
            outputs = model(**inputs)
            probs = torch.softmax(outputs.logits, dim=-1)

        for j in range(len(batch_texts)):
            prob_dict = {
                FINBERT_LABELS[k]: float(probs[j, k])
                for k in range(len(FINBERT_LABELS))
            }
            pred_idx = int(probs[j].argmax())
            all_labels.append(FINBERT_LABELS[pred_idx])
            all_scores.append(prob_dict)

    return all_labels, all_scores
```

函数按批次推理。`padding=True` 把同一批次文本补齐到相同长度。`truncation=True` 表示文本超过最大长度时截断。`max_length=512` 是 BERT 类模型的常见最大输入长度。`torch.no_grad()` 表示不计算梯度，降低显存和时间消耗。`torch.softmax` 把 logits 转为概率。每条文本的预测标签为概率最大类别，概率字典保留后续情绪分数计算所需的信息。

```python
test_texts = X_test_clean.tolist()

t0 = perf_counter()
finbert_preds, finbert_scores = finbert_predict_batch(test_texts)
finbert_time = perf_counter() - t0
print(f"FinBERT 推理耗时: {finbert_time:.2f}s ({len(test_texts)} 条)")
print(f"  每条平均: {finbert_time / len(test_texts) * 1000:.1f}ms")
```

这段代码在 PhraseBank 测试集上运行 FinBERT。记录耗时可以帮助学生理解精度与速度之间的权衡。GPU 环境下 453 条句子通常可以在较短时间内完成；CPU 环境可能慢得多。


### 4.6.1.10 Embedding 版本的文本清洗

嵌入模型对标点、大小写和语序更敏感，也更能利用这些信息。因此清洗不应过度。

```python
def clean_text_for_embedding(text: str) -> str:
    """轻量清洗，只去噪并保留语义信息。"""
    if not isinstance(text, str):
        return ""
    text = re.sub(r"<[^>]+>", " ", text)
    text = re.sub(r"http\S+", " ", text)
    text = re.sub(r"\s+", " ", text).strip()
    return text


def clean_text_bow(text: str) -> str:
    """词袋模型使用更激进的清洗。"""
    text = clean_text_for_embedding(text)
    text = re.sub(r"[^a-zA-Z0-9\s.,!?'-]", " ", text)
    return re.sub(r"\s+", " ", text).strip().lower()


X_train_embed_clean = X_train.map(clean_text_for_embedding)
X_test_embed_clean = X_test.map(clean_text_for_embedding)
X_train_bow = X_train.map(clean_text_bow)
X_test_bow = X_test.map(clean_text_bow)

print("嵌入版清洗示例:", X_test_embed_clean.iloc[0][:80])
print("词袋版清洗示例:", X_test_bow.iloc[0][:80])
```

`clean_text_for_embedding` 保留大小写和大部分标点。`clean_text_bow` 则适合 TF-IDF 模型。把两套清洗方式分开，是为了避免用一种预处理规则强行适配所有模型。

### 4.6.1.12 批量嵌入与本地缓存

API 调用有费用和延迟，重复请求同一文本是不必要的。缓存的基本思想是用文本哈希作为键，把嵌入向量存到本地文件。

```python
CACHE_DIR = Path("data/embedding_cache")
CACHE_DIR.mkdir(parents=True, exist_ok=True)


def _text_hash(text: str) -> str:
    """用 SHA256 生成文本指纹。"""
    return hashlib.sha256(text.encode("utf-8")).hexdigest()


def _load_cache(cache_file: Path) -> dict[str, np.ndarray]:
    """从 npz 文件读取嵌入缓存。"""
    if not cache_file.exists():
        return {}
    data = np.load(cache_file, allow_pickle=False)
    return {k: data[k] for k in data.files}


def _save_cache(cache: dict[str, np.ndarray], cache_file: Path) -> None:
    """把嵌入缓存写入 npz 文件。"""
    np.savez(cache_file, **cache)
```

`_text_hash` 对文本求 SHA256，确保相同文本总是映射到同一个键。`_load_cache` 读取已有 `.npz` 文件。`_save_cache` 把字典写回本地。这个缓存方式简单可靠，但不适合并发写入。生产环境可以使用 SQLite、对象存储或向量数据库管理缓存。

```python
from tenacity import retry, stop_after_attempt, wait_exponential


@retry(stop=stop_after_attempt(5), wait=wait_exponential(min=1, max=30))
def _embed_batch_api(texts: list[str]) -> list[np.ndarray]:
    """调用一次嵌入 API。"""
    # Ref: https://www.alibabacloud.com/help/zh/model-studio/embedding-interfaces-compatible-with-openai
    resp = client.embeddings.create(
        model=EMBED_MODEL_NAME,
        input=texts,
        dimensions=EMBED_DIM,
        encoding_format="float",
    )
    return [np.array(d.embedding, dtype=np.float32) for d in resp.data]
```

`tenacity.retry` 用于失败重试。网络抖动、服务限流和临时错误在 API 调用中很常见，重试机制可以提高稳定性。`wait_exponential` 使用指数退避，避免短时间内连续重试造成更严重限流。

```python
def embed_texts(
    texts: list[str],
    cache_name: str,
    max_chars: int = 8000,
    verbose: bool = True,
) -> np.ndarray:
    """批量嵌入并缓存，返回 shape=(N, EMBED_DIM) 的矩阵。"""
    cache_file = CACHE_DIR / f"{cache_name}.npz"
    cache = _load_cache(cache_file)

    processed = [(t or " ")[:max_chars] for t in texts]
    keys = [_text_hash(t) for t in processed]

    todo_idx = [i for i, k in enumerate(keys) if k not in cache]
    if verbose:
        print(f"[{cache_name}] 总计 {len(texts)} 条，"
              f"命中缓存 {len(texts) - len(todo_idx)} 条，"
              f"待请求 {len(todo_idx)} 条")

    for start in range(0, len(todo_idx), EMBED_BATCH_SIZE):
        batch_idx = todo_idx[start : start + EMBED_BATCH_SIZE]
        batch_texts = [processed[i] for i in batch_idx]
        vecs = _embed_batch_api(batch_texts)
        for i, v in zip(batch_idx, vecs):
            cache[keys[i]] = v
        if verbose and (start // EMBED_BATCH_SIZE) % 20 == 0:
            print(f"  进度 {start + len(batch_idx)}/{len(todo_idx)}", end="\r")

    if todo_idx:
        _save_cache(cache, cache_file)
        if verbose:
            print(f"\n  已写入缓存: {cache_file}")

    return np.stack([cache[k] for k in keys], axis=0)
```

`processed` 对文本进行空值处理和长度截断。`keys` 是每条文本的缓存键。`todo_idx` 找出尚未缓存的文本索引。循环按 `EMBED_BATCH_SIZE` 分批请求 API。请求完成后写入缓存。最后按原始输入顺序组装矩阵。这里把文本截断到 8000 字符，是为了控制 token 成本和接口限制。若使用新闻正文，截断策略会影响语义，应根据任务决定使用标题、摘要还是正文。

## 4.6.2 事件驱动模型设计

### 4.6.2.1 从单条新闻情绪到日度情绪指标

事件驱动分析需要把新闻级别的情绪转化为日期级别的指标。设第 t 日有 `n_t` 条新闻，每条新闻情绪分数为 `s_{t,i}`，日度平均情绪定义为

$$S_t=\frac{1}{n_t}\sum_{i=1}^{n_t}s_{t,i}$$

其中

$$s_{t,i}=P_{t,i}(positive)-P_{t,i}(negative)$$

新闻数量也很重要。若某天只有一条新闻，平均情绪容易受单条文本影响。若某天有大量新闻且情绪方向一致，说明事件关注度更高。因此本章保留 `news_count` 作为新闻强度指标。

Pearson 相关系数衡量线性相关关系，定义为

$$\rho_{S,R}=\frac{Cov(S_t,R_t)}{\sigma_S\sigma_R}$$

若相关系数为正，说明情绪较高的日期往往收益率也较高。若相关系数接近 0，说明线性同日关系较弱。若相关系数为负，说明正面情绪与当日收益反向变化。实验中可能得到较高或较低相关系数，这取决于样本期、新闻来源、时间对齐方式和市场状态。不能把一个样本中的相关系数解释为稳定因果关系。

### 4.6.2.5 事件窗口分析

事件窗口分析关注事件日前后若干天的异常收益。设事件日为 0，窗口为 `[-L,+L]`。本章实验代码把样本平均收益作为正常收益基准，异常收益为

$$AR_{t}=R_t-\bar{R}$$

若要计算累计异常收益，应该进一步求和

$$CAR(a,b)=\sum_{\tau=a}^{b}AR_{\tau}$$

需要特别说明，下面程序输出的是各相对偏移日的平均异常收益率，而不是严格意义上的累计异常收益率。若教材或报告称其为 CAR，应在解释中修正为“平均异常收益率曲线”，或增加累积步骤。

```python
def event_window_analysis(
    df: pd.DataFrame,
    window: int = 2,
    quantile: float = 0.05,
) -> pd.DataFrame | None:
    """简易事件窗口分析。"""
    if len(df) < 50:
        print("数据量不足，需要至少 50 条合并记录才能进行事件窗口分析。")
        return None

    df = df.sort_values("date").reset_index(drop=True)
    avg_ret = df["return"].mean()

    pos_threshold = df["avg_sentiment"].quantile(1 - quantile)
    neg_threshold = df["avg_sentiment"].quantile(quantile)

    results = []
    for event_type, mask in [
        ("极正面", df["avg_sentiment"] >= pos_threshold),
        ("极负面", df["avg_sentiment"] <= neg_threshold),
    ]:
        event_indices = df[mask].index.tolist()
        car_by_offset = {off: [] for off in range(-window, window + 1)}

        for idx in event_indices:
            for off in range(-window, window + 1):
                target = idx + off
                if 0 <= target < len(df):
                    ar = df.loc[target, "return"] - avg_ret
                    car_by_offset[off].append(ar)

        for off, ars in car_by_offset.items():
            if ars:
                results.append({
                    "事件类型": event_type,
                    "偏移天数": off,
                    "平均异常收益率": np.mean(ars),
                    "事件数": len(ars),
                })

    return pd.DataFrame(results)
```

函数首先检查样本量。少于 50 条合并记录时，极端分位事件数量太少，分析不稳定。`quantile=0.05` 表示把情绪最高 5% 的日期定义为极正面事件，把情绪最低 5% 的日期定义为极负面事件。`avg_ret` 是正常收益基准。对每个事件日，函数收集事件日前后两天的异常收益，并按偏移日求平均。

这种事件窗口有明显简化。第一，它没有使用市场模型或行业基准计算异常收益。第二，它没有处理多个事件窗口重叠。第三，它没有考虑新闻发布时间。第四，它只考察已聚合的日度情绪，而不是单条重大新闻。因此，它适合教学演示，不足以支持严肃投资结论。

```python
car_df = event_window_analysis(merged)
if car_df is not None:
    print(car_df.to_string(index=False))

    fig, ax = plt.subplots(figsize=(8, 5))
    for evt_type, grp in car_df.groupby("事件类型"):

    ax.axvline(0, color="gray", linestyle=":", linewidth=0.8)

    ax.set_ylabel("平均异常收益率")
    ax.set_title(f"{TARGET_TICKER} 情绪事件窗口分析")
    ax.legend()
    plt.tight_layout()
    plt.show()
```

输出表展示极正面和极负面事件在 `-2,-1,0,+1,+2` 天的平均异常收益。若极正面事件在事件日附近异常收益为正，极负面事件在事件日附近异常收益为负，说明情绪指标与价格变化存在一定同向关系。但这仍然不是因果证明。若事件日前就出现异常收益，可能说明市场先动、新闻后报道，或新闻情绪反映了此前已发生的信息扩散。

### 4.6.2.7 为什么不用简单关键词或直接回归价格

关键词方法容易解释，但不能处理复杂语义。例如“not bad”与“bad”方向相反，“profit growth slowed”既有增长也有放缓，“beats expectations”中的 expectations 是关键信息。金融文本中真正有价值的情绪往往来自上下文和预期差，而不是单词本身。因此本章没有采用简单正负词典作为主方法。

直接用情绪分数回归当日价格或未来价格也不理想。股票价格是非平稳序列，直接拟合价格容易得到误导性高精度。更合理的做法是研究收益率、异常收益率、方向标签或风险事件概率。事件窗口分析比直接拟合价格更适合展示新闻冲击与市场反应。

本章的事件分析仍然是简化版本。它适合作为理解新闻情绪与市场联动的第一步，但不能替代完整的因果识别。若要判断情绪是否领先收益，应进一步使用滞后相关、Granger 因果检验、高频时间戳、盘前盘后归因和控制变量模型。

## 4.6.3 智能 Agent 预测市场情绪与风险

### 4.6.3.1 Agent 在新闻舆情系统中的定位

新闻舆情 Agent 不应直接根据一条新闻自动买卖股票。新闻情绪模型存在误判，新闻与股价关系也不稳定。Agent 更适合承担监控、汇总、诊断、预警和报告生成任务。

在本章场景中，Agent 的输入包括日度平均情绪、新闻数量、股票收益率、极端情绪阈值和事件窗口结果。Agent 的输出包括市场情绪状态、风险等级、触发原因、建议复核事项和审计日志。它的作用是把模型输出转化为业务人员能理解的风险提示，而不是绕过人工和风控流程直接执行交易。

### 4.6.3.2 构造 Agent 所需特征

```python
def build_sentiment_monitoring_table(
    merged_df: pd.DataFrame,
    sentiment_window: int = 20,
    volume_window: int = 20,
) -> pd.DataFrame:
    """构造新闻情绪风险监控表。"""
    monitor = merged_df.sort_values("date").copy()
    monitor["sentiment_ma"] = (
        monitor["avg_sentiment"].rolling(sentiment_window, min_periods=5).mean()
    )
    monitor["sentiment_std"] = (
        monitor["avg_sentiment"].rolling(sentiment_window, min_periods=5).std()
    )
    monitor["sentiment_z"] = (
        (monitor["avg_sentiment"] - monitor["sentiment_ma"])
        / monitor["sentiment_std"].replace(0, np.nan)
    )
    monitor["news_count_ma"] = (
        monitor["news_count"].rolling(volume_window, min_periods=5).mean()
    )
    monitor["news_intensity"] = monitor["news_count"] / monitor["news_count_ma"]
    monitor["abs_return"] = monitor["return"].abs()
    return monitor


monitor_df = build_sentiment_monitoring_table(merged)
display(monitor_df.tail())
```

`sentiment_ma` 是滚动平均情绪，表示近期舆情基准。`sentiment_std` 是滚动情绪波动。`sentiment_z` 衡量当日情绪相对于近期基准的偏离程度。若 `sentiment_z < -2`，说明当日情绪明显低于近期水平。`news_count_ma` 是近期平均新闻数量。`news_intensity` 表示当日新闻数量相对平时是否异常放大。`abs_return` 用于观察当日市场波动是否同步放大。

使用 z-score 而不是固定情绪阈值，是因为不同公司和不同时期的新闻情绪均值不同。某些公司长期新闻偏正面，某些时期市场整体偏负面。滚动标准化可以更好地识别相对异常。

### 4.6.3.3 规则型新闻风险 Agent

```python
class NewsSentimentRiskAgent:
    """基于新闻情绪和收益率联动的规则型风险 Agent。"""

    def __init__(
        self,
        monitor_table: pd.DataFrame,
        policy_version: str = "news_sentiment_risk_policy_v1",
        z_warning: float = 1.5,
        z_critical: float = 2.0,
        intensity_warning: float = 1.5,
        return_warning: float = 0.03,
    ) -> None:
        self.monitor_table = monitor_table.copy()
        self.policy_version = policy_version
        self.z_warning = z_warning
        self.z_critical = z_critical
        self.intensity_warning = intensity_warning
        self.return_warning = return_warning
```

`NewsSentimentRiskAgent` 接收监控表和规则参数。`z_warning` 是一般情绪异常阈值。`z_critical` 是严重情绪异常阈值。`intensity_warning` 表示新闻数量相对均值放大到多少倍时触发关注。`return_warning` 表示当日收益率绝对值超过多少时触发市场波动提示。

这些阈值不是放之四海而皆准。高波动股票、低波动股票、金融机构、科技公司和宏观新闻敏感资产应使用不同阈值。正式系统中，阈值应通过历史回测、业务经验和风控政策共同确定。

```python
    def score_latest_day(self) -> dict:
        """对最新一个交易日生成风险评分。"""
        clean = self.monitor_table.dropna(subset=["avg_sentiment", "sentiment_z"])
        if clean.empty:
            return {
                "风险等级": "无法判断",
                "原因": "可用情绪历史不足",
                "建议": "继续收集新闻并检查数据管道",
            }

        latest = clean.iloc[-1]
        reasons = []
        risk_score = 0

        if latest["sentiment_z"] <= -self.z_critical:
            risk_score += 3
            reasons.append("负面情绪显著低于近期水平")
        elif latest["sentiment_z"] <= -self.z_warning:
            risk_score += 2
            reasons.append("负面情绪低于近期水平")
        elif latest["sentiment_z"] >= self.z_critical:
            risk_score += 1
            reasons.append("正面情绪显著高于近期水平，需关注过度乐观")

        if latest.get("news_intensity", np.nan) >= self.intensity_warning:
            risk_score += 1
            reasons.append("新闻数量明显放大")

        if latest.get("abs_return", np.nan) >= self.return_warning:
            risk_score += 1
            reasons.append("股价波动同步放大")

        if risk_score >= 4:
            level = "高风险"
        elif risk_score >= 2:
            level = "中风险"
        elif risk_score >= 1:
            level = "低风险关注"
        else:
            level = "正常"
            reasons.append("未触发主要舆情风险规则")

        return {
            "日期": latest["date"],
            "风险等级": level,
            "风险分数": risk_score,
            "平均情绪": round(float(latest["avg_sentiment"]), 4),
            "情绪Z分数": round(float(latest["sentiment_z"]), 4),
            "新闻强度": round(float(latest.get("news_intensity", np.nan)), 4),
            "当日收益率": round(float(latest["return"]), 4),
            "原因": "；".join(reasons),
            "建议": "复核当日新闻标题、来源、发布时间和是否存在重大公司事件。",
        }
```

`score_latest_day` 取最新可用日期并根据规则评分。负面情绪显著异常的权重最高，因为风险管理通常更关注下行风险。正面情绪过强只加 1 分，是因为它可能代表利好，也可能代表过度乐观或拥挤预期。新闻数量放大和收益率波动放大作为辅助信号。

该方法体现了规则型 Agent 的可审计性。每个风险等级都有明确触发原因。业务人员可以检查规则是否合理，而不是面对黑箱结论。

```python
    def summarize_recent_events(self, lookback_days: int = 10) -> pd.DataFrame:
        """输出近期异常舆情日期。"""
        recent = self.monitor_table.dropna(subset=["sentiment_z"]).tail(lookback_days).copy()
        if recent.empty:
            return pd.DataFrame()

        recent["是否负面异常"] = recent["sentiment_z"] <= -self.z_warning
        recent["是否新闻放量"] = recent["news_intensity"] >= self.intensity_warning
        recent["是否价格异动"] = recent["abs_return"] >= self.return_warning
        cols = [
            "date", "avg_sentiment", "sentiment_z", "news_count",
            "news_intensity", "return", "是否负面异常", "是否新闻放量", "是否价格异动",
        ]
        return recent[cols].reset_index(drop=True)
```

`summarize_recent_events` 返回最近若干日期的监控状态，便于分析风险是否持续。单日异常可能只是噪声，连续多日负面情绪和新闻放量更值得关注。

```python
    def build_audit_log(self) -> pd.DataFrame:
```

### 4.6.3.4 为什么不让 LLM 直接判断市场风险

可以把新闻标题发送给通用 LLM，让它直接输出“买入、卖出、观望”或“高风险、低风险”。但这种方式不适合作为金融风控主流程。第一，LLM 输出可能不稳定，同一输入在不同提示词或模型版本下得到不同结论。第二，LLM 很难自动保证时间戳、新闻重复、样本外验证和风险阈值的一致性。第三，直接交易建议涉及合规风险。第四，LLM 可能给出看似合理但缺乏统计验证的解释。

本章选择规则型 Agent，是因为它透明、可审计、可复现。LLM 可以作为辅助工具，用于摘要新闻、解释风险原因、生成报告草稿和提醒人工复核要点。但最终风险等级应由可检查的数据指标和规则驱动，交易决策应由受控系统和人工流程共同决定。

## 本章模型选择的综合讨论

朴素贝叶斯是低成本基线。它训练快、部署简单，适合在数据和算力有限时建立初始系统。它的缺点是语义理解弱，容易漏掉金融语境中的复杂负面表达。因此它适合作为基线，不适合作为唯一生产模型。

FinBERT 是金融新闻情绪分类的强方法。它利用金融领域预训练知识，能更好识别专业表达和上下文关系。若机构具备 GPU 或可接受推理成本，FinBERT 适合用于高精度情绪打分。它的局限是模型推理更重，长文本需要截断，且领域模型仍可能对新型事件、讽刺标题和跨市场新闻不敏感。

LLM Embedding 加逻辑回归是一种工程上灵活的方法。它把大模型语义能力转化为可缓存向量，再由轻量分类器完成业务标签映射。它适合无本地 GPU、希望快速接入云端语义能力的场景。它的主要风险是 API 成本、数据隐私、模型版本漂移和外部依赖。

事件窗口分析是连接文本和市场的第一步，但它不能证明因果。情绪与收益同日相关可能来自新闻影响市场，也可能来自市场变化引发新闻报道。若要提升严谨性，应考虑盘前盘后映射、市场模型异常收益、多股票面板数据、滞后变量、Granger 因果检验、工具变量或高频事件研究。

Agentic AI 的价值在于流程化和治理化。它可以自动汇总新闻情绪、识别异常日期、生成风险提示、记录审计日志和提醒人工复核。它不应该绕过模型评估、合规审查和交易风控直接下单。

## 拓展方向

1. 将自然日期新闻映射到交易日。盘后新闻归入下一交易日，周末新闻归入下一个交易日。
2. 使用正文、摘要和标题分别构建情绪指标，比较它们与收益率的关系。
3. 对 FinBERT 进行小规模微调，观察在 FNSPID 或机构自有新闻标签上的表现变化。
4. 用市场模型计算异常收益，例如用标普 500 或行业指数作为基准。
5. 在事件窗口中加入置信区间和显著性检验，避免只凭图形判断。
6. 对多只股票构建面板数据，比较公司规模、行业和新闻数量对情绪冲击的影响。
7. 使用滞后情绪预测未来收益方向，采用 Walk-Forward Validation 避免前视偏差。
8. 引入新闻来源权重，对权威媒体、公告、转载文章和分析师评论设置不同权重。
9. 使用去重和相似度聚类，减少同一新闻被多家媒体转载造成的重复计数。
10. 将新闻嵌入存入向量数据库，用于相似历史事件检索、RAG 报告生成和异常新闻检测。
11. 在 Agent 中加入人工反馈机制，把误报和漏报记录为后续模型更新数据。
12. 构建合规审计模块，记录新闻原文、模型版本、情绪分数、触发规则和人工处置结果。
